# 多处理器启动

xv6 的多处理器（SMP）启动流程：BSP（引导处理器）完成初始化后，通过 STARTUP IPI 逐个唤醒 AP（应用处理器）。

## 核心机制

### MP 配置探测

`mp.c` 中的 `mpsearch()` 在三个位置搜索 Intel MP Floating Pointer Structure：
1. EBDA（扩展 BIOS 数据区）的第一个 KB
2. 系统基本内存的最后一个 KB
3. BIOS ROM 0xE0000–0xFFFFF 区域

```c
// mp.c:38 — 搜索 _MP_ 签名并校验
static struct mp* mpsearch1(uint a, int len) {
  for(p = addr; p < e; p += sizeof(struct mp))
    if(memcmp(p, "_MP_", 4) == 0 && sum(p, sizeof(struct mp)) == 0)
      return (struct mp*)p;
}
```

### BSP 启动 AP 流程

`main.c` 中的 `startothers()` 负责唤醒所有 AP：

1. 将 `entryother.S` 代码拷贝到物理地址 0x7000
2. 为每个 AP 分配内核栈、设置入口函数 `mpenter`、写入页目录地址
3. 通过 `lapicstartap()` 发送 STARTUP IPI
4. 忙等 AP 设置 `started` 标志

```c
// main.c:79 — startothers 核心循环
stack = kalloc();
*(void**)(code-4) = stack + KSTACKSIZE;        // 栈顶
*(void(**)(void))(code-8) = mpenter;            // 入口
*(int**)(code-12) = (void *) V2P(entrypgdir);   // 页目录
lapicstartap(c->apicid, V2P(code));
```

### AP 执行路径

AP 从 `entryother.S` 开始，实模式 → 保护模式 → 开启分页 → 跳转到 `mpenter()`：

```
entryother.S (实模式)
  → 切换保护模式
  → 开启分页（使用 entrypgdir）
  → 跳转 mpenter()（C 代码）
    → switchkvm() 切换到内核页表
    → seginit() 初始化段描述符
    → lapicinit() 初始化本地 APIC
    → mpmain() → idtinit() → scheduler()
```

### 数据结构

```c
// proc.h — 每个 CPU 的状态
struct cpu {
  uchar apicid;
  struct context *scheduler;
  struct taskstate ts;
  struct segdesc gdt[NSEGS];
  struct proc *proc;        // 当前运行的进程
  int started;              // AP 是否已完成启动
  // ...
};

struct cpu cpus[NCPU];  // 所有 CPU 实例
int ncpu;               // 检测到的 CPU 数量
```

## 关键要点
> xv6 的 MP 启动是「BSP 主导」模型：BSP 完成所有初始化后，将一小段实模式代码拷贝到低地址，通过 LAPIC 发送 IPI 唤醒每个 AP，AP 自行切换到保护模式并进入调度器。
