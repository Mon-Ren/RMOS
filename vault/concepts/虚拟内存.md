# 虚拟内存

虚拟内存是操作系统提供的抽象层，让每个进程拥有独立的地址空间，物理内存的分配和保护由内核透明管理。

## 核心思想

```
进程看到的地址（虚拟地址） ≠ 物理内存地址
```

- 每个进程拥有 **0 ~ 2^32（32位）** 的虚拟地址空间
- CPU 发出虚拟地址 → MMU 翻译 → 物理地址
- 翻译失败 → **缺页中断（Page Fault）** → 内核处理

## 三大功能

### 1. 地址隔离

进程 A 写地址 `0x8000` 和进程 B 写地址 `0x8000` 映射到不同的物理页。一个进程崩溃不影响其他进程。

### 2. 内存超卖

所有进程的虚拟内存总和可以超过物理内存。不常用的页换出到磁盘（swap），需要时换入。

### 3. 内存保护

页表项包含权限位：读/写/执行、用户/内核。非法访问触发异常。

## 地址翻译过程

```
虚拟地址 [31:22] → 页目录索引 → 页目录项 → 页表基址
虚拟地址 [21:12] → 页表索引   → 页表项   → 物理页基址
虚拟地址 [11:0]  → 页内偏移 → 物理地址
```

xv6 使用二级页表（页目录 + 页表），每级 1024 项，每页 4KB。

## 缺页中断处理

```
CPU 检查 PTE 的 Present 位
  ├─ Present=1 → 正常翻译
  └─ Present=0 → 触发 Page Fault (#14)
                  ├─ 内核检查：合法地址？
                  │   ├─ 是：分配物理页，更新 PTE，重试指令
                  │   └─ 否：发送 SIGSEGV，终止进程
                  └─ 可能是：按需分配（lazy allocation）、COW、swap 换入
```

## 关键概念

| 概念 | 说明 |
|------|------|
| **页（Page）** | 虚拟地址空间的分块，通常 4KB |
| **页帧（Frame）** | 物理内存的分块，大小与页相同 |
| **PTE** | 页表项，存储物理页号 + 权限位 |
| **TLB** | 页表缓存，加速地址翻译 |
| **Working Set** | 进程活跃使用的页集合，决定驻留内存需求 |
| **Thrashing** | 频繁换入换出导致性能崩溃 |

## xv6 中的实现

xv6 的虚拟内存子系统由 `vm.c` 实现：

- **`setupkvm()`** — 建立内核虚拟地址映射（恒等映射 + 设备映射）
- **`copyuvm()`** — fork 时深拷贝用户页表
- **`allocuvm()`** — sbrk 扩展用户内存
- **`deallocuvm()`** — 收缩用户内存

xv6 **不支持 swap**，物理内存用完就 `panic`。这是教学简化。

## 关键要点

> 虚拟内存是现代 OS 的基石。它不仅是"内存不够用磁盘"，更重要的是提供了**隔离、保护和灵活性**。没有虚拟内存，多任务操作系统几乎不可能实现。
