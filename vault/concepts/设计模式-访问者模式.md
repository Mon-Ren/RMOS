# Visitor 访问者模式

在不修改元素类的前提下，定义作用于这些元素的新操作。

## 意图与场景

- **核心意图**：将数据结构与数据操作分离，新增操作不影响已有类
- **典型场景**：
  - AST 遍历与变换（解释器、编译器）
  - 文档导出为多种格式（HTML / JSON / Text）
  - 文件系统遍历（不同类型文件的不同处理）
  - 报表生成
  - 序列化/反序列化

## C++ 实现

### 经典实现：accept() + Double Dispatch

```cpp
#include <memory>
#include <vector>
#include <string>
#include <iostream>

// 前置声明
class Circle;
class Rectangle;
class Triangle;

// Visitor 接口：每种可访问元素一个 visit 方法
class ShapeVisitor {
public:
    virtual ~ShapeVisitor() = default;
    virtual void visit(const Circle& c) = 0;
    virtual void visit(const Rectangle& r) = 0;
    virtual void visit(const Triangle& t) = 0;
};

// 元素基类
class Shape {
public:
    virtual ~Shape() = default;
    virtual void accept(ShapeVisitor& v) const = 0;
};

// 具体元素
class Circle : public Shape {
    double radius_;
public:
    explicit Circle(double r) : radius_(r) {}
    [[nodiscard]] double radius() const { return radius_; }
    void accept(ShapeVisitor& v) const override { v.visit(*this); }
};

class Rectangle : public Shape {
    double width_, height_;
public:
    Rectangle(double w, double h) : width_(w), height_(h) {}
    [[nodiscard]] double width() const { return width_; }
    [[nodiscard]] double height() const { return height_; }
    void accept(ShapeVisitor& v) const override { v.visit(*this); }
};

class Triangle : public Shape {
    double a_, b_, c_;
public:
    Triangle(double a, double b, double c) : a_(a), b_(b), c_(c) {}
    [[nodiscard]] double a() const { return a_; }
    [[nodiscard]] double b() const { return b_; }
    [[nodiscard]] double c() const { return c_; }
    void accept(ShapeVisitor& v) const override { v.visit(*this); }
};

// 具体 Visitor：计算面积
class AreaCalculator : public ShapeVisitor {
    double area_ = 0;
public:
    void visit(const Circle& c) override {
        area_ = 3.14159265 * c.radius() * c.radius();
    }
    void visit(const Rectangle& r) override {
        area_ = r.width() * r.height();
    }
    void visit(const Triangle& t) override {
        double s = (t.a() + t.b() + t.c()) / 2;
        area_ = std::sqrt(s * (s - t.a()) * (s - t.b()) * (s - t.c()));
    }
    [[nodiscard]] double area() const { return area_; }
};

// 具体 Visitor：导出 SVG
class SvgExporter : public ShapeVisitor {
    std::string svg_;
public:
    void visit(const Circle& c) override {
        svg_ += "<circle r=\"" + std::to_string(c.radius()) + "\"/>\n";
    }
    void visit(const Rectangle& r) override {
        svg_ += "<rect width=\"" + std::to_string(r.width())
             + "\" height=\"" + std::to_string(r.height()) + "\"/>\n";
    }
    void visit(const Triangle&) override {
        svg_ += "<polygon points=\"...\"/>\n";
    }
    [[nodiscard]] const std::string& svg() const { return svg_; }
};
```

### 现代替代：`std::variant` + `std::visit`

```cpp
#include <variant>
#include <iostream>

// 不需要继承和 accept() 方法
struct Circle   { double radius; };
struct Rectangle { double width; double height; };
struct Triangle { double a, b, c; };

using Shape = std::variant<Circle, Rectangle, Triangle>;

// Visitor 是 lambda 的组合
struct AreaVisitor {
    double operator()(const Circle& c) const {
        return 3.14159265 * c.radius * c.radius;
    }
    double operator()(const Rectangle& r) const {
        return r.width * r.height;
    }
    double operator()(const Triangle& t) const {
        double s = (t.a + t.b + t.c) / 2;
        return std::sqrt(s * (s - t.a) * (s - t.b) * (s - t.c));
    }
};

// 或使用泛型 lambda（C++17）
auto printShape = [](const auto& s) {
    using T = std::decay_t<decltype(s)>;
    if constexpr (std::is_same_v<T, Circle>)
        std::cout << "Circle(r=" << s.radius << ")\n";
    else if constexpr (std::is_same_v<T, Rectangle>)
        std::cout << "Rectangle(" << s.width << "x" << s.height << ")\n";
    else if constexpr (std::is_same_v<T, Triangle>)
        std::cout << "Triangle(" << s.a << "," << s.b << "," << s.c << ")\n";
};

// 使用
int main() {
    std::vector<Shape> shapes = {
        Circle{5.0},
        Rectangle{3.0, 4.0},
        Triangle{3.0, 4.0, 5.0}
    };

    for (const auto& s : shapes) {
        std::cout << "Area: " << std::visit(AreaVisitor{}, s) << "\n";
        std::visit(printShape, s);
    }
}
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 新增操作不修改元素类（符合开闭原则） | 新增元素类型需修改所有 Visitor（违反开闭原则） |
| 操作集中在一个类中 | 破坏元素的封装（visitor 需访问元素内部） |
| 双重分发实现类型安全 | 经典实现需要侵入式修改元素类（添加 accept） |
| `std::variant` 替代避免侵入 | variant 元素数量固定，扩展类型不灵活 |

## 关键要点

> **核心思想**：当你要对一组不同类型的对象执行多种操作，且不想在每个类里添加这些操作时，Visitor 将操作集中到独立的类中。

> **经典 vs 现代**：
> - 经典 Visitor：需要 `accept()` 方法 + 双重分发，侵入式，适合频繁新增操作
> - `std::variant` + `std::visit`：非侵入式，适合类型集合固定、操作频繁变化的场景

> **双重分发（Double Dispatch）**：`obj.accept(visitor)` 实现两次虚函数调用——第一次根据 obj 的类型选择 accept，第二次根据 visitor 的类型选择 visit。

> **新增操作 vs 新增类型**：
> - 操作频繁变化 → Visitor 模式（新增 Visitor 不改元素）
> - 类型频繁变化 → 不适合 Visitor（每个 Visitor 都要改）

> **实际选择指南**：
> - 编译器 AST 处理 → 经典 Visitor（类型相对稳定，操作种类多）
> - 消息/协议类型处理 → `std::variant`（类型固定，处理方式多）
> - 简单场景 → `std::visit` + overloaded lambda
