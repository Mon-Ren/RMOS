# Strategy 策略模式

定义一系列算法，将每一个封装起来，并使它们可以互换。策略模式让算法独立于使用它的客户端变化。

## 意图与场景

- **核心意图**：将算法的定义与使用分离，运行时可切换算法
- **典型场景**：
  - 排序策略（快速排序 vs 归并排序 vs 插入排序）
  - 压缩算法选择（ZIP vs GZIP vs LZ4）
  - 支付方式（支付宝 vs 微信 vs 银行卡）
  - 路线规划（最短路径 vs 最快路径 vs 避开收费）
  - 日志输出策略（控制台 vs 文件 vs 远程）

## C++ 实现

### 经典实现：Strategy 接口 + Context 组合

```cpp
#include <memory>
#include <vector>
#include <iostream>
#include <algorithm>
#include <string>

// 策略接口
class SortStrategy {
public:
    virtual ~SortStrategy() = default;
    virtual void sort(std::vector<int>& data) = 0;
    virtual std::string name() const = 0;
};

// 具体策略：快速排序
class QuickSort : public SortStrategy {
public:
    void sort(std::vector<int>& data) override {
        std::sort(data.begin(), data.end()); // 用 std::sort 实现
    }
    std::string name() const override { return "QuickSort"; }
};

// 具体策略：冒泡排序（小数据集友好）
class BubbleSort : public SortStrategy {
public:
    void sort(std::vector<int>& data) override {
        for (size_t i = 0; i < data.size(); ++i)
            for (size_t j = 0; j + 1 < data.size() - i; ++j)
                if (data[j] > data[j + 1])
                    std::swap(data[j], data[j + 1]);
    }
    std::string name() const override { return "BubbleSort"; }
};

// Context：排序器
class Sorter {
    std::unique_ptr<SortStrategy> strategy_;
public:
    explicit Sorter(std::unique_ptr<SortStrategy> s)
        : strategy_(std::move(s)) {}

    void setStrategy(std::unique_ptr<SortStrategy> s) {
        strategy_ = std::move(s);
    }

    void sort(std::vector<int>& data) const {
        std::cout << "使用策略: " << strategy_->name() << "\n";
        strategy_->sort(data);
    }
};
```

### 现代实现：`std::function` 替代接口

```cpp
#include <functional>
#include <iostream>

// 方式 1：std::function 作为策略类型
using CompressFn = std::function<std::vector<uint8_t>(const std::vector<uint8_t>&)>;

class Compressor {
    CompressFn strategy_;
public:
    explicit Compressor(CompressFn fn) : strategy_(std::move(fn)) {}

    void setStrategy(CompressFn fn) { strategy_ = std::move(fn); }

    std::vector<uint8_t> compress(const std::vector<uint8_t>& data) const {
        return strategy_(data);
    }
};

// 方式 2：模板策略（编译时多态，零开销）
template<typename SortPolicy>
class TypedSorter {
    SortPolicy policy_;
public:
    void sort(std::vector<int>& data) { policy_.sort(data); }
};

struct StdSortPolicy {
    void sort(std::vector<int>& d) { std::sort(d.begin(), d.end()); }
};

struct ReverseSortPolicy {
    void sort(std::vector<int>& d) {
        std::sort(d.begin(), d.end(), std::greater<>{});
    }
};

// 使用: TypedSorter<StdSortPolicy> sorter;
```

### 函数对象策略

```cpp
// 策略作为函数对象，天然可组合
struct LoggingPolicy {
    template<typename F>
    auto operator()(F&& fn) const {
        std::cout << "开始执行...\n";
        auto result = std::forward<F>(fn)();
        std::cout << "执行完毕\n";
        return result;
    }
};

struct TimingPolicy {
    template<typename F>
    auto operator()(F&& fn) const {
        auto start = std::chrono::steady_clock::now();
        auto result = std::forward<F>(fn)();
        auto end = std::chrono::steady_clock::now();
        std::cout << "耗时: "
                  << std::chrono::duration<double, std::milli>(end - start).count()
                  << "ms\n";
        return result;
    }
};
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 符合开闭原则：新增策略不动已有代码 | 客户端需要知道有哪些策略 |
| 消除条件分支 | 策略类数量可能膨胀 |
| 运行时切换算法 | 简单场景可能过度设计 |
| 算法可独立测试 | 策略与 Context 之间可能有数据传递开销 |

## 关键要点

> **核心思想**：与其用 `if-else` 选择算法，不如将每个算法封装为独立对象，让 Context 通过组合持有它。

> **C++ 最佳实践**：
> - 简单场景 → `std::function`（最轻量）
> - 性能敏感 → 模板策略（编译时多态，零虚函数开销）
> - 复杂场景 → 经典接口 + 多态（运行时灵活性）

> **策略 vs 状态**：两者结构几乎相同，但——
> - [[设计模式-状态模式|State]]：状态自己决定转换到哪个状态
> - **Strategy**：客户端选择使用哪个策略

> **经典应用**：STL 的 `std::sort` 第三个参数就是策略模式——比较器可以是函数指针、函数对象或 lambda。

## 相关模式

- [[设计模式-状态模式|State 模式]]：结构相似，State 的状态之间可以互相转换，Strategy 由外部切换
- [[设计模式-模板方法模式|Template Method 模式]]：Template Method 通过继承改变算法步骤，Strategy 通过组合改变整个算法
- [[设计模式-桥接模式|Bridge 模式]]：Bridge 的实现层本质上也是一种策略
