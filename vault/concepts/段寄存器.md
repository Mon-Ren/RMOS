段寄存器是 x86 内存分段机制的核心，存放段选择子（segment selector），用于在保护模式下定位内存段的基址和属性。

## 6 个段寄存器

| 寄存器 | 名称 | 默认使用场景 |
|--------|------|-------------|
| CS     | 代码段 (Code)     | 指令获取，EIP 相对于 CS |
| DS     | 数据段 (Data)     | 数据访问的默认段 |
| SS     | 栈段 (Stack)      | 栈操作（PUSH/POP/ESP）|
| ES     | 附加段 (Extra)    | 字符串目标（EDI）|
| FS     | 附加段            | 线程局部存储（Linux/glibc）|
| GS     | 附加段            | 内核 per-CPU 数据（Linux x86-64）|

## 实模式下的段基址

在实模式中，段寄存器存放的是**段基址的高 16 位**：

```
物理地址 = 段寄存器 × 16 + 偏移量
```

```asm
; 实模式：计算物理地址
mov ax, 0x1000       ; 段基址 = 0x1000
mov ds, ax
mov bx, 0x0050       ; 偏移 = 0x0050
; 物理地址 = 0x1000 * 16 + 0x0050 = 0x10050
mov al, [ds:bx]      ; 读取地址 0x10050 处的字节
```

> [!info] 实模式寻址
> - 段值左移 4 位（乘 16）加上偏移，得到 20 位物理地址
> - 可寻址范围：段值×16 到 段值×16 + 0xFFFF
> - 这导致段之间有重叠（每 16 字节就有一个新段起始点）

## 保护模式下的段选择子

在保护模式中，段寄存器不再直接存放基址，而是存放一个 16 位的**段选择子**：

```
 15                3  2   0
┌──────────────────┬──┬───┐
│     Index (13)   │TI│RPL│
└──────────────────┴──┴───┘
```

- **Index (13 位)**：GDT/LDT 中的索引号
- **TI (1 位)**：0 = GDT，1 = LDT
- **RPL (2 位)**：请求特权级（0-3）

```asm
; 保护模式下加载段选择子
mov ax, 0x08         ; Index=1, TI=0(GDT), RPL=0
mov ds, ax           ; DS 指向 GDT 第 1 个描述符

; CS 不能用 MOV 加载，只能用远跳转
jmp 0x08:next_label  ; 设置 CS = 0x08
next_label:
```

CPU 通过选择子查 GDT/LDT 得到段描述符，其中包含：
- 段基址（32 位）
- 段限长（20 位）
- 访问权限和类型

## Linux 的平坦模型 (Flat Model)

现代 Linux 和大多数 OS 使用**平坦模型**，让分段机制形同虚设：

```asm
; 所有段寄存器基址=0, 限长=4GB
; 虚拟地址 = 偏移量（段基址为 0）
; 实际内存保护由分页机制（Paging）负责

; Linux x86-64 中 FS/GS 仍有实际用途：
mov rax, fs:[0x28]   ; 读取 TLS 中的 canary 值（栈保护）
mov rax, gs:[0x0]    ; 读取当前 CPU 的 per-CPU 数据
```

> [!warning] 重要区分
> - 32 位 Linux：CS/DS/SS/ES 全部指向同一 4GB 段（基址=0）
> - 64 位 Linux：分段基本废弃，但 FS/GS 用于线程本地存储和内核 per-CPU 数据
> - Windows：GS 用于 TEB（Thread Environment Block）

## 段前缀覆盖

可以在指令前加段前缀来临时使用非默认段：

```asm
mov eax, [ebx]           ; 默认使用 DS
mov eax, es:[ebx]        ; 强制使用 ES
mov eax, fs:[ecx]        ; 强制使用 FS（Linux TLS）
mov eax, gs:[edx*8]      ; 强制使用 GS
```

> [!tip] 关键要点
> - 实模式：段寄存器直接存放基址高 16 位，物理地址 = 段值×16 + 偏移
> - 保护模式：段寄存器存放选择子，通过 GDT/LDT 查找段描述符
> - 现代 OS 使用平坦模型，分段名存实亡，分页才是真正的内存保护机制
> - FS/GS 在 x86-64 中仍活跃，用于 TLS 和内核数据
