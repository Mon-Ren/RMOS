# 页表机制

xv6 使用 x86 二级页表实现虚拟内存，每个进程拥有独立的地址空间。

## x86 二级页表结构

```
虚拟地址 (32位)
┌──────────┬──────────┬────────────┐
│ PDX (10b)│ PTX (10b)│ Offset(12b)│
└──────────┴──────────┴────────────┘
     ↓           ↓
  页目录项    页表项
  (PDE)       (PTE)
     ↓           ↓
  页表物理地址  物理页帧地址
```

- **页目录**：1024 个 PDE，每个指向一个页表
- **页表**：1024 个 PTE，每个指向一个 4KB 物理页
- 总映射空间：1024 × 1024 × 4KB = 4GB

## 关键函数

### walkpgdir() — 查找 PTE

```c
static pte_t *
walkpgdir(pde_t *pgdir, const void *va, int alloc)
```

给定虚拟地址 `va`，沿两级页表找到对应的 PTE。
- `alloc=1`：若页表不存在则自动分配
- `alloc=0`：仅查询，不存在返回 0

### mappages() — 建立映射

```c
static int
mappages(pde_t *pgdir, void *va, uint size, uint pa, int perm)
```

将虚拟地址 `[va, va+size)` 映射到物理地址 `[pa, pa+size)`，设置权限 `perm`。

### copyuvm() — 深拷贝用户页表

fork 时使用：创建新的页目录 + 内核映射（`setupkvm`）+ 逐页拷贝用户空间数据。

## 内核地址空间布局

```
虚拟地址
0xFFFFFFFF ┌──────────────────┐
           │ 内核映射区        │ ← setupkvm() 映射所有物理内存
0x80000000 │ (KERNBASE)       │
           ├──────────────────┤
           │ 未映射（空洞）    │
0x00000000 ├──────────────────┤
           │ 用户空间          │ ← 每个进程独立
           └──────────────────┘
```

内核通过 `KERNBASE + 物理地址` 访问所有物理内存。用户空间从 0 开始向上增长。

## entrypgdir — 启动时的临时页表

```c
pde_t entrypgdir[NPDENTRIES] = {
  [0] = (0) | PTE_P | PTE_W | PTE_PS,
  [KERNBASE>>PDXSHIFT] = (0) | PTE_P | PTE_W | PTE_PS,
};
```

双重映射技巧：
- VA `[0, 4MB)` → PA `[0, 4MB)`（启动时 PC 还在低地址）
- VA `[KERNBASE, KERNBASE+4MB)` → PA `[0, 4MB)`（内核期望的高地址）

开启分页后，PC 的值不变，但解释方式变成了高地址映射。

## PTE 权限位

| 位 | 含义 |
|----|------|
| PTE_P | 页面存在 |
| PTE_W | 可写 |
| PTE_U | 用户态可访问 |
| PTE_PS | 4MB 大页（仅 PDE） |

## 关键要点

> 页表的核心价值是**隔离**：每个进程看到独立的地址空间，内核地址对用户不可见。`walkpgdir` + `mappages` 是所有映射操作的基础，`copyuvm` 实现了 fork 的写时隔离（注意 xv6 没有 COW，是完整拷贝）。
