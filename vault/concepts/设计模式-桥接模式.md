# Bridge 桥接模式

将抽象（Abstraction）与实现（Implementation）分离到两个独立的类层次中，使两者可以独立变化，避免类爆炸（class explosion）。

## 意图与场景

当一个类存在两个独立变化的维度时，继承会导致类数量爆炸。Bridge 用组合替代继承，将两个维度解耦。

**典型场景：**
- 图形系统：Shape（圆/矩形）× Renderer（OpenGL/DirectX/Vulkan）
- 跨平台 UI：Widget × OS 实现
- Pimpl 惯用法：编译防火墙

**类爆炸问题：**
```
不使用 Bridge：OpenGLCircle, DXCircle, VulkanCircle,
              OpenGLRect, DXRect, VulkanRect → 6 类

使用 Bridge：Circle, Rect + OpenGLRenderer, DXRenderer, VulkanRenderer → 5 类（且可继续扩展）
```

## C++ 实现

### 图形渲染示例

```cpp
#include <iostream>
#include <memory>
#include <numbers>

// --- Implementation 层 ---
class IRenderer {
public:
    virtual ~IRenderer() = default;
    virtual void renderCircle(double x, double y, double r) = 0;
    virtual void renderRect(double x, double y, double w, double h) = 0;
};

class OpenGLRenderer : public IRenderer {
public:
    void renderCircle(double x, double y, double r) override {
        std::cout << "[OpenGL] Circle at (" << x << "," << y
                  << ") r=" << r << "\n";
    }
    void renderRect(double x, double y, double w, double h) override {
        std::cout << "[OpenGL] Rect at (" << x << "," << y
                  << ") " << w << "x" << h << "\n";
    }
};

class VulkanRenderer : public IRenderer {
public:
    void renderCircle(double x, double y, double r) override {
        std::cout << "[Vulkan] Circle at (" << x << "," << y
                  << ") r=" << r << "\n";
    }
    void renderRect(double x, double y, double w, double h) override {
        std::cout << "[Vulkan] Rect at (" << x << "," << y
                  << ") " << w << "x" << h << "\n";
    }
};

// --- Abstraction 层 ---
class Shape {
protected:
    std::shared_ptr<IRenderer> renderer_;  // 桥接指针
public:
    explicit Shape(std::shared_ptr<IRenderer> renderer)
        : renderer_(std::move(renderer)) {}
    virtual ~Shape() = default;
    virtual void draw() = 0;
};

class Circle : public Shape {
    double x_, y_, radius_;
public:
    Circle(std::shared_ptr<IRenderer> r, double x, double y, double radius)
        : Shape(std::move(r)), x_(x), y_(y), radius_(radius) {}

    void draw() override {
        renderer_->renderCircle(x_, y_, radius_);
    }

    double area() const {
        return std::numbers::pi * radius_ * radius_;
    }
};

class Rectangle : public Shape {
    double x_, y_, w_, h_;
public:
    Rectangle(std::shared_ptr<IRenderer> r, double x, double y, double w, double h)
        : Shape(std::move(r)), x_(x), y_(y), w_(w), h_(h) {}

    void draw() override {
        renderer_->renderRect(x_, y_, w_, h_);
    }
};

// --- 使用 ---
int main() {
    auto gl = std::make_shared<OpenGLRenderer>();
    auto vk = std::make_shared<VulkanRenderer>();

    Circle c(gl, 5, 5, 3);
    Rectangle r(vk, 0, 0, 10, 20);

    c.draw();   // [OpenGL] Circle ...
    r.draw();   // [Vulkan] Rect ...
}
```

### Pimpl 惯用法（编译防火墙）

Pimpl（Pointer to Implementation）是 Bridge 模式在 C++ 中最实用的应用：

```cpp
// --- widget.h（头文件，稳定不变）---
#pragma once
#include <memory>

class Widget {
    struct Impl;              // 前置声明
    std::unique_ptr<Impl> pImpl_;
public:
    Widget();
    ~Widget();                // 必须在 .cpp 中定义
    Widget(Widget&&) noexcept;
    Widget& operator=(Widget&&) noexcept;

    void doWork(int value);
    int getValue() const;
};

// --- widget.cpp（实现文件，可自由修改）---
#include "widget.h"
#include <string>
#include <vector>

struct Widget::Impl {
    std::string name;
    std::vector<int> data;
    int cachedValue{0};
    
    // 内部实现细节随意变化，不影响头文件
    void recalculate() { /* ... */ }
};

Widget::Widget() : pImpl_(std::make_unique<Impl>()) {}
Widget::~Widget() = default;  // unique_ptr 需要完整类型
Widget::Widget(Widget&&) noexcept = default;
Widget& Widget::operator=(Widget&&) noexcept = default;

void Widget::doWork(int value) {
    pImpl_->data.push_back(value);
    pImpl_->recalculate();
}

int Widget::getValue() const {
    return pImpl_->cachedValue;
}
```

> **Pimpl 优势：** 修改 `Impl` 的内容不需要重新编译所有包含 `widget.h` 的翻译单元，大幅减少编译时间。

## 优缺点

**优点：**
- 抽象和实现可独立变化，符合开闭原则
- 避免两个维度交叉导致的类爆炸
- Pimpl 减少编译依赖、加快编译速度
- 运行时可切换实现

**缺点：**
- 增加了代码复杂度和间接层
- 需要额外的指针解引用开销
- Pimpl 需要手动管理特殊成员函数（或用 `unique_ptr`）

## 关键要点

> - Bridge 的核心是「组合替代继承」——用指针连接两个独立变化的层次
> - Pimpl 是 Bridge 模式在 C++ 中最实用的变体，提供编译防火墙
> - `std::unique_ptr<Impl>` 是 Pimpl 的现代 C++ 实现方式
> - 与 [[设计模式-适配器模式|Adapter]] 的区别：Bridge 在设计阶段分离抽象与实现；Adapter 在事后转换已有接口
> - 当类有两个独立变化的维度时，优先考虑 Bridge
