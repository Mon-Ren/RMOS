# 栈帧结构

每次函数调用在栈上创建一个**栈帧**（stack frame），形成链式结构。

## 标准栈帧

```
高地址
┌───────────────────┐  ← 调用者的栈帧
│   参数 N          │
│   ...             │
│   参数 1          │
├───────────────────┤
│   返回地址 (EIP)   │  ← call 指令自动压入
├───────────────────┤
│   旧 EBP          │  ← push ebp; mov ebp, esp
├──────────────[EBP]┤  ← 当前帧基址
│   局部变量 1       │
│   局部变量 2       │
│   ...             │
├──────────────[ESP]┤  ← 栈顶
│   （继续向下生长）  │
低地址
```

## 标准序言（Prologue）

```asm
push ebp          ; 保存调用者的帧指针
mov  ebp, esp     ; 建立新的帧指针
sub  esp, N       ; 为局部变量分配空间
push ebx          ; 保存 callee-saved 寄存器
push esi
push edi
```

## 标准结尾（Epilogue）

```asm
pop  edi          ; 恢复 callee-saved 寄存器
pop  esi
pop  ebx
mov  esp, ebp     ; 释放局部变量
pop  ebp          ; 恢复调用者的帧指针
ret               ; 弹出返回地址并跳转
```

## 参数和局部变量访问

```asm
[ebp+12]  ; 第 2 个参数
[ebp+8]   ; 第 1 个参数
[ebp+4]   ; 返回地址
[ebp]     ; 旧 ebp
[ebp-4]   ; 第 1 个局部变量
[ebp-8]   ; 第 2 个局部变量
```

## EBP 链回溯调用栈

```
EBP → [旧EBP] → [旧EBP] → [旧EBP] → ...
        ↑          ↑          ↑
     函数C       函数B       函数A
```

每帧的 `[ebp+4]` 是返回地址，沿 EBP 链可以重建整个调用栈。这就是：
- GDB 的 `bt`（backtrace）命令原理
- xv6 的 `getcallerpcs()` 函数实现

## xv6 中的应用

```c
void getcallerpcs(void *v, uint pcs[]) {
  uint *ebp = (uint*)v - 2;  // v 指向 context 的 ebp 字段
  for(int i = 0; i < 10; i++) {
    pcs[i] = ebp[1];         // 返回地址
    ebp = (uint*)ebp[0];     // 上一帧的 ebp
  }
}
```

## 关键要点

> 栈帧是函数调用的物理实现——每个函数在栈上有一块连续区域存放参数、返回地址、局部变量和保存的寄存器。EBP 形成链表，沿链回溯就能重建调用栈。调试器的 backtrace、异常处理的堆栈展开、以及 xv6 的调用栈记录都依赖这个结构。
