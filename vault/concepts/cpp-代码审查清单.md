# C++ 代码审查清单

代码审查是质量保证的关键环节——这份清单覆盖 C++ 代码审查的核心检查点。

## 内存安全

- [ ] 没有裸 `new`/`delete`（用 `make_unique`/`make_shared`）
- [ ] 没有悬垂指针/引用（返回局部变量的指针、迭代器失效）
- [ ] 虚基类有虚析构函数
- [ ] 容器中指针的内存被正确释放
- [ ] 没有数组 `new[]` 和标量 `delete` 的不匹配

## 资源管理

- [ ] 所有资源用 RAII 管理（文件、锁、socket）
- [ ] 遵循 Rule of Zero 或 Rule of Five
- [ ] 移动操作标记 `noexcept`
- [ ] 没有在持有锁时做 I/O 或获取其他锁

## 类型安全

- [ ] 没有 C 风格强制转换（用 `static_cast` 等）
- [ ] `enum class` 代替旧式 `enum`
- [ ] `explicit` 用于单参数构造函数
- [ ] 没有有符号/无符号混合运算

## 异常安全

- [ ] 函数有明确的异常安全级别
- [ ] 析构函数不抛异常
- [ ] 先做可能失败的操作，再做不可逆操作
- [ ] 没有在异常路径上泄漏资源

## 并发安全

- [ ] 共享数据有同步保护
- [ ] 没有数据竞争
- [ ] 锁的获取顺序一致（避免死锁）
- [ ] `jthread` 或 RAII 管理线程生命周期

## 性能

- [ ] 函数参数避免不必要的拷贝（`const&`）
- [ ] 大容器预分配（`reserve`）
- [ ] 循环中没有重复的容器查询（缓存 `size()`）
- [ ] 算法复杂度合理

## 代码质量

- [ ] 变量初始化于声明处
- [ ] `const` 尽可能使用
- [ ] 命名清晰、一致
- [ ] 没有魔法数字（用命名常量）

## 关键要点

> 代码审查的核心：安全性 > 正确性 > 可读性 > 性能。先确保没有安全漏洞，再看逻辑正确。

> 自动化工具（clang-tidy、cppcheck、ASan）能检查大部分问题——审查应聚焦设计和逻辑。
