# Observer 观察者模式

定义对象间一对多的依赖关系，当一个对象状态改变时，所有依赖于它的对象都会得到通知并自动更新。

## 意图与场景

- **核心意图**：建立发布-订阅机制，实现松耦合的事件通知
- **典型场景**：
  - GUI 事件系统（按钮点击通知处理器）
  - 消息队列 / 事件总线
  - MVC 架构（Model 变化通知 View）
  - 股票价格变动通知投资者
  - 日志系统（多输出：控制台、文件、远程）
  - Qt Signal/Slot 机制

## C++ 实现

### 经典实现：Subject + Observer 接口

```cpp
#include <vector>
#include <algorithm>
#include <memory>
#include <string>
#include <iostream>
#include <functional>

// 观察者接口
class Observer {
public:
    virtual ~Observer() = default;
    virtual void onNotify(const std::string& event, const void* data) = 0;
};

// 被观察者
class Subject {
    std::vector<Observer*> observers_; // 非拥有，原始指针
public:
    void attach(Observer* obs) {
        observers_.push_back(obs);
    }
    void detach(Observer* obs) {
        std::erase(observers_, obs);
    }
    void notify(const std::string& event, const void* data = nullptr) {
        for (auto* obs : observers_) {
            obs->onNotify(event, data);
        }
    }
};

// 具体被观察者：天气站
class WeatherStation : public Subject {
    double temperature_ = 0;
public:
    void setTemperature(double temp) {
        temperature_ = temp;
        notify("temperature_changed", &temperature_);
    }
    [[nodiscard]] double temperature() const { return temperature_; }
};

// 具体观察者：显示屏
class Display : public Observer {
    std::string name_;
public:
    explicit Display(std::string name) : name_(std::move(name)) {}
    void onNotify(const std::string& event, const void* data) override {
        if (event == "temperature_changed" && data) {
            auto temp = *static_cast<const double*>(data);
            std::cout << "[" << name_ << "] 温度更新: " << temp << "°C\n";
        }
    }
};
```

### 现代实现：`std::function` + 类型安全事件

```cpp
#include <functional>
#include <unordered_map>
#include <vector>
#include <any>
#include <typeindex>

// 类型安全的事件总线
class EventBus {
    std::unordered_map<std::type_index, std::vector<std::function<void(const std::any&)>>> handlers_;
public:
    template<typename Event>
    void subscribe(std::function<void(const Event&)> handler) {
        auto key = std::type_index(typeid(Event));
        handlers_[key].push_back([h = std::move(handler)](const std::any& data) {
            h(std::any_cast<const Event&>(data));
        });
    }

    template<typename Event>
    void publish(const Event& event) {
        auto key = std::type_index(typeid(Event));
        if (auto it = handlers_.find(key); it != handlers_.end()) {
            for (const auto& h : it->second) {
                h(event);
            }
        }
    }
};

// 事件定义
struct TemperatureChanged { double value; };
struct HumidityChanged { double value; };

// 使用
// EventBus bus;
// bus.subscribe<TemperatureChanged>([](const auto& e) { ... });
// bus.publish(TemperatureChanged{25.5});
```

### 回调列表模式（轻量方案）

```cpp
template<typename... Args>
class Signal {
    using Callback = std::function<void(Args...)>;
    std::vector<Callback> callbacks_;
    int nextId_ = 0;
    std::vector<std::pair<int, Callback>> indexed_;
public:
    // 连接，返回 ID 用于断开
    int connect(Callback cb) {
        int id = nextId_++;
        indexed_.emplace_back(id, std::move(cb));
        return id;
    }

    void disconnect(int id) {
        std::erase_if(indexed_, [id](const auto& p) { return p.first == id; });
    }

    void emit(Args... args) const {
        for (const auto& [id, cb] : indexed_) {
            cb(args...);
        }
    }

    // 操作符重载，语法糖
    void operator()(Args... args) const { emit(std::forward<Args>(args)...); }
};

// 使用
// Signal<int, std::string> sig;
// auto id = sig.connect([](int x, std::string s) { ... });
// sig(42, "hello");
// sig.disconnect(id);
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 松耦合：Subject 不需要知道 Observer 的具体类型 | 通知顺序不可控 |
| 支持广播通信 | 可能导致意外的级联更新 |
| 符合开闭原则 | 调试困难（隐式依赖） |
| 运行时动态订阅/退订 | Observer 的生命周期管理复杂（悬空指针风险） |

## 关键要点

> **核心思想**：将"谁关心变化"这件事从被观察者身上剥离。Subject 只管发通知，不关心谁在听。

> **Qt Signal/Slot**：Qt 的信号槽是观察者模式的经典实现。`connect()` 建立关系，`emit` 触发通知，自动处理线程安全和类型匹配。

> **生命周期陷阱**：C++ 中最常见的 Observer 问题——Observer 被销毁但 Subject 仍持有其指针。解决方案：
> - `shared_ptr<Observer>` + `weak_ptr` 检测
> - RAII 连接管理器（类似 Qt 的 `QMetaObject::Connection`）
> - `detach()` 在 Observer 析构时自动调用

> **同步 vs 异步**：经典 Observer 是同步通知（调用栈中直接执行）。生产系统通常需要异步队列来避免级联阻塞和死锁。
