内联汇编允许在 C/C++ 代码中直接嵌入汇编指令，是操作系统内核、性能关键代码与硬件交互的重要手段。

## GCC 基本内联汇编

最简单的形式——直接嵌入汇编字符串：

```c
// 基本内联汇编
__asm__("nop");                        // 单条指令
__asm__("movl $42, %eax\n\t"
        "addl $8, %eax");             // 多条指令，用 \n\t 分隔
```

## 扩展内联汇编（核心）

扩展形式允许指定输入/输出/破坏列表，让编译器正确优化：

```c
// 扩展内联汇编格式
__asm__ __volatile__ (
    "汇编模板"
    : 输出操作数           // 可选
    : 输入操作数           // 可选
    : 破坏列表             // 可选
);
```

### 实际示例

```c
// 原子交换：将 new_val 写入 *ptr，返回旧值
int atomic_xchg(int *ptr, int new_val) {
    __asm__ __volatile__ (
        "xchgl %0, %1"
        : "+r" (new_val),   // %0: 输入输出
          "+m" (*ptr)        // %1: 输入输出（内存）
        :
        : "memory"
    );
    return new_val;
}

// 系统调用封装
static inline int syscall3(int num, int a1, int a2, int a3) {
    int ret;
    __asm__ __volatile__ (
        "int $0x80"
        : "=a" (ret)          // 输出：EAX → ret
        : "a" (num),          // 输入：num → EAX
          "b" (a1),           //       a1  → EBX
          "c" (a2),           //       a2  → ECX
          "d" (a3)            //       a3  → EDX
        : "memory"
    );
    return ret;
}
```

## 约束符号详解

### 常用约束符

| 约束 | 含义 | 说明 |
|------|------|------|
| `r` | 寄存器 | 编译器选择任意通用寄存器 |
| `a` | EAX/RAX | 特定寄存器 |
| `b` | EBX/RBX | 特定寄存器 |
| `c` | ECX/RCX | 特定寄存器 |
| `d` | EDX/RDX | 特定寄存器 |
| `S` | ESI/RSI | 特定寄存器 |
| `D` | EDI/RDI | 特定寄存器 |
| `m` | 内存操作数 | 直接内存访问 |
| `i` | 立即数（编译时常量） | 如 42, offset |
| `n` | 立即数（已知数值） | 运行时也可 |
| `g` | 通用 | 寄存器/内存/立即数均可 |

### 修饰符

```c
// 修饰符在约束前
"=r" (out)    // = 表示只写（输出）
"+r" (var)    // + 表示读写（输入输出）
"r"  (in)     // 无修饰符 = 只读（输入）
"&r" (out)    // & 表示 earlyclobber（输出在输入使用前被写）
```

### 数字约束（复用操作数）

```c
// 用数字引用前面的操作数
int add(int a, int b) {
    __asm__ (
        "addl %1, %0"     // %0 = 输出, %1 = 第二个输入
        : "=r" (a)        // 输出复用 a 的位置
        : "r" (b)         // 输入
    );
    return a;
}
```

## volatile 关键字

```c
// volatile: 防止编译器优化掉或移动这段汇编
__asm__ __volatile__ (
    "cpuid"
    : "=a" (eax_out), "=b" (ebx_out), "=c" (ecx_out), "=d" (edx_out)
    : "a" (func), "c" (subfunc)
);

// 什么时候不用 volatile？
// 如果纯计算、无副作用，可以让编译器优化
__asm__ (
    "leal (%1,%1,4), %0"    // x*5
    : "=r" (result)
    : "r" (x)
);
```

> **规则**：有副作用（IO、内存访问、CPUID 等）必须加 `volatile`；纯算术可以不加，让编译器优化。

## 内存约束和 clobber

```c
// "memory" clobber — 告诉编译器：这段汇编可能修改任意内存
__asm__ __volatile__ (
    "mfence"
    ::: "memory"
);

// 多个 clobber 列表
__asm__ __volatile__ (
    "rdtsc"                   // 读取时间戳计数器
    : "=a" (low), "=d" (high) // EAX = low, EDX = high
    :
    : "memory"                // 可能影响内存
);
```

## 为什么 xv6 大量使用内联汇编

xv6（MIT 教学操作系统）使用内联汇编实现：

```c
// xv6 — 开启分页
static inline void lcr3(uint val) {
    __asm__ __volatile__ ("movl %0, %%cr3" : : "r" (val));
}

// xv6 — 读取 CPU ID
static inline int cpuid(void) {
    __asm__ __volatile__ (
        "movl %%ebx, %0"
        : "=r" (id)
        :
        : "memory"
    );
    return id;
}

// xv6 — 关中断
static inline void cli(void) {
    __asm__ __volatile__ ("cli" ::: "memory");
}
```

原因：
1. **C 无法表达**：CR3、CLI/STI、系统调用等没有对应的 C 操作
2. **精确控制**：指定使用哪个寄存器，保证与硬件接口一致
3. **原子性**：XCHG、CMPXCHG 等需要汇编级保证
4. **简短**：只有一两行，不值得单独写 .s 文件

## 完整示例：RDTSC 读取时间戳

```c
#include <stdint.h>

static inline uint64_t rdtsc(void) {
    uint32_t lo, hi;
    __asm__ __volatile__ (
        "rdtsc"
        : "=a" (lo), "=d" (hi)   // EAX → lo, EDX → hi
        :
        : "memory"
    );
    return ((uint64_t)hi << 32) | lo;
}

// 带序列化的版本（防止乱序）
static inline uint64_t rdtscp(void) {
    uint32_t lo, hi, aux;
    __asm__ __volatile__ (
        "rdtscp"
        : "=a" (lo), "=d" (hi), "=c" (aux)
        :
        : "memory"
    );
    return ((uint64_t)hi << 32) | lo;
}

// 使用示例
int main(void) {
    uint64_t start = rdtsc();
    // ... 一些操作 ...
    uint64_t end = rdtsc();
    printf("Cycles: %lu\n", end - start);
    return 0;
}
```

## 关键要点

> **基本格式**：`__asm__("指令")` — 简单但无法与编译器交互。扩展格式加 `:输出:输入:破坏` 才能正确优化。

> **约束符号**：`r`=寄存器, `m`=内存, `i`=立即数, `a/b/c/d`=指定 EAX/EBX/ECX/EDX。`=`=输出, `+`=输入输出。

> **volatile 必须加**：除非你的汇编是纯计算。有副作用（IO/内存/CPUID）不加 volatile 编译器可能把它优化掉。

> **xv6 模式**：内核中大量 `static inline` 函数封装单条汇编指令，形成硬件抽象层——这是系统编程的标准模式。
