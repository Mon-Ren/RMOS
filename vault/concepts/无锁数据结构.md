# 无锁数据结构

不使用锁实现并发安全的数据结构。核心依赖 CAS（Compare-And-Swap）原子操作。

## CAS（比较并交换）

```c
bool CAS(int *addr, int expected, int newval) {
  if(*addr == expected) {
    *addr = newval;
    return true;
  }
  return false;
}
// x86: cmpxchgl 指令，原子执行
```

## 无锁队列（Michael-Scott Queue）

```
Head → [Node1] → [Node2] → [Node3] → NULL
                                  ← Tail
```

### 入队（enqueue）

```c
void enqueue(Queue *q, int val) {
  Node *node = new Node(val);
  while(true) {
    Node *tail = q->tail;          // 读取尾节点
    Node *next = tail->next;       // 读取尾节点的 next
    if(tail == q->tail) {          // 检查 tail 未被修改
      if(next == NULL) {
        // tail 确实是最后一个节点
        if(CAS(&tail->next, NULL, node)) {  // 链接新节点
          CAS(&q->tail, tail, node);        // 移动 tail（允许失败）
          return;
        }
      } else {
        CAS(&q->tail, tail, next);  // 帮助其他线程推进 tail
      }
    }
  }
}
```

### 关键：ABA 问题

```
线程1: CAS(addr, A, B)  // 读到 A
线程2: 将 A 改为 B
线程3: 将 B 改回 A
线程1: CAS(addr, A, B)  // 值还是 A，但中间已被修改！
```

**解决方案：**
- 版本号：`CAS(addr, (A, ver), (B, ver+1))`
- Hazard Pointer：延迟回收
- Epoch-based Reclamation

## 自旋锁 vs 无锁

| | 自旋锁 | 无锁 |
|---|--------|------|
| 死锁 | 可能 | 不可能 |
| 优先级反转 | 可能 | 不可能 |
| 复杂度 | 简单 | 高 |
| 性能 | 低争用好 | 高争用好 |
| 可维护性 | 好 | 差 |

## 关键要点

> 无锁数据结构的承诺是**系统级进步**（至少一个线程有进展），代价是极端的复杂度。ABA 问题、内存回收、代码可读性都是挑战。xv6 没有用无锁数据结构——自旋锁简单且对教学 OS 够用。生产环境中的无锁结构（如 Linux 的 `lockless` 链表）由专家编写并经过多年调试。
