# 设备驱动模型

OS 通过驱动程序屏蔽硬件差异，向上层提供统一接口。

## xv6 的设备架构

```
用户程序 → syscall → file.c → 具体驱动
                      ├── ide.c (磁盘)
                      ├── console.c (终端)
                      ├── pipe.c (管道)
                      └── uart.c (串口)
```

### file 结构体

```c
struct file {
  enum { FD_NONE, FD_PIPE, FD_INODE } type;
  int ref;          // 引用计数
  char readable;
  char writable;
  struct pipe *pipe;
  struct inode *ip;
  uint off;         // 读写偏移
};
```

复用同一套 read/write 接口，通过 type 区分底层实现。

## 三种 IO 方式

### 1. 轮询（Polling）

```c
// ide.c: 等待磁盘就绪
while((inb(0x1F7) & IDE_BSY) || !(inb(0x1F7) & IDE_DRDY))
  ;
```

CPU 一直检查状态寄存器。简单但浪费 CPU。

### 2. 中断驱动（Interrupt-driven）

```c
// 发出 IO 请求后，进程 sleep
ideintr() {
  wakeup(requesting_process);  // 中断时唤醒
}
ideread() {
  issue_disk_command();
  sleep(&disk_buffer, &idelock);  // 睡眠等中断
  // 中断唤醒后继续
}
```

IO 期间 CPU 可以做其他事。

### 3. DMA（Direct Memory Access）

硬件直接在内存和设备之间传输数据，CPU 只设置传输参数。

## Linux 设备模型

```
用户空间: /dev/sda, /dev/tty0
  │
  ▼
VFS (虚拟文件系统)
  │
  ▼
块设备层 / 字符设备层
  │
  ▼
具体驱动 (AHCI, NVMe, USB...)
  │
  ▼
硬件
```

### 设备号

```
主设备号: 驱动类型
次设备号: 具体设备实例

crw-rw-rw- 1 root tty 5, 0 /dev/tty   (字符设备, 主=5, 次=0)
brw-rw---- 1 root disk 8, 0 /dev/sda   (块设备, 主=8, 次=0)
```

## 中断 vs 轮询选择

| | 轮询 | 中断 |
|---|------|------|
| 延迟 | 低（立即响应） | 高（中断开销） |
| CPU 利用 | 差（空等） | 好（可做其他事） |
| 适用 | 高频短操作 | 低频长操作 |
| 例 | 网卡收发 (NAPI) | 磁盘、键盘 |

现代驱动混合使用：高频时轮询，低频时切换到中断。

## 关键要点

> 设备驱动的核心挑战是**抽象**——磁盘、终端、管道本质完全不同，但要统一成 read/write/open/close 接口。xv6 通过 file 结构体的 type 字段做分派，Linux 通过 VFS + 设备号。中断驱动是 IO 的主流方式——发起请求后 sleep，中断唤醒继续。
