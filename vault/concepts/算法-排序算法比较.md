# 排序算法比较

系统对比常见排序算法的时间/空间复杂度、稳定性与适用场景。

## 全面对比

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定 | 原地 |
|------|----------|----------|------|------|------|
| 冒泡排序 | O(n²) | O(n²) | O(1) | ✅ | ✅ |
| 插入排序 | O(n²) | O(n²) | O(1) | ✅ | ✅ |
| 选择排序 | O(n²) | O(n²) | O(1) | ❌ | ✅ |
| 快速排序 | O(n log n) | O(n²) | O(log n) | ❌ | ✅ |
| 归并排序 | O(n log n) | O(n log n) | O(n) | ✅ | ❌ |
| 堆排序 | O(n log n) | O(n log n) | O(1) | ❌ | ✅ |
| 计数排序 | O(n+k) | O(n+k) | O(k) | ✅ | ❌ |
| 基数排序 | O(d(n+k)) | O(d(n+k)) | O(n+k) | ✅ | ❌ |
| 桶排序 | O(n+k) | O(n²) | O(n+k) | ✅ | ❌ |

> k = 值域范围，d = 最大位数

## 稳定性

**稳定排序**：相等元素的相对顺序不变。
- 需要多键排序时必须用稳定排序（先按次要键排，再按主要键排）

```python
# 稳定性示例
students = [("Alice", 90), ("Bob", 85), ("Carol", 90)]
# 先按分数排序（稳定），再按名字排序
# 稳定排序保证同分时 Alice 仍在 Carol 前面
```

## 原地排序

**原地**：只用 O(1) 额外空间。
- 归并排序非原地（需要 O(n) 合并空间）
- 快排原地但递归栈 O(log n)

## 场景选择指南

```python
# 小数据量 (n < 50)：插入排序
# 实际运行最快，常数因子小，且是稳定排序

# 通用排序：快速排序
# 平均最快，大多数标准库的核心算法
# Python 的 sorted() 用 Timsort（归并+插入混合）

# 需要稳定 + 保证 O(n log n)：归并排序
# 外部排序、链表排序的首选

# 需要 O(1) 空间 + O(n log n)：堆排序
# 对缓存不友好，但空间效率高

# 整数排序且值域小：计数排序
# O(n+k) 线性时间

# 多关键字排序：基数排序
# 按位从低到高，每次用稳定排序
```

## 各算法核心思路

```python
# 冒泡：相邻比较交换
for i in range(n):
    for j in range(n-1-i):
        if arr[j] > arr[j+1]:
            arr[j], arr[j+1] = arr[j+1], arr[j]

# 插入：构建有序序列
for i in range(1, n):
    key = arr[i]
    j = i - 1
    while j >= 0 and arr[j] > key:
        arr[j+1] = arr[j]
        j -= 1
    arr[j+1] = key

# 选择：每轮选最小放到前面
for i in range(n):
    min_idx = i
    for j in range(i+1, n):
        if arr[j] < arr[min_idx]:
            min_idx = j
    arr[i], arr[min_idx] = arr[min_idx], arr[i]
```

## 关键要点

> [!important] 没有"最好"的排序算法
> 最佳选择取决于数据规模、特征（是否近乎有序）、稳定性需求、内存限制

> [!important] O(n log n) 是比较排序的下界
> 任何基于比较的排序算法最坏情况下至少需要 O(n log n) 次比较

> [!important] 工程中的混合排序
> Python Timsort、Java Dual-Pivot Quicksort 都是结合多种算法优点的混合方案
