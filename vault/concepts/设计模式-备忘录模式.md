# Memento 备忘录模式

在不破坏封装的前提下，捕获并外部化对象的内部状态，以便稍后将对象恢复到该状态。

## 意图与场景

- **核心意图**：保存和恢复对象状态，而不暴露其内部实现细节
- **典型场景**：
  - 游戏存档 / 读档
  - 文本编辑器撤销/重做（配合 [[设计模式-命令模式|Command]] 模式）
  - 数据库事务回滚
  - 虚拟机快照
  - 表单草稿保存

## C++ 实现

### Originator + Memento + Caretaker

```cpp
#include <string>
#include <vector>
#include <iostream>
#include <memory>

// Memento：不可变的快照
class Memento {
    // friend 只授予 Originator，不授予 Caretaker
    friend class Document;
    std::string content_;
    int cursorPos_;
    explicit Memento(std::string content, int pos)
        : content_(std::move(content)), cursorPos_(pos) {}
public:
    // Caretaker 只能获取元信息，不能访问状态
    [[nodiscard]] std::string info() const {
        return "Snapshot [cursor=" + std::to_string(cursorPos_)
             + ", len=" + std::to_string(content_.size()) + "]";
    }
};

// Originator：文档
class Document {
    std::string content_;
    int cursorPos_ = 0;
public:
    void type(const std::string& text) {
        content_.insert(cursorPos_, text);
        cursorPos_ += static_cast<int>(text.size());
    }

    void moveCursor(int pos) {
        cursorPos_ = std::clamp(pos, 0, static_cast<int>(content_.size()));
    }

    // 创建快照
    [[nodiscard]] std::unique_ptr<Memento> save() const {
        return std::unique_ptr<Memento>(new Memento(content_, cursorPos_));
    }

    // 从快照恢复
    void restore(const Memento& m) {
        content_ = m.content_;
        cursorPos_ = m.cursorPos_;
    }

    void print() const {
        std::cout << "Content: \"" << content_
                  << "\" | Cursor: " << cursorPos_ << "\n";
    }
};

// Caretaker：历史管理器
class History {
    std::vector<std::unique_ptr<Memento>> snapshots_;
public:
    void push(std::unique_ptr<Memento> m) {
        snapshots_.push_back(std::move(m));
    }

    std::unique_ptr<Memento> pop() {
        if (snapshots_.empty()) return nullptr;
        auto m = std::move(snapshots_.back());
        snapshots_.pop_back();
        return m;
    }

    [[nodiscard]] bool empty() const { return snapshots_.empty(); }
    [[nodiscard]] size_t size() const { return snapshots_.size(); }
};
```

### 使用示例

```cpp
int main() {
    Document doc;
    History history;

    doc.type("Hello");
    history.push(doc.save()); // 保存状态 1

    doc.type(" World");
    history.push(doc.save()); // 保存状态 2

    doc.type("!!!");
    doc.print(); // Content: "Hello World!!!" | Cursor: 14

    // 撤销
    if (auto m = history.pop()) {
        doc.restore(*m);
        doc.print(); // Content: "Hello World" | Cursor: 11
    }

    // 再撤销
    if (auto m = history.pop()) {
        doc.restore(*m);
        doc.print(); // Content: "Hello" | Cursor: 5
    }
}
```

### 深拷贝 vs 序列化

```cpp
// 方式 1：深拷贝（简单直接）
// Memento 持有完整的状态副本

// 方式 2：增量快照（节省内存）
class DeltaMemento {
    // 只记录变化量
    size_t pos_;
    std::string deleted_;
    std::string inserted_;
};

// 方式 3：序列化（支持持久化）
#include <sstream>
#include <archive.h> // libarchive 等

class SerializableMemento {
    std::string serialized_;
public:
    template<typename T>
    static SerializableMemento create(const T& obj) {
        std::ostringstream oss;
        // 序列化逻辑
        return SerializableMemento{oss.str()};
    }
};
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 保持封装性（只有 Originator 能访问状态） | 频繁快照消耗大量内存 |
| 简化 Originator（不必自己管理历史） | 大对象的深拷贝代价高 |
| 支持任意时刻的状态回滚 | Caretaker 需要管理快照生命周期 |
| 实现简单直观 | 如果状态含指针/资源，拷贝语义复杂 |

## 关键要点

> **核心思想**：把"保存状态"和"使用状态"分离。Originator 负责创建/恢复快照，Caretaker 负责持有快照，Memento 本身是不透明的容器。

> **封装的关键**：Memento 的状态应该只对 Originator 可见（`friend class`），Caretaker 不应能读取或修改快照内容。

> **内存优化**：
> - 增量快照：只记录 diff
> - 原型快照：用 [[设计模式-原型模式|Prototype]] 克隆
> - 写时复制（COW）：共享未修改的部分

> **与 Command 协作**：Memento 常与 [[设计模式-命令模式|Command]] 模式配合——Command 的 `undo()` 调用 Originator 的 `restore()`。

## 相关模式

- [[设计模式-命令模式|Command 模式]]：Command 的 undo 机制常用 Memento 保存命令执行前的状态
- [[设计模式-迭代器模式|Iterator 模式]]：Caretaker 的快照栈可以用 Iterator 遍历历史
- [[设计模式-原型模式|Prototype 模式]]：通过克隆实现快照是 Memento 的一种替代方式
