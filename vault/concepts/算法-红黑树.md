# 红黑树

红黑树是一种自平衡 BST，通过颜色标记和旋转保证近似平衡，是工程中最广泛使用的平衡树。

## 五条性质

1. 每个节点要么是红色，要么是黑色
2. 根节点是黑色
3. 叶子节点 (NIL) 是黑色
4. 红色节点的两个子节点必须是黑色（不能有连续红色节点）
5. 从任一节点到其所有叶子的路径上包含相同数量的黑色节点

> 性质 5 保证最长路径不超过最短路径的 2 倍 → 高度 = O(log n)

## 插入修复

插入的节点默认为**红色**，只需处理"父节点也是红色"的冲突。

### 修复策略（叔叔节点分类）

```python
# Case 1：叔叔是红色
#   → 父、叔变黑，祖父变红，向上递归
# Case 2：叔叔是黑，自己是父的右子（LR）
#   → 对父左旋，转化为 Case 3
# Case 3：叔叔是黑，自己是父的左子（LL）
#   → 对祖父右旋，父变黑，祖父变红
```

### 伪代码

```python
def fix_insert(node):
    while node.parent and node.parent.color == RED:
        uncle = get_uncle(node)
        grandparent = node.parent.parent

        if uncle and uncle.color == RED:
            # Case 1
            node.parent.color = BLACK
            uncle.color = BLACK
            grandparent.color = RED
            node = grandparent       # 向上继续
        else:
            if is_left_child(node.parent):
                if is_right_child(node):
                    # Case 2 (LR)
                    node = node.parent
                    left_rotate(node)
                # Case 3 (LL)
                node.parent.color = BLACK
                grandparent.color = RED
                right_rotate(grandparent)
            else:
                # 对称的 RL 和 RR
                ...
    root.color = BLACK  # 根节点始终为黑
```

## 删除修复

删除比插入复杂得多，涉及双黑 (double black) 概念，需要分多种情况讨论。工程中通常参考 CLRS 书的完整实现。

## 工程应用

| 系统 | 应用 |
|------|------|
| Linux 内核 | `rb_tree`：进程调度、内存管理 |
| Java | `TreeMap`, `TreeSet`, `HashMap` (链表转红黑树) |
| C++ STL | `std::map`, `std::set` |
| Nginx | 定时器管理 |
| epoll | 文件描述符管理 |

## 复杂度

| 操作 | 时间 | 说明 |
|------|------|------|
| 查找 | O(log n) | 高度 ≤ 2·log₂(n+1) |
| 插入 | O(log n) | 最多 2 次旋转 |
| 删除 | O(log n) | 最多 3 次旋转 |
| 空间 | O(n) | 每节点 1 bit 颜色 + 指针 |

## 红黑树 vs AVL

| 维度 | 红黑树 | AVL |
|------|--------|-----|
| 平衡程度 | 近似平衡 | 严格平衡 |
| 查找 | 略慢 | ✅ 更快 |
| 插入/删除 | ✅ 旋转少 | 旋转多 |
| 空间 | 1 bit 颜色 | height 字段 |
| 工程首选 | ✅ | 查找密集场景 |

## 关键要点

> [!important] 红黑树 = 2-3-4 树的等价形式
> 颜色标记本质上是节点分裂/合并的另一种表达

> [!important] 插入修复最多旋转 2 次
> 比 AVL 的删除修复简单，旋转次数有保证

> [!important] 工程中的王者
> Linux、Java、C++ 标准库的有序容器底层都是红黑树
