**重定位**是链接器将目标文件中的占位地址替换为最终运行地址的过程，**符号解析**则将未定义引用匹配到具体定义——两者共同完成从多个 `.o` 到可执行文件的转换。

## 重定位的基本问题

汇编器生成 `.o` 时，不知道外部符号的最终地址：

```nasm
; main.s
extern printf
section .text
main:
    push msg
    call printf         ; 汇编器不知道 printf 在哪
    add esp, 4
    
    mov eax, [global_var] ; 全局变量地址也未知
    ret

section .data
    msg db "Hello", 0
    global_var dd 0
```

汇编器在机器码中留下**占位符**，生成**重定位条目**告诉链接器："这个位置需要填入某个符号的地址"。

## 重定位类型

### R_386_32 — 绝对地址重定位

```
最终值 = S + A
S = 符号的最终地址
A = 重定位位置已有的 addend（偏移量）
```

```nasm
; 使用场景：mov eax, [symbol] 中的地址
; 原始机器码: B8 00 00 00 00 (占位符 0)
; 链接后: B8 [最终地址 4字节]
```

### R_386_PC32 — PC 相对重定位

```
最终值 = S + A - P
P = 被重定位位置的地址
```

```nasm
; 使用场景：call printf — 相对于当前指令的偏移
; 原始: E8 00 00 00 00 (call +0, 占位)
; 链接后: E8 [printf_addr - (call指令地址+5)]
```

### 常见重定位类型速查

| 类型 | 公式 | 用途 |
|------|------|------|
| `R_386_32` | S + A | 绝对地址引用 |
| `R_386_PC32` | S + A - P | 相对跳转/call |
| `R_386_GOT32` | G + A | GOT 表项偏移 |
| `R_386_PLT32` | L + A - P | PLT 条目 |
| `R_386_COPY` | — | 数据复制（动态链接） |
| `R_386_GLOB_DAT` | S | 填充 GOT |
| `R_386_JMP_SLOT` | S | PLT 懒绑定 |
| `R_X86_64_PC32` | S + A - P | 64 位相对跳转 |
| `R_X86_64_64` | S + A | 64 位绝对地址 |

## 符号表 (.symtab)

```bash
# 查看符号表
objdump -t main.o
nm main.o
readelf -s main.o
```

```
符号表输出示例：
Num  Value  Size  Type    Bind    Vis     Ndx  Name
 0   0000   0     NOTYPE  LOCAL   DEFAULT UND
 1   0000   0     SECTION LOCAL   DEFAULT 1   .text
 2   0000   32    FUNC    GLOBAL  DEFAULT 1   main
 3   0000   0     NOTYPE  GLOBAL  DEFAULT UND printf
 4   0000   4     OBJECT  GLOBAL  DEFAULT 3   global_var
```

### 符号类型

| 类型 | 含义 |
|------|------|
| `NOTYPE` | 未指定类型 |
| `FUNC` | 函数 |
| `OBJECT` | 数据对象 |
| `SECTION` | 段符号 |
| `FILE` | 源文件名 |

### 符号绑定

| 绑定 | 含义 | 说明 |
|------|------|------|
| `LOCAL` | 局部 | 仅当前文件可见 |
| `GLOBAL` | 全局 | 跨文件可见，必须唯一 |
| `WEAK` | 弱 | 可被 GLOBAL 覆盖，无定义时不报错 |

## 全局 vs 局部 vs 弱符号

```c
// 全局符号 — 强定义
int global_var = 42;
void global_func(void) { }

// 局部符号（static）
static int local_var = 10;
static void local_func(void) { }

// 弱符号（GCC 扩展）
__attribute__((weak)) int weak_var = 100;
__attribute__((weak)) void weak_func(void) { }

// 未初始化的全局（COMMON 符号）
int common_var;             // 链接时合并
```

```nasm
; NASM 中
global strong_sym
global weak_sym:default     ; 弱符号

section .data
strong_sym dd 42
weak_sym   dd 100
```

### 链接器处理规则

```
强 + 强 → 链接错误（多重定义）
强 + 弱 → 选择强定义
弱 + 弱 → 选择任意一个（通常第一个）
```

## PLT/GOT — 动态链接延迟绑定

动态链接库的函数地址在运行时才确定。x86 使用 **PLT**（Procedure Linkage Table）和 **GOT**（Global Offset Table）实现延迟绑定。

### 调用流程

```
call printf
  → 调用 PLT[printf] 条目
    → 第一次：跳转 GOT[printf]（初始指向回 PLT + 下一条指令）
      → 调用动态链接器 (_dl_runtime_resolve)
      → 解析 printf 真实地址
      → 写入 GOT[printf]
      → 跳转到真正的 printf
    → 以后：直接跳转 GOT[printf] → 真正的 printf（无额外开销）
```

### 伪代码表示

```nasm
; PLT 条目结构
plt_printf:
    jmp *GOT_PRINTF           ; 跳转 GOT 表项
    push offset "printf"      ; 符号索引
    jmp PLT_RESOLVER          ; 调用解析器

; GOT 初始状态（第一次调用前）
GOT_PRINTF: dd plt_printf + 6  ; 指回 PLT 下一条指令

; 解析后
GOT_PRINTF: dd 0x7f1234567890  ; printf 的真实地址
```

### 查看 PLT/GOT

```bash
objdump -d -j .plt program     # PLT 反汇编
objdump -s -j .got.plt program # GOT 内容
readelf -r program             # 重定位条目
```

## 关键要点

> **重定位 = 填地址**：汇编器留下占位符，链接器填入最终值。两种核心类型：绝对（`S+A`）和相对（`S+A-P`）。

> **符号绑定决定了可见性和覆盖规则**：`GLOBAL` 跨文件可见（强），`WEAK` 可被覆盖，`LOCAL` 仅本文件。

> **PLT/GOT 实现动态链接延迟绑定**：第一次调用时解析地址并缓存到 GOT，后续调用直接跳转——零额外开销。

> **R_386_PC32 是最常见的重定位类型**：`call` 和 `jmp` 指令几乎都是相对寻址，因为位置无关代码（PIC）天然支持。
