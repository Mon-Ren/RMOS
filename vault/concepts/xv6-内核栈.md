# xv6 内核栈

每个用户进程都有独立的内核栈，用于在内核模式下执行系统调用、处理中断和陷阱时保存状态。

## 核心机制

### 内核栈分配

`allocproc()` 在进程创建时分配内核栈：

```c
// proc.c:73 — allocproc 分配内核栈
if((p->kstack = kalloc()) == 0){
  p->state = UNUSED;
  return 0;
}
sp = p->kstack + KSTACKSIZE;  // KSTACKSIZE = 4096 (1 页)
```

每个进程的内核栈就是一个物理页（4KB），通过 `kalloc()` 从空闲内存池分配。

### 内核栈布局

内核栈从高到低依次存放：

```
高地址 (p->kstack + KSTACKSIZE)
┌─────────────────────┐
│   未使用空间         │
├─────────────────────┤
│   trapframe          │ ← p->tf 指向这里
│   (保存用户态寄存器)  │
├─────────────────────┤
│   返回地址 (trapret) │ ← *(uint*)sp = trapret
├─────────────────────┤
│   context            │ ← p->context 指向这里
│   (内核上下文切换)    │
└─────────────────────┘
低地址 (p->kstack)
```

### 用途

- **系统调用**：用户程序执行 `INT T_SYSCALL` 时，CPU 切换到内核栈
- **中断处理**：硬件中断发生时，trapframe 压入内核栈
- **上下文切换**：`swtch.S` 在内核栈上保存/恢复寄存器
- **内核函数调用**：内核态的所有函数调用都使用该栈

### 与用户栈的关系

xv6 中用户栈和内核栈是**分离的**：
- **用户栈**：在进程数据段内，`sp` 寄存器指向
- **内核栈**：独立分配的物理页，进入内核态时使用

```
用户态: %esp → 用户栈（在进程地址空间内）
     ↓ INT / 中断
内核态: %esp → 内核栈（独立物理页）
     ↓ IRET
用户态: %esp → 用户栈
```

### 内核栈固定大小的限制

内核栈只有 4KB（一个页），没有动态扩展机制。如果内核函数嵌套过深或使用大局部变量，会**栈溢出**并导致未定义行为。这是 xv6 的一个简化设计——生产 OS 通常使用更大的内核栈（8KB-16KB）并有溢出检测。

## 关键要点
> xv6 每个进程拥有独立的 4KB 内核栈，在 allocproc 时分配，用于系统调用和中断处理。内核栈与用户栈分离，固定大小不可扩展。
