# x86 实模式与保护模式

x86 CPU 启动时处于实模式，必须手动切换到保护模式才能使用现代 OS 特性。

## 实模式（Real Mode）

- **16 位**，地址总线 20 位，寻址 1MB
- 地址计算：`物理地址 = 段寄存器 × 16 + 偏移`（段基址左移 4 位）
- 无内存保护：任何代码可以访问任意地址
- 无特权级：所有代码平等
- 直接访问硬件 IO 端口
- BIOS 中断可用（`int 0x10` 显示、`int 0x13` 磁盘）

### 段寄存器

CS、DS、SS、ES、FS、GS——实模式下直接存储段基址的高 16 位。

## 保护模式（Protected Mode）

- **32 位**，地址总线 32 位，寻址 4GB
- 地址转换：`段选择子 → GDT → 段基址 + 偏移 → 线性地址 → 页表 → 物理地址`
- 内存保护：页级读/写/执行权限
- 特权级：Ring 0（内核）到 Ring 3（用户）
- 支持分页、多任务、硬件保护

### 段选择子

```
 15                    3  2  1  0
┌──────────────────────┬──┬─────┐
│      Index (13b)     │TI│RPL  │
└──────────────────────┴──┴─────┘
TI: 0=GDT, 1=LDT
RPL: Requested Privilege Level (0-3)
```

## 切换过程（xv6 bootasm.S）

```asm
# 1. 关中断
cli

# 2. 开启 A20 地址线（突破 1MB 限制）
# 写 0xd1 到端口 0x64，写 0xdf 到端口 0x60

# 3. 加载 GDT
lgdt gdtdesc

# 4. 设置 CR0 的 PE 位
movl %cr0, %eax
orl  $CR0_PE, %eax
movl %eax, %cr0

# 5. 长跳转刷新 CS
ljmp $(SEG_KCODE<<3), $start32

# 6. 设置数据段寄存器
.code32
movw $(SEG_KDATA<<3), %ax
movw %ax, %ds
```

### A20 地址线

8086 只有 20 位地址，地址回绕（`0xFFFF:0xFFFF = 0x10FFEF` 回到 `0xFFEF`）。80286+ 有 24 位，但为了兼容，A20 默认关闭。必须手动开启才能访问 1MB 以上内存。

## 关键要点

> 实模式是 x86 的历史包袱——CPU 必须先以 8086 兼容模式启动，然后才能切换到现代保护模式。A20 开启 + GDT 加载 + CR0.PE 置位是三步缺一不可。理解这段代码是理解整个 x86 boot 流程的起点。
