# 哈希表

哈希表通过哈希函数将键映射到数组下标，实现均摊 O(1) 的查找、插入、删除。

## 哈希函数

好的哈希函数应满足：
1. **确定性**：同一键始终返回同一哈希值
2. **均匀性**：哈希值尽量均匀分布
3. **高效性**：计算速度快

```python
# 简单哈希函数示例
def simple_hash(key, table_size):
    h = 0
    for ch in str(key):
        h = (h * 31 + ord(ch)) % table_size
    return h
```

## 冲突解决

### 1. 链地址法 (Chaining)

每个槽位存一个链表，冲突元素追加到链表。

```python
class HashTableChaining:
    def __init__(self, size=16):
        self.size = size
        self.buckets = [[] for _ in range(size)]

    def put(self, key, value):
        idx = hash(key) % self.size
        for i, (k, v) in enumerate(self.buckets[idx]):
            if k == key:
                self.buckets[idx][i] = (key, value)
                return
        self.buckets[idx].append((key, value))

    def get(self, key):
        idx = hash(key) % self.size
        for k, v in self.buckets[idx]:
            if k == key:
                return v
        raise KeyError(key)
```

### 2. 开放寻址 (Open Addressing)

冲突时在表内寻找下一个空槽。

```python
# 线性探测
def linear_probe(hash_table, key, i, size):
    return (hash(key) + i) % size

# 二次探测
def quadratic_probe(hash_table, key, i, size):
    return (hash(key) + i * i) % size

# 双重哈希
def double_hash(key, i, size):
    h1 = hash(key) % size
    h2 = 1 + (hash(key) % (size - 1))
    return (h1 + i * h2) % size
```

## 负载因子与扩容

**负载因子 α = 已存储元素数 / 表大小**

- 链地址法：α 可以 > 1
- 开放寻址：α 必须 < 1（通常 α > 0.7 时扩容）

**扩容**：创建 2 倍大小的新表，重新哈希所有元素 → O(n)，但均摊到每次操作为 O(1)。

## 复杂度

| 操作 | 平均 | 最坏（所有 key 冲突） |
|------|------|----------------------|
| 查找 | O(1) | O(n) |
| 插入 | O(1) | O(n) |
| 删除 | O(1) | O(n) |

> 最坏情况在好的哈希函数下极少发生，实际视为 O(1)。

## Python dict 实现

```python
# Python dict 内部使用开放寻址 + 伪随机探测
d = {}
d["apple"] = 3      # 均摊 O(1)
val = d["apple"]     # 均摊 O(1)
del d["apple"]       # 均摊 O(1)

# Python 3.7+ dict 保持插入顺序
```

## 关键要点

> [!important] 均摊 O(1) 不等于每次 O(1)
> 扩容时单次操作可达 O(n)，但 n 次操作总计 O(n)

> [!important] 哈希碰撞攻击
> 恶意构造大量冲突 key 可将哈希表退化为 O(n)，用随机哈希种子防御

> [!important] 负载因子是关键
> 负载因子过高 → 冲突多 → 性能下降；过低 → 浪费空间
