x86 提供完整的算术运算指令集，涵盖加减乘除和各种变体，运算结果会影响 [[标志寄存器 EFLAGS]]。

## 加法

### ADD — 加法

```asm
add eax, ebx       ; EAX += EBX
add eax, 42        ; EAX += 42
add BYTE PTR [var], 1  ; *var += 1

; 影响的标志：CF, ZF, SF, OF, AF, PF
```

### ADC — 带进位加法 (Add with Carry)

```asm
; ADC dest, src → dest = dest + src + CF
; 用于大数加法（超过 32/64 位）

; 64 位加法（在 32 位系统上）：
; result = a + b（a,b 均为 64 位，存储在两个 32 位寄存器中）
mov eax, [a_low]
add eax, [b_low]       ; 低 32 位相加，设置 CF
mov edx, [a_high]
adc edx, [b_high]      ; 高 32 位相加 + 进位

; 128 位加法同理
```

### INC — 自增

```asm
inc eax            ; EAX++
inc DWORD PTR [counter]  ; counter++

; INC 不影响 CF！（其他标志正常更新）
; 这就是为什么 LOOP 用 DEC 而不是 INC 来检测计数
```

## 减法

### SUB — 减法

```asm
sub eax, ebx       ; EAX -= EBX
sub esp, 16        ; 分配局部变量空间

; 影响的标志：CF(借位), ZF, SF, OF, AF, PF
```

### SBB — 带借位减法 (Subtract with Borrow)

```asm
; SBB dest, src → dest = dest - src - CF
; 用于大数减法

; 64 位减法（32 位系统）：
mov eax, [a_low]
sub eax, [b_low]       ; 低 32 位相减，设置 CF（借位）
mov edx, [a_high]
sbb edx, [b_high]      ; 高 32 位相减 - 借位
```

### DEC — 自减

```asm
dec ecx            ; ECX--
dec DWORD PTR [counter]  ; counter--

; DEC 不影响 CF！
```

### NEG — 取反（求负）

```asm
neg eax            ; EAX = -EAX（等价于 0 - EAX）
neg eax            ; NEG 操作：CF = (EAX != 0)，ZF = (EAX == 0)

; 对 0 取反
mov eax, 0
neg eax            ; EAX = 0, CF = 0, ZF = 1

; 对最小值取反（溢出）
mov eax, 0x80000000
neg eax            ; EAX = 0x80000000, OF = 1
```

## 乘法

### MUL — 无符号乘法

```asm
; 8 位:  AX = AL × r/m8
mul bl               ; AX = AL * BL

; 16 位: DX:AX = AX × r/m16
mul bx               ; DX:AX = AX * BX

; 32 位: EDX:EAX = EAX × r/m32
mul ebx              ; EDX:EAX = EAX * EBX

; 64 位: RDX:RAX = RAX × r/m64
mul rbx              ; RDX:RAX = RAX * RBX
```

### IMUL — 有符号乘法

```asm
; 单操作数形式（同 MUL）
imul ebx             ; EDX:EAX = EAX * EBX（有符号）

; 双操作数形式
imul eax, ebx        ; EAX = EAX * EBX
imul eax, ecx, 10    ; EAX = ECX * 10（优化器常用）

; 三操作数形式
imul eax, [array], 4  ; EAX = array[0] * 4
```

> [!info] MUL vs IMUL
> - MUL 结果是无符号的，且总是存入两倍宽度的寄存器
> - IMUL 的单操作数形式同 MUL
> - IMUL 的双/三操作数形式只保留低位（编译器常用）
> - 如果高位全是 0（MUL）或符号扩展（IMUL），CF=OF=0

## 除法

### DIV — 无符号除法

```asm
; 8 位:  AL = AX / r/m8,  AH = AX % r/m8
mov ax, 100
mov bl, 7
div bl               ; AL = 14 (商), AH = 2 (余数)

; 32 位: EAX = EDX:EAX / r/m32, EDX = 余数
xor edx, edx         ; 清除高位！
mov eax, 1000
mov ecx, 7
div ecx              ; EAX = 142 (商), EDX = 6 (余数)
```

### IDIV — 有符号除法

```asm
; 使用前需要用 CDQ 符号扩展 EAX 到 EDX:EAX
mov eax, -100
cdq                  ; EDX:EAX = 符号扩展的 -100
mov ecx, 7
idiv ecx             ; EAX = -14 (商), EDX = -2 (余数)
```

> [!warning] 除法陷阱
> - DIV/IDIV 除以 0 触发 `#DE` 异常（除法错误）
> - 商溢出（结果超出目标寄存器范围）也触发 `#DE`
> - **必须**在 DIV 前清零 EDX（或在 IDIV 前用 CDQ）

## 常量乘法优化

编译器常用 LEA + 移位代替乘法：

```asm
; EAX * 3
lea eax, [eax + eax*2]     ; EAX = EAX + EAX*2

; EAX * 5
lea eax, [eax + eax*4]     ; EAX = EAX + EAX*4

; EAX * 9
lea eax, [eax + eax*8]     ; EAX = EAX + EAX*8

; EAX * 10
lea eax, [eax + eax*4]     ; EAX * 5
add eax, eax               ; * 2 → EAX * 10
```

> [!tip] 关键要点
> - ADD/SUB 影响所有算术标志
> - INC/DEC **不影响 CF**（这点很特殊）
> - ADC/SBB 用于大数运算，利用进位链
> - MUL/DIV 隐式使用 EDX:EAX，注意准备高位
> - IDIV 前用 CDQ 做符号扩展
> - 常量乘法优先用 LEA 优化
