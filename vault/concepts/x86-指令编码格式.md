x86 指令采用变长编码格式，由多个可选字段组成，从 1 字节到 15 字节不等，理解编码格式有助于调试和逆向工程。

## 指令格式总览

```
┌──────────┬─────────┬────────┬─────┬─────┬──────────────┬────────────┐
│ Prefixes │ Opcode  │ ModR/M │ SIB │ Disp│  Immediate   │            │
│ (0-4 B)  │ (1-3 B) │(0-1 B) │(0-1)│(0-4)│   (0-4 B)    │            │
└──────────┴─────────┴────────┴─────┴─────┴──────────────┴────────────┘
  可选        必需      可选     可选  可选     可选
```

## 1. 指令前缀 (Prefixes, 0-4 字节)

可选的前缀字节，可以组合使用：

| 前缀类别 | 值 | 作用 |
|---------|-----|------|
| LOCK    | F0  | 原子操作锁 |
| REP     | F3  | 重复前缀（字符串） |
| REPNE   | F2  | 不等重复前缀 |
| 段覆盖  | 2E,36,3E,26,64,65 | CS/SS/DS/ES/FS/GS |
| 操作数大小 | 66 | 切换 16/32 位操作数 |
| 地址大小 | 67 | 切换 16/32 位地址 |

```asm
; LOCK 前缀示例
lock xchg [counter], eax    ; 编码：F0 87 05 ...

; 操作数大小覆盖
mov ax, 0x1234              ; 编码：66 B8 34 12（加 66 前缀切换到 16 位）
```

## 2. 操作码 (Opcode, 1-3 字节)

指令的核心标识。大部分指令用 1 字节，复杂指令用 2-3 字节。

```
单字节示例：
  B8+rd  → MOV r32, imm32    (B8=MOV EAX,imm / B9=MOV ECX,imm / ...)
  01 /r  → ADD r/m32, r32
  C3     → RET
  90     → NOP

两字节示例（0F 前缀）：
  0F 84  → JE rel32 (条件跳转)
  0F AF  → IMUL r32, r/m32
```

## 3. ModR/M 字段 (0-1 字节)

指定操作数的寻址方式：

```
  7  6  5  4  3  2  1  0
┌──────┬────────┬────────┐
│ Mod  │  Reg   │  R/M   │
│ (2位)│  (3位) │  (3位) │
└──────┴────────┴────────┘
```

- **Mod (2 位)**：寻址模式（00=无偏移, 01=8位偏移, 10=32位偏移, 11=寄存器）
- **Reg (3 位)**：寄存器编号或 opcode 扩展
- **R/M (3 位)**：寄存器或内存操作数

```asm
; MOV EAX, EBX 的 ModR/M = 11 000 011 = C3
; Mod=11(寄存器), Reg=000(EAX), R/M=011(EBX)

; MOV EAX, [EBX] 的 ModR/M = 00 000 011 = 03
; Mod=00(无偏移), Reg=000(EAX), R/M=011(EBX 指向的内存)
```

## 4. SIB 字段 (Scale-Index-Base, 0-1 字节)

当 R/M = 100（ESP）且 Mod ≠ 11 时，需要 SIB 字节实现复杂寻址：

```
  7  6  5  4  3  2  1  0
┌──────┬────────┬────────┐
│ Scale│ Index  │  Base  │
│ (2位)│  (3位) │  (3位) │
└──────┴────────┴────────┘
```

- **Scale**：00=×1, 01=×2, 10=×4, 11=×8
- **Index**：索引寄存器（无索引用 ESP 编码）
- **Base**：基址寄存器

## 5. 偏移量 (Displacement, 0/1/2/4 字节)

Mod ≠ 11 时可能存在的立即偏移量：

- Mod=00, R/M≠101 → 无偏移
- Mod=01 → 1 字节有符号偏移
- Mod=10 或 (Mod=00, R/M=101) → 4 字节偏移

## 6. 立即数 (Immediate, 0/1/2/4 字节)

指令中的常量值。

## 完整示例分析

### `MOV EAX, [EBX+ECX*4+8]`

```
指令字节：  8B 44 8B 08
            │  │  │  └── Displacement = 0x08
            │  │  └───── SIB = 10 001 011
            │  │            Scale=10(×4), Index=001(ECX), Base=011(EBX)
            │  └──────── ModR/M = 01 000 100
            │               Mod=01(8位偏移), Reg=000(EAX), R/M=100(SIB follows)
            └─────────── Opcode = 8B (MOV r32, r/m32)

操作数大小 = 32 位（无 66 前缀）
地址大小 = 32 位（无 67 前缀）
```

### `ADD DWORD PTR [EBP-4], 7`

```
指令字节：  83 45 FC 07
            │  │  │  └── Immediate = 0x07
            │  │  └───── Displacement = 0xFC (-4 的补码)
            │  └──────── ModR/M = 01 000 101
            │               Mod=01(8位偏移), Reg=000(opcode扩展=ADD), R/M=101(EBP)
            └─────────── Opcode = 83 (ADD r/m32, imm8 — 符号扩展)
```

> [!tip] 关键要点
> - x86 指令是变长的（1-15 字节），理解编码格式对逆向和调试至关重要
> - ModR/M 是理解寻址方式的核心：Mod 决定偏移类型，R/M 决定操作数来源
> - SIB 仅在需要 `[base+index*scale+disp]` 形式时出现
> - 66 前缀切换操作数大小，67 前缀切换地址大小
> - Opcode 的 "+rd" 形式直接将寄存器编号编码进操作码（如 B8+EAX→B8）
