# 冒泡排序与插入排序

两种最基础的 O(n²) 排序算法，简单直观，适合小数据量和教学。

## 冒泡排序

反复比较相邻元素，将较大的元素"冒泡"到数组末尾。

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        swapped = False
        for j in range(0, n - 1 - i):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:    # 提前终止优化
            break
    return arr
```

**复杂度**：
- 最好：O(n)（已排序，加 swapped 优化后）
- 最坏/平均：O(n²)
- 空间：O(1)
- 稳定：✅

## 插入排序

逐个将元素插入到已排序部分的正确位置，类似扑克牌整理手牌。

```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]    # 后移
            j -= 1
        arr[j + 1] = key           # 插入
    return arr
```

**复杂度**：
- 最好：O(n)（已排序，内层 while 不执行）
- 最坏/平均：O(n²)
- 空间：O(1)
- 稳定：✅

## 对比

| 特性 | 冒泡排序 | 插入排序 |
|------|----------|----------|
| 交换次数 | 多（相邻交换） | 少（赋值代替交换） |
| 最好情况 | O(n) | O(n) |
| 近乎有序 | O(n) | O(n) ✅ 更优 |
| 实际效率 | 较慢 | **小数据量最快之一** |
| 稳定性 | ✅ | ✅ |

## 为什么插入排序实际很快

```python
# 原因 1：内层循环简单——只有比较和移动
# 原因 2：对近乎有序数据，内层循环几乎不执行
# 原因 3：缓存友好——顺序访问
# 原因 4：常数因子极小

# Python stdlib 的 Timsort 和 C++ std::sort 都在
# 小子数组（通常 n < 16~64）时切换为插入排序
```

## 希尔排序（插入排序的改进）

通过间隔序列分组，逐步缩小间隔到 1：

```python
def shell_sort(arr):
    n = len(arr)
    gap = n // 2
    while gap > 0:
        for i in range(gap, n):
            temp = arr[i]
            j = i
            while j >= gap and arr[j - gap] > temp:
                arr[j] = arr[j - gap]
                j -= gap
            arr[j] = temp
        gap //= 2
    return arr
```

希尔排序最坏可达 O(n^1.3)，打破了 O(n²) 的限制。

## 关键要点

> [!important] 小数据量用插入排序
> n < 50 时插入排序通常比快排/归并更快，常数因子极小

> [!important] 插入排序是稳定排序
> 冒泡也是稳定排序，但实际中插入排序更常用

> [!important] 工程中的角色
> Timsort 和 Introsort 都以插入排序作为小规模子问题的处理方案
