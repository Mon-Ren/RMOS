# 时间复杂度与空间复杂度

算法复杂度是衡量算法效率的核心指标，用 Big-O 记法描述算法随输入规模增长的趋势。

## Big-O 常见阶

| 复杂度 | 名称 | 示例 |
|--------|------|------|
| O(1) | 常数 | 数组随机访问、哈希查找 |
| O(log n) | 对数 | 二分查找 |
| O(n) | 线性 | 遍历数组 |
| O(n log n) | 线性对数 | 归并排序、快排平均 |
| O(n²) | 平方 | 冒泡排序、双重循环 |
| O(2ⁿ) | 指数 | 子集枚举、暴力递归 |

> [!tip] 记忆口诀
> 常数 < 对数 < 线性 < 线性对数 < 平方 < 指数 < 阶乘

## 最好、最坏、平均情况

以快速排序为例：
- **最好**：每次 partition 均匀分 → O(n log n)
- **最坏**：每次 partition 只分出一个元素 → O(n²)
- **平均**：随机 pivot，期望 O(n log n)

通常用 **最坏情况** 作为算法复杂度的标准度量。

## 主定理 (Master Theorem)

对于形如 T(n) = aT(n/b) + O(nᵈ) 的递推式：

```python
# 设 a ≥ 1, b > 1, d ≥ 0
# 三种情况：
# 1. log_b(a) > d  → T(n) = O(n^(log_b(a)))
# 2. log_b(a) = d  → T(n) = O(n^d · log n)
# 3. log_b(a) < d  → T(n) = O(n^d)
```

**例**：归并排序 T(n) = 2T(n/2) + O(n)
- a=2, b=2, d=1 → log_2(2) = 1 = d → T(n) = O(n log n)

## 摊还分析 (Amortized Analysis)

某些操作偶尔代价高，但整体均摊后很低。

**例：动态数组扩容**
- 单次 `append` 可能触发 O(n) 的扩容
- 均摊下来每次 insert 仍为 O(1)

摊还方法：
1. **聚集分析**：n 次操作总代价 / n
2. **记账法**：便宜操作多付费，补贴贵操作
3. **势能法**：定义势函数，ΔΦ = 摊还代价 - 实际代价

## Python 示例

```python
# O(1) — 哈希查找
d = {i: i*i for i in range(100)}
val = d[50]  # 常数时间

# O(n) — 线性扫描
def find_max(arr):
    result = arr[0]
    for x in arr:       # 遍历 n 次
        if x > result:
            result = x
    return result

# O(n²) — 双重循环
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):          # n 次
        for j in range(n-1-i):  # 最多 n 次
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]

# O(2^n) — 子集枚举
def subsets(arr):
    if not arr:
        return [[]]
    rest = subsets(arr[1:])     # 两次递归调用
    return rest + [[arr[0]] + s for s in rest]
```

## 关键要点

> [!important] 复杂度只看趋势
> Big-O 忽略常数和低阶项。O(100n) = O(n)，O(n² + n) = O(n²)

> [!important] 空间复杂度 = 额外空间
> 不计输入本身占用的空间，只算算法运行时申请的额外内存

> [!important] 时间-空间权衡
> 多数算法可以通过牺牲空间来换取时间（如哈希表缓存），反之亦然
