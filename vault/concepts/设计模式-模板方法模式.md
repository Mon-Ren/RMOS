# Template Method 模板方法模式

定义一个操作中的算法骨架，将某些步骤延迟到子类中实现。模板方法让子类在不改变算法结构的前提下重定义算法的某些步骤。

## 意图与场景

- **核心意图**：封装不变部分，扩展可变部分
- **典型场景**：
  - 数据解析框架（解析流程固定，具体格式可变）
  - 测试框架（setup → test → teardown 固定，测试内容可变）
  - 构建系统（构建流程固定，编译/链接步骤可变）
  - 渲染管线（固定管线，具体着色器可变）
  - 数据库访问模板（连接 → 执行 → 释放）

## C++ 实现

### 非虚接口惯用法（NVI, Non-Virtual Interface）

```cpp
#include <string>
#include <iostream>
#include <fstream>
#include <sstream>

// 基类：数据导出器
class DataExporter {
public:
    // 模板方法：固定算法骨架，不可覆盖
    void exportData(const std::string& filepath) {
        openFile(filepath);
        writeHeader();       // 抽象步骤
        writeBody();         // 抽象步骤
        writeFooter();       // 钩子（可选覆盖）
        closeFile();
    }

    virtual ~DataExporter() = default;

protected:
    // NVI：public 非虚方法调用 protected 虚方法
    virtual void writeHeader() = 0;
    virtual void writeBody() = 0;

    // 钩子方法：有默认实现，子类可选覆盖
    virtual void writeFooter() {
        stream_ << "<!-- Generated by DataExporter -->\n";
    }

    // 实现细节
    std::ostringstream stream_;

private:
    void openFile(const std::string& path) {
        std::cout << "打开文件: " << path << "\n";
    }
    void closeFile() {
        std::cout << "关闭文件，内容:\n" << stream_.str();
    }
};

// 具体实现：CSV 导出
class CsvExporter : public DataExporter {
protected:
    void writeHeader() override {
        stream_ << "Name,Age,City\n";
    }
    void writeBody() override {
        stream_ << "Alice,30,Beijing\n";
        stream_ << "Bob,25,Shanghai\n";
    }
    void writeFooter() override {
        stream_ << "# End of CSV data\n";
    }
};

// 具体实现：JSON 导出
class JsonExporter : public DataExporter {
protected:
    void writeHeader() override {
        stream_ << "[\n";
    }
    void writeBody() override {
        stream_ << R"(  {"name":"Alice","age":30},)" << "\n";
        stream_ << R"(  {"name":"Bob","age":25})" << "\n";
    }
    // 使用默认 writeFooter
};
```

### CRTP 编译时替代（零虚函数开销）

```cpp
// Curiously Recurring Template Pattern
template<typename Derived>
class DataParser {
public:
    // 模板方法
    void parse(const std::string& input) {
        auto tokens = static_cast<Derived*>(this)->tokenize(input);
        auto ast = static_cast<Derived*>(this)->buildAST(tokens);
        static_cast<Derived*>(this)->execute(ast);
    }
};

class JsonParser : public DataParser<JsonParser> {
public:
    auto tokenize(const std::string& input) {
        std::cout << "[JSON] 词法分析\n";
        return std::vector<std::string>{};
    }
    auto buildAST(const std::vector<std::string>& tokens) {
        std::cout << "[JSON] 构建 AST\n";
        return std::string{"json_ast"};
    }
    void execute(const std::string& ast) {
        std::cout << "[JSON] 执行解析\n";
    }
};

// 使用
// JsonParser parser;
// parser.parse("{\"key\": \"value\"}");
// 编译器直接内联，无虚函数开销
```

### C++20 Concepts 约束的模板方法

```cpp
#include <concepts>

template<typename T>
concept Exportable = requires(T t, std::string s) {
    { t.doWriteHeader(s) } -> std::convertible_to<std::string>;
    { t.doWriteBody(s) } -> std::convertible_to<std::string>;
};

template<Exportable T>
void exportTemplate(T& impl, const std::string& data) {
    std::cout << "=== 开始导出 ===\n";
    std::cout << impl.doWriteHeader(data);
    std::cout << impl.doWriteBody(data);
    std::cout << "=== 导出完成 ===\n";
}
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 复用公共代码，避免重复 | 维护成本：修改模板方法影响所有子类 |
| 符合开闭原则 | 子类必须实现所有抽象步骤 |
| 控制反转：父类调用子类 | 层级过深时难以理解控制流 |
| 算法结构集中可见 | 子类与基类强耦合（继承关系） |

## 关键要点

> **核心思想**：将"不变的部分"（算法骨架）和"变化的部分"（具体步骤）分离。不变的放基类，变化的放子类。

> **NVI 惯用法**：C++ 中推荐的做法——public 非虚方法（模板方法）调用 protected 虚方法（步骤）。好处：
> - 模板方法不可被子类覆盖
> - 可以在虚调用前后添加前置/后置逻辑
> - 接口（public）和扩展点（protected）分离清晰

> **CRTP vs 虚函数**：
> - 虚函数：运行时多态，灵活但有开销
> - CRTP：编译时多态，零开销但丧失运行时灵活性
> - 选择：需要异构容器存不同子类 → 虚函数；类型在编译期确定 → CRTP

> **Template Method vs Strategy**：Template Method 通过继承改变部分步骤，[[设计模式-策略模式|Strategy]] 通过组合改变整个算法。继承 vs 组合的典型权衡。

## 相关模式

- [[设计模式-策略模式|Strategy 模式]]：Template Method 用继承变体算法，Strategy 用组合替换整个算法
- [[设计模式-访问者模式|Visitor 模式]]：两者都涉及双重分发，但意图不同
- [[设计模式-工厂方法模式|Factory Method 模式]]：Factory Method 是 Template Method 的特例——模板方法中"创建对象"这一步由子类实现
