# Mediator 中介者模式

用中介对象封装一系列对象的交互，使各对象不需要显式地相互引用。

## 意图与场景

- **核心意图**：将多对多的网状通信转换为一对多的星型通信，降低耦合
- **典型场景**：
  - 聊天室（用户之间不直接通信）
  - 航空管制系统（飞机通过塔台协调）
  - GUI 组件协调（按钮点击影响列表刷新、状态栏更新）
  - 微服务 API Gateway
  - 游戏中 NPC 之间的协调

## C++ 实现

### Mediator 接口 + Colleague 注册

```cpp
#include <string>
#include <vector>
#include <iostream>
#include <algorithm>
#include <memory>

// 前置声明
class Colleague;

// 中介者接口
class Mediator {
public:
    virtual ~Mediator() = default;
    virtual void notify(Colleague* sender, const std::string& event) = 0;
    virtual void addColleague(std::unique_ptr<Colleague> col) = 0;
};

// 同事基类
class Colleague {
protected:
    Mediator* mediator_;
    std::string name_;
public:
    Colleague(Mediator* med, std::string name)
        : mediator_(med), name_(std::move(name)) {}
    virtual ~Colleague() = default;

    const std::string& name() const { return name_; }
    virtual void receive(const std::string& msg, Colleague* from) {
        std::cout << "[" << name_ << "] 收到来自 ["
                  << from->name() << "] 的消息: " << msg << "\n";
    }
    void send(const std::string& msg) {
        std::cout << "[" << name_ << "] 发送: " << msg << "\n";
        if (mediator_) mediator_->notify(this, msg);
    }
};

// 具体中介者：聊天室
class ChatRoom : public Mediator {
    std::vector<std::unique_ptr<Colleague>> members_;
public:
    void addColleague(std::unique_ptr<Colleague> col) override {
        members_.push_back(std::move(col));
    }

    void notify(Colleague* sender, const std::string& event) override {
        // 广播给除发送者外的所有成员
        for (auto& m : members_) {
            if (m.get() != sender) {
                m->receive(event, sender);
            }
        }
    }
};

// 具体同事：用户
class User : public Colleague {
public:
    using Colleague::Colleague;
};
```

### 使用示例

```cpp
int main() {
    auto room = std::make_unique<ChatRoom>();

    auto alice = std::make_unique<User>(room.get(), "Alice");
    auto bob   = std::make_unique<User>(room.get(), "Bob");
    auto carol = std::make_unique<User>(room.get(), "Carol");

    // 先添加到房间
    room->addColleague(std::move(alice));
    room->addColleague(std::move(bob));
    room->addColleague(std::move(carol));

    // 通过中介者通信（指针已被 unique_ptr 管理，这里简化演示）
    // auto* a = 获取成员引用;
    // a->send("大家好！");
}
```

### 现代简化：基于 `std::function` 的事件总线

```cpp
#include <functional>
#include <unordered_map>
#include <vector>

class EventBus {
    using Handler = std::function<void(const std::string&)>;
    std::unordered_map<std::string, std::vector<Handler>> subscribers_;
public:
    void subscribe(const std::string& event, Handler h) {
        subscribers_[event].push_back(std::move(h));
    }
    void publish(const std::string& event, const std::string& data) {
        if (auto it = subscribers_.find(event); it != subscribers_.end()) {
            for (const auto& h : it->second) h(data);
        }
    }
};
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 减少对象间耦合（网状 → 星型） | 中介者可能变得过于复杂（上帝对象） |
| 集中控制交互逻辑 | 中介者承担过多职责 |
| 符合单一职责（各 Colleague 只关心自己） | 中介者成为新的耦合点 |
| 易于新增/删除同事对象 | 调试时跳转增多 |

## 关键要点

> **核心思想**：当多个对象之间的通信变成"谁都认识谁"的网状结构时，引入一个中介者来集中管理交互，将网状通信转为星型。

> **中介者 vs 观察者**：[[设计模式-观察者模式|Observer]] 是广播通知（一对多），Mediator 是协调转发（多对多的中转站）。两者可以结合使用。

> **避免上帝对象**：中介者模式的核心风险是中介者本身变成"知道太多"的 God Object。应按功能拆分中介者，或用事件总线分散职责。

> **适用判断**：当你发现对象之间有大量交叉引用，修改一个类需要修改多个关联类时，就是引入中介者的信号。

## 相关模式

- [[设计模式-观察者模式|Observer 模式]]：Mediator 内部可以用 Observer 机制实现通知
- [[设计模式-外观模式|Facade 模式]]：Facade 是子系统的简化接口（单向），Mediator 是对象间的协调（双向）
- [[设计模式-命令模式|Command 模式]]：中介者可以用 Command 对象来封装交互
