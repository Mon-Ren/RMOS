# 快速排序

快速排序是最常用的基于比较的排序算法，平均 O(n log n)，原地排序，通过 partition 将数组分为"小于 pivot"和"大于 pivot"两部分。

## 核心思想

```
1. 选 pivot
2. Partition：将数组分为 ≤pivot 和 ≥pivot 两部分
3. 递归排序两部分
```

## Lomuto Partition

```python
def quicksort(arr, lo, hi):
    if lo < hi:
        p = partition_lomuto(arr, lo, hi)
        quicksort(arr, lo, p - 1)
        quicksort(arr, p + 1, hi)

def partition_lomuto(arr, lo, hi):
    pivot = arr[hi]           # 选最后一个元素作 pivot
    i = lo                    # i 指向"小于区"的下一个位置
    for j in range(lo, hi):
        if arr[j] <= pivot:
            arr[i], arr[j] = arr[j], arr[i]
            i += 1
    arr[i], arr[hi] = arr[hi], arr[i]
    return i
```

## Hoare Partition

```python
def partition_hoare(arr, lo, hi):
    pivot = arr[lo]           # 选第一个元素作 pivot
    i, j = lo - 1, hi + 1
    while True:
        i += 1
        while arr[i] < pivot: i += 1
        j -= 1
        while arr[j] > pivot: j -= 1
        if i >= j:
            return j
        arr[i], arr[j] = arr[j], arr[i]
```

> Hoare 比 Lomuto 交换次数少约 3 倍，实际更快。

## 随机化 Pivot

避免最坏情况（已排序数组）：

```python
import random

def quicksort_random(arr, lo, hi):
    if lo < hi:
        # 随机选 pivot，交换到末尾
        ri = random.randint(lo, hi)
        arr[ri], arr[hi] = arr[hi], arr[ri]
        p = partition_lomuto(arr, lo, hi)
        quicksort_random(arr, lo, p - 1)
        quicksort_random(arr, p + 1, hi)
```

## 尾递归优化

减少栈深度，保证 O(log n) 栈空间：

```python
def quicksort_tail(arr, lo, hi):
    while lo < hi:
        p = partition_lomuto(arr, lo, hi)
        # 递归较小的一边，循环处理较大的一边
        if p - lo < hi - p:
            quicksort_tail(arr, lo, p - 1)
            lo = p + 1        # 尾递归变循环
        else:
            quicksort_tail(arr, p + 1, hi)
            hi = p - 1
```

## 复杂度分析

| 情况 | 时间 | 说明 |
|------|------|------|
| 最好 | O(n log n) | 每次均匀分割 |
| 平均 | O(n log n) | 随机 pivot 期望值 |
| 最坏 | O(n²) | 每次只分出一个元素 |

**空间**：O(log n)（递归栈深度，最坏 O(n)）

## 关键要点

> [!important] 快排不稳定的
> Partition 过程中相等元素可能交换位置

> [!important] 实际最快
> 常数因子小，缓存友好，大多数标准库的主力排序

> [!important] 三数取中
> 取 lo/hi/mid 三者的中位数作 pivot，比随机化更稳定
