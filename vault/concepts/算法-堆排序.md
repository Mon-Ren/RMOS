# 堆排序

堆排序利用二叉堆的性质实现排序，O(n log n) 时间、O(1) 额外空间，但不稳定且缓存不友好。

## 二叉堆性质

**最大堆**：任意节点的值 ≥ 其子节点的值。
- 父节点：`parent(i) = (i-1) // 2`
- 左子节点：`left(i) = 2*i + 1`
- 右子节点：`right(i) = 2*i + 2`

用数组表示，无需指针，空间紧凑。

## 核心操作：Heapify

```python
def heapify(arr, n, i):
    """维护以 i 为根的最大堆性质"""
    largest = i
    l = 2 * i + 1
    r = 2 * i + 2

    if l < n and arr[l] > arr[largest]:
        largest = l
    if r < n and arr[r] > arr[largest]:
        largest = r

    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        heapify(arr, n, largest)  # 递归维护子树
```

## 堆排序算法

```python
def heap_sort(arr):
    n = len(arr)

    # 1. 建堆：从最后一个非叶节点开始 heapify
    #    最后一个非叶节点 = n//2 - 1
    for i in range(n // 2 - 1, -1, -1):
        heapify(arr, n, i)

    # 2. 排序：逐个将堆顶（最大值）放到末尾
    for i in range(n - 1, 0, -1):
        arr[0], arr[i] = arr[i], arr[0]  # 堆顶与末尾交换
        heapify(arr, i, 0)               # 缩小堆，重新维护
```

## 复杂度分析

### 建堆：O(n)（不是 O(n log n)）

```
高度 h 的节点数：
h=0: n/2 个叶节点，无需 heapify
h=1: n/4 个节点，最多 1 次交换
h=2: n/8 个节点，最多 2 次交换
...
总代价 = Σ (n/2^(h+1)) · h < n
```

### 排序：O(n log n)

- n-1 次 extract-max
- 每次 heapify 最坏 O(log n)
- 总计 O(n log n)

### 总复杂度

- **时间**：O(n) + O(n log n) = **O(n log n)**
- **空间**：**O(1)**（原地排序）
- **稳定性**：不稳定（父子节点交换破坏相对顺序）

## 为什么堆排序不如快排流行

| 方面 | 堆排序 | 快速排序 |
|------|--------|----------|
| 缓存友好度 | ❌ 跳跃访问 | ✅ 顺序访问 |
| 常数因子 | 较大 | 较小 |
| 最坏保证 | ✅ O(n log n) | ❌ O(n²) |

> 堆排序在缓存不友好的场景下实际速度通常慢于快排。

## 关键要点

> [!important] 建堆是 O(n)
> Floyd 建堆算法从下往上 heapify，总代价线性

> [!important] 堆排序不稳定
> 例：[5a, 5b, 3] → 排序后 5b 可能在 5a 前面

> [!important] O(1) 额外空间
> 不需要递归栈（可用迭代 heapify），是最省空间的 O(n log n) 排序
