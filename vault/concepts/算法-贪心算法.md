# 贪心算法

贪心算法在每一步选择中都采取当前状态下最优的决策，期望通过局部最优达到全局最优。

## 两个核心性质

1. **贪心选择性质**：全局最优解可以通过局部最优选择达到（无需回溯）
2. **最优子结构**：问题的最优解包含子问题的最优解

## 经典例题

### 区间调度（LeetCode 435）

给定区间集合，求最多能选多少不重叠区间：

```python
def max_non_overlapping(intervals):
    intervals.sort(key=lambda x: x[1])  # 按结束时间排序
    count = 0
    end = float('-inf')
    
    for start, finish in intervals:
        if start >= end:
            count += 1
            end = finish
    
    return count
# 时间 O(n log n)，空间 O(1)
```

### 找零钱（贪心适用的情况）

硬币面额为 1, 5, 10, 25 时，贪心有效：

```python
def coin_change_greedy(coins, amount):
    coins.sort(reverse=True)
    count = 0
    for coin in coins:
        count += amount // coin
        amount %= coin
    return count if amount == 0 else -1
```

> [!warning] 贪心不一定正确
> 面额为 `[1, 3, 4]` 时目标金额 6，贪心选 4+1+1=3 枚，实际最优为 3+3=2 枚。

### 霍夫曼编码

用贪心构建最优前缀编码树：每次合并频率最小的两个节点。

```python
import heapq

def huffman_codes(freq):
    heap = [[f, [ch, ""]] for ch, f in freq.items()]
    heapq.heapify(heap)
    
    while len(heap) > 1:
        lo = heapq.heappop(heap)
        hi = heapq.heappop(heap)
        for pair in lo[1:]:
            pair[1] = '0' + pair[1]
        for pair in hi[1:]:
            pair[1] = '1' + pair[1]
        heapq.heappush(heap, [lo[0]+hi[0]] + lo[1:] + hi[1:])
    
    return dict(heap[0][1:])
```

## 贪心 vs 动态规划

> [!info] 关键区别
> | 维度 | 贪心 | 动态规划 |
> |------|------|---------|
> | 决策方式 | 每步局部最优 | 考虑所有子问题 |
> | 是否回溯 | 否 | 是（或记忆化） |
> | 正确性 | 需证明 | 自然满足最优子结构 |
> | 效率 | 通常更高效 | 通常 O(n²) 或更高 |
> | 适用面 | 有贪心选择性质 | 有最优子结构即可 |
