# x86 特权级

x86 提供 4 个特权级（Ring 0-3），形成保护环。现代 OS 只用 Ring 0（内核）和 Ring 3（用户）。

## 四个 Ring

```
        Ring 0 (内核)
       ╱            ╲
    Ring 1          Ring 2  ← x86 定义但几乎不用
       ╲            ╱
        Ring 3 (用户)
```

## 三种特权级

| 缩写 | 全称 | 存储位置 | 含义 |
|------|------|----------|------|
| CPL | Current Privilege Level | CS 寄存器低 2 位 | 当前代码的特权级 |
| DPL | Descriptor Privilege Level | GDT/IDT 描述符 | 段/门的最低访问级别 |
| RPL | Requested Privilege Level | 段选择子低 2 位 | 请求者的声称级别 |

## 检查规则

### 数据段访问
```
有效特权级 EPL = max(CPL, RPL)
允许访问条件：EPL <= DPL
```

### 代码段跳转（call/jmp）
```
CPL <= DPL（不能跳到更低特权级的代码）
```

### 中断/异常
```
硬件中断：无条件从用户态进入内核（int 指令有额外 DPL 检查）
软件中断（int n）：CPL <= 门描述符 DPL
```

## 特权级切换

### 用户态 → 内核态（通过中断）
1. `int 0x80` 或硬件中断
2. CPU 检查 IDT 门的 DPL（int 指令时）
3. 从 TSS 获取内核栈 SS0:ESP0
4. 压入用户 SS、ESP、EFLAGS、CS、EIP
5. 加载内核 CS:EIP，CPL 变为 0

### 内核态 → 用户态（通过 iret）
1. `iret` 弹出 EIP、CS、EFLAGS、ESP、SS
2. CS 低 2 位 = 3，CPL 变为 3
3. 继续执行用户代码

## TSS（Task State Segment）

```c
struct taskstate {
  uint ss0;   // 内核栈段
  uint esp0;  // 内核栈指针
  // ... 其他字段
};
```

CPU 在 Ring 3→Ring 0 时，从 TSS 自动加载内核栈指针。每个 CPU 一个 TSS。

## 关键要点

> 特权级的核心是 **CPL/DPL 检查**——每次段访问、门调用、中断都会检查。硬件保证了 Ring 3 代码无法直接访问 Ring 0 的段。TSS 是跨越特权级的桥梁——CPU 需要知道"如果从用户态陷入内核，应该用哪个栈"。现代 OS 简化为两级，但 x86 的四级设计保留了历史兼容性。
