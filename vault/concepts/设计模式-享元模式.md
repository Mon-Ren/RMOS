# Flyweight 享元模式

通过共享来高效地支持大量细粒度对象，将对象的内在状态（可共享）与外在状态（上下文相关）分离。

## 意图与场景

当系统中有大量相似对象时，内存开销巨大。Flyweight 将共同部分提取为共享对象，减少内存使用。

**内在状态 vs 外在状态：**

| 类型 | 内在状态（Intrinsic） | 外在状态（Extrinsic） |
|------|----------------------|----------------------|
| 定义 | 对象固有的、可共享的部分 | 随上下文变化的、不可共享的部分 |
| 存储 | 由 Flyweight 对象持有 | 由客户端维护并传入 |
| 例子 | 字符的字形数据 | 字符在文档中的位置、颜色 |

**典型场景：**
- 文字处理器：每个字符对象共享字形数据，位置/颜色由客户端传入
- 游戏粒子系统：粒子类型共享纹理/颜色，位置/速度为外在状态
- GUI 框架：相同图标在多处显示，共享图标数据

## C++ 实现

### 游戏粒子系统

```cpp
#include <iostream>
#include <memory>
#include <string>
#include <unordered_map>
#include <vector>
#include <utility>

// --- Flyweight：可共享的粒子类型 ---
class ParticleType {
    std::string texture_;
    std::string color_;
    float size_;
    // 内在状态：所有同类型粒子共享

public:
    ParticleType(std::string texture, std::string color, float size)
        : texture_(std::move(texture))
        , color_(std::move(color))
        , size_(size) {}

    void render(float x, float y, float vx, float vy) const {
        std::cout << "  [" << color_ << " " << texture_
                  << " sz=" << size_
                  << " @ (" << x << "," << y << ")"
                  << " vel=(" << vx << "," << vy << ")]\n";
    }

    // 用于哈希比较
    bool operator==(const ParticleType& o) const {
        return texture_ == o.texture_ && color_ == o.color_ && size_ == o.size_;
    }
};

// --- Flyweight 工厂 ---
class ParticleTypeFactory {
    struct TypeHash {
        size_t operator()(const std::string& key) const {
            return std::hash<std::string>{}(key);
        }
    };

    std::unordered_map<std::string, std::shared_ptr<ParticleType>, TypeHash> types_;

public:
    std::shared_ptr<ParticleType> getType(
        const std::string& texture,
        const std::string& color,
        float size)
    {
        auto key = texture + ":" + color + ":" + std::to_string(size);
        auto it = types_.find(key);
        if (it != types_.end()) {
            return it->second;
        }
        auto type = std::make_shared<ParticleType>(texture, color, size);
        types_[key] = type;
        return type;
    }

    size_t typeCount() const { return types_.size(); }
};

// --- 含有外在状态的具体对象 ---
struct Particle {
    std::shared_ptr<ParticleType> type;  // 共享的内在状态
    float x, y;                          // 外在状态：位置
    float vx, vy;                        // 外在状态：速度

    void render() const {
        type->render(x, y, vx, vy);
    }
};

// --- 使用 ---
int main() {
    ParticleTypeFactory factory;

    // 1000 个粒子，但只有 3 种类型 → 内存节省巨大
    std::vector<Particle> particles;

    for (int i = 0; i < 1000; ++i) {
        auto type = factory.getType(
            i % 3 == 0 ? "fire" : (i % 3 == 1 ? "smoke" : "spark"),
            i % 3 == 0 ? "red"  : (i % 3 == 1 ? "gray" : "gold"),
            i % 3 == 0 ? 5.0f  : (i % 3 == 1 ? 8.0f  : 2.0f)
        );
        particles.push_back({
            type,
            static_cast<float>(i % 100),     // x
            static_cast<float>(i / 100),      // y
            static_cast<float>(i % 10 - 5),   // vx
            -2.0f                              // vy
        });
    }

    std::cout << "Particles: " << particles.size() << "\n";
    std::cout << "Unique types: " << factory.typeCount() << "\n";
    // 1000 个粒子，只有 3 种类型对象

    particles[0].render();
    particles[1].render();
    particles[2].render();
}
```

### 文字处理字符

```cpp
#include <string>
#include <unordered_map>
#include <memory>
#include <vector>

// Flyweight：字符字形
class Glyph {
    char character_;
    std::string font_;
    int fontSize_;
    // 内在状态：字形数据

public:
    Glyph(char c, std::string font, int size)
        : character_(c), font_(std::move(font)), fontSize_(size) {}

    void draw(int x, int y, const std::string& color) const {
        // 外在状态 x, y, color 由客户端传入
    }
};

class GlyphFactory {
    std::unordered_map<std::string, std::shared_ptr<Glyph>> cache_;

public:
    std::shared_ptr<Glyph> getGlyph(char c, const std::string& font, int size) {
        auto key = std::string(1, c) + ":" + font + ":" + std::to_string(size);
        if (!cache_.contains(key)) {
            cache_[key] = std::make_shared<Glyph>(c, font, size);
        }
        return cache_[key];
    }
};
```

## 优缺点

**优点：**
- 大幅减少内存使用（共享内在状态）
- 对客户端透明，使用方式不变
- 适合大量相似细粒度对象的场景

**缺点：**
- 需要区分内在/外在状态，增加设计复杂度
- 外在状态由客户端管理，增加客户端负担
- 代码变得更复杂，牺牲可读性换取空间

## 关键要点

> - Flyweight 的核心是「共享不变部分，外置变化部分」
> - 内在状态不可变（immutable），可安全共享
> - 工厂模式是 Flyweight 的天然搭档——确保同一内在状态只创建一个对象
> - 适用于对象数量巨大且大部分属性重复的场景
> - 现代 C++ 中 `std::shared_ptr` + `std::unordered_map` 是实现 Flyweight 的简洁方式
