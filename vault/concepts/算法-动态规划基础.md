# 动态规划基础

动态规划通过将复杂问题分解为重叠子问题，利用最优子结构性质，避免重复计算来高效求解。

## 两个核心条件

1. **最优子结构**：原问题的最优解包含子问题的最优解
2. **重叠子问题**：递归求解时，相同子问题被反复计算

## 两种实现方式

### 自顶向下（记忆化递归）

```python
def fib_memo(n, memo={}):
    if n <= 1:
        return n
    if n in memo:
        return memo[n]
    memo[n] = fib_memo(n-1, memo) + fib_memo(n-2, memo)
    return memo[n]
```

### 自底向上（填表法）

```python
def fib_dp(n):
    if n <= 1:
        return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]
```

## 解题框架

1. **状态定义**：确定 `dp[i]` 或 `dp[i][j]` 的含义
2. **状态转移方程**：找到状态之间的递推关系
3. **初始化**：确定 base case
4. **计算顺序**：确保计算某个状态时，其依赖的状态已算好
5. **返回结果**：通常为 `dp[n]` 或 `dp[m][n]`

## 爬楼梯问题（LeetCode 70）

每次可爬 1 或 2 阶，问到达第 n 阶的方法数：

```python
def climb_stairs(n):
    if n <= 2:
        return n
    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, a + b
    return b
# 时间 O(n)，空间 O(1)
```

> [!tip] 复杂度分析
> - **时间**：通常 O(n) ~ O(n²)，视状态维度而定
> - **空间**：O(n) ~ O(n²)，可压缩至 O(1)（滚动变量）

> [!warning] 常见误区
> - 混淆重叠子问题与分治法的子问题（分治子问题互不重叠）
> - 忘记初始化 base case
> - 状态定义不清晰导致转移方程写不出
