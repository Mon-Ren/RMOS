x86 提供丰富的内存寻址模式，从简单的立即数到复杂的 SIB 寻址，让编译器能高效地表达各种数据访问模式。

## 立即寻址 (Immediate)

操作数直接编码在指令中。

```asm
mov eax, 42          ; EAX = 42（立即数）
add eax, 100         ; EAX += 100
```

## 寄存器寻址 (Register)

操作数在寄存器中。

```asm
mov eax, ebx         ; EAX = EBX
add ecx, edx         ; ECX += EDX
```

## 直接寻址 (Direct / Absolute)

直接访问内存中的绝对地址。

```asm
mov eax, [0x12345678]          ; EAX = *(uint32_t*)0x12345678
mov BYTE PTR [my_var], 0x42    ; my_var = 0x42

section .data
my_var: dd 0
```

> [!warning] x86-64 注意
> 64 位模式下**没有** 64 位绝对地址直接寻址。数据访问默认使用 RIP 相对寻址。

## 间接寻址 (Register Indirect)

通过寄存器中存放的地址访问内存。

```asm
mov eax, [ebx]       ; EAX = *EBX（读取 EBX 指向的内存）
mov [ecx], edx       ; *ECX = EDX
```

## 基址 + 偏移 (Base + Displacement)

结构体成员访问的经典形式。

```asm
; 访问结构体成员
mov eax, [ebx + 12]     ; EAX = *(EBX + 12)

; 栈帧中访问参数和局部变量
mov eax, [ebp + 8]      ; 第一个参数（跳过返回地址和旧 EBP）
mov ebx, [ebp - 4]      ; 第一个局部变量
mov ecx, [ebp + 12]     ; 第二个参数
```

## 索引寻址 (Index × Scale + Base)

数组访问的经典形式。

```asm
; int array[ecx]
mov eax, [ebx + ecx*4]       ; EAX = array[ECX]（int 类型，4 字节）

; 双字节数组
movzx eax, WORD PTR [esi + edx*2]   ; EAX = array[EDX]（short 类型）

; 无基址寄存器
mov eax, [ecx*4 + array_base]  ; SIB 编码
```

## 完整 SIB 寻址 (Base + Index × Scale + Displacement)

最复杂的寻址形式，常用于二维数组或结构体数组。

```asm
; 二维数组访问：matrix[row][col]
; matrix[i][j] = base + i*row_size + j*element_size
mov eax, [ebx + ecx*4 + 8]    ; EAX = *(EBX + ECX*4 + 8)

; 等价于 C 代码：
; struct Entry { int a, b, data; };  // 12 字节
; eax = entries[i].data;             // data 在偏移 8
; lea eax, [ebx + ecx*8 + ecx*4 + 8]  ; 优化为 ECX*(8+4)+8
```

SIB 字节编码：

```
[base + index*scale + disp]
  base  = 寄存器编号
  index = 寄存器编号（ESP 表示无索引）
  scale = 00(×1), 01(×2), 10(×4), 11(×8)
```

## RIP 相对寻址 (x86-64)

64 位模式下数据访问的主要方式。地址 = RIP + 偏移。

```asm
; x86-64 下的全局变量访问
mov eax, [rel my_var]     ; EAX = *(RIP + offset)
lea rax, [rel my_var]     ; RAX = &my_var

; 等价写法（NASM 默认 rel）
mov eax, [my_var]         ; 自动生成 RIP 相对编码

section .data
my_var: dd 42
```

优势：代码可以加载到任意地址，无需重定位（PIC 代码的基础）。

## 段覆盖前缀

```asm
; 默认段规则：
; [EBP], [ESP] → 默认 SS 段
; 其他 [reg]   → 默认 DS 段
; 字符串目标   → 默认 ES 段（EDI）

mov eax, fs:[0x30]      ; Windows: 读取 PEB
mov rax, gs:[0x0]       ; Linux: 读取 per-CPU 数据
```

## 操作数大小指定

```asm
mov BYTE PTR [ebx], 0x42      ; 写入 1 字节
mov WORD PTR [ebx], 0x42      ; 写入 2 字节
mov DWORD PTR [ebx], 0x42     ; 写入 4 字节
mov QWORD PTR [rbx], 0x42     ; 写入 8 字节（x86-64）
```

> [!tip] 关键要点
> - 最简单的寻址：立即数、寄存器
> - `[reg]` 间接寻址是内存访问的基础
> - `[base+disp]` 用于结构体成员和栈帧访问
> - `[base+index*scale+disp]` 用于数组索引
> - x86-64 数据访问默认使用 RIP 相对寻址
> - scale 只能是 1/2/4/8，其他值需要额外乘法
