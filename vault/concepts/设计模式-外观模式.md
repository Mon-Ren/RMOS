# Facade 外观模式

为子系统中的一组接口提供一个统一的高层接口，使子系统更易于使用。

## 意图与场景

复杂系统由多个协作的类组成，客户端需要理解大量细节。Facade 提供一个简洁的入口，隐藏内部复杂性。

**典型场景：**
- 编译器前端：词法分析 → 语法分析 → 语义分析 → 中间代码
- 多媒体播放器：打开文件 → 解码 → 音频输出 → 视频渲染
- 数据库连接池：创建连接 → 认证 → 查询 → 归还连接
- 第三方库封装：提供面向业务的简洁 API

**Facade 不是替代子系统**——高级用户仍可直接使用子系统类。

## C++ 实现

### 多媒体播放器示例

```cpp
#include <iostream>
#include <memory>
#include <string>
#include <string_view>

// --- 子系统类 ---
class AudioDecoder {
public:
    void load(std::string_view file) {
        std::cout << "AudioDecoder: loading " << file << "\n";
    }
    void decode() {
        std::cout << "AudioDecoder: decoding audio stream\n";
    }
    void setVolume(int level) {
        std::cout << "AudioDecoder: volume = " << level << "\n";
    }
};

class VideoDecoder {
public:
    void load(std::string_view file) {
        std::cout << "VideoDecoder: loading " << file << "\n";
    }
    void decode() {
        std::cout << "VideoDecoder: decoding video stream\n";
    }
    void setResolution(int w, int h) {
        std::cout << "VideoDecoder: resolution = " << w << "x" << h << "\n";
    }
};

class AudioOutput {
public:
    void initialize() {
        std::cout << "AudioOutput: initialized\n";
    }
    void play() {
        std::cout << "AudioOutput: playing audio\n";
    }
};

class Display {
public:
    void initialize() {
        std::cout << "Display: initialized\n";
    }
    void render() {
        std::cout << "Display: rendering video\n";
    }
};

// --- Facade ---
class MediaPlayer {
    AudioDecoder audioDecoder_;
    VideoDecoder videoDecoder_;
    AudioOutput audioOutput_;
    Display display_;

public:
    // 一个方法代替客户端对 4 个子系统类的 6+ 次调用
    void play(std::string_view file) {
        std::cout << "MediaPlayer: starting playback of " << file << "\n\n";

        audioDecoder_.load(file);
        audioDecoder_.decode();
        audioDecoder_.setVolume(80);

        videoDecoder_.load(file);
        videoDecoder_.decode();
        videoDecoder_.setResolution(1920, 1080);

        audioOutput_.initialize();
        audioOutput_.play();

        display_.initialize();
        display_.render();

        std::cout << "\nMediaPlayer: playback started ✓\n";
    }

    void stop() {
        std::cout << "MediaPlayer: stopped\n";
    }

    // 仍允许高级用户访问子系统
    AudioDecoder& audio() { return audioDecoder_; }
    VideoDecoder& video() { return videoDecoder_; }
};

// --- 使用 ---
int main() {
    MediaPlayer player;
    player.play("movie.mp4");

    // 高级用法：直接操控子系统
    player.audio().setVolume(50);
    player.video().setResolution(3840, 2160);
}
```

### 编译器前端示例

```cpp
#include <iostream>
#include <string>

class Lexer {
public:
    std::string tokenize(const std::string& source) {
        std::cout << "Lexer: tokenizing...\n";
        return "TOKEN_STREAM";
    }
};

class Parser {
public:
    std::string parse(const std::string& tokens) {
        std::cout << "Parser: building AST...\n";
        return "AST";
    }
};

class SemanticAnalyzer {
public:
    std::string analyze(const std::string& ast) {
        std::cout << "SemanticAnalyzer: type checking...\n";
        return "TYPED_AST";
    }
};

class CodeGenerator {
public:
    std::string generate(const std::string& typedAst) {
        std::cout << "CodeGenerator: emitting IR...\n";
        return "LLVM_IR";
    }
};

// Facade：一条调用完成整个编译流水线
class Compiler {
    Lexer lexer_;
    Parser parser_;
    SemanticAnalyzer analyzer_;
    CodeGenerator codegen_;

public:
    std::string compile(const std::string& source) {
        auto tokens = lexer_.tokenize(source);
        auto ast = parser_.parse(tokens);
        auto typedAst = analyzer_.analyze(ast);
        return codegen_.generate(typedAst);
    }
};
```

## 优缺点

**优点：**
- 简化客户端代码，降低使用门槛
- 解耦客户端与子系统内部实现
- 不屏蔽高级用户的直接访问
- 符合最少知识原则（Law of Demeter）

**缺点：**
- Facade 可能成为"上帝对象"，承担过多职责
- 子系统变化时 Facade 可能需要同步更新
- 过度封装可能限制高级用户的灵活性

## 关键要点

> - Facade 是「简化的入口」——它不替代子系统，只是提供快捷方式
> - 遵循最少知识原则：客户端不需要了解子系统的所有类
> - Facade 对子系统是「非侵入式」的，子系统不依赖 Facade
> - 与 [[设计模式-适配器模式|Adapter]] 的区别：Adapter 转换接口使不兼容的类协同；Facade 为已有协作类提供统一入口
> - Facade 可以与 [[设计模式-单例模式|Singleton]] 结合，确保全局唯一入口

## 相关模式

- [[设计模式-适配器模式|Adapter 模式]] — Adapter 包装单个类转换接口；Facade 为整个子系统提供高层接口
- [[Mediator 中介者模式]] — Mediator 协调对象间的双向通信；Facade 为子系统提供单向简化入口
- [[设计模式-抽象工厂模式|Abstract Factory]] — Facade 可用工厂来创建子系统对象
