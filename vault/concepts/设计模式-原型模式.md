# 原型模式 (Prototype)

通过复制已有实例来创建新对象，而非通过构造函数从零构建。

## 意图与场景

**核心问题**：当对象的创建成本很高（深拷贝数据库记录、解析复杂配置）或类的类型在编译时未知时，克隆比重新构造更高效。

**典型场景**：
- 对象创建代价高昂（如从数据库加载的大对象）
- 运行时才知道具体类型（如从网络接收的多态对象）
- 需要大量相似但略有不同的对象（如游戏中的敌人实例）
- 撤销/重做机制（保存状态快照）

## C++ 实现

### `clone()` 虚函数模式

C++ 没有内置的 `clone` 关键字，通过虚函数 + 协变返回类型实现：

```cpp
#include <memory>
#include <string>
#include <iostream>
#include <vector>

class Shape {
public:
    virtual ~Shape() = default;
    virtual std::unique_ptr<Shape> clone() const = 0;
    virtual void describe() const = 0;

    // 深拷贝友好的移动语义
    Shape() = default;
    Shape(const Shape&) = default;
    Shape& operator=(const Shape&) = default;
    Shape(Shape&&) = default;
    Shape& operator=(Shape&&) = default;
};

class Circle : public Shape {
    double radius_;
    std::string color_;
public:
    Circle(double radius, std::string color)
        : radius_(radius), color_(std::move(color)) {}

    std::unique_ptr<Shape> clone() const override {
        return std::make_unique<Circle>(*this);  // 拷贝构造
    }

    void describe() const override {
        std::cout << "Circle(r=" << radius_
                  << ", color=" << color_ << ")\n";
    }

    void set_color(std::string c) { color_ = std::move(c); }
};

class Rectangle : public Shape {
    double width_, height_;
public:
    Rectangle(double w, double h) : width_(w), height_(h) {}

    std::unique_ptr<Shape> clone() const override {
        return std::make_unique<Rectangle>(*this);
    }

    void describe() const override {
        std::cout << "Rectangle(" << width_ << "x" << height_ << ")\n";
    }
};

// 使用：从已有实例创建变体
void demo() {
    auto base = std::make_unique<Circle>(10.0, "red");
    auto variant = base->clone();  // 克隆
    // 对克隆体修改，不影响原对象
    static_cast<Circle*>(variant.get())->set_color("blue");

    base->describe();    // Circle(r=10, color=red)
    variant->describe(); // Circle(r=10, color=blue)
}
```

### 原型注册表

用 map 管理"原型模板"，通过名称获取克隆体：

```cpp
#include <unordered_map>
#include <functional>
#include <stdexcept>

class PrototypeRegistry {
    std::unordered_map<std::string, std::unique_ptr<Shape>> prototypes_;
public:
    void register_prototype(std::string name, std::unique_ptr<Shape> proto) {
        prototypes_[std::move(name)] = std::move(proto);
    }

    std::unique_ptr<Shape> create(const std::string& name) const {
        auto it = prototypes_.find(name);
        if (it == prototypes_.end())
            throw std::invalid_argument("Unknown prototype: " + name);
        return it->second->clone();
    }
};

// 使用
// PrototypeRegistry registry;
// registry.register_prototype("red-circle",
//     std::make_unique<Circle>(10.0, "red"));
// auto shape = registry.create("red-circle");
```

### 深拷贝 vs 浅拷贝

```cpp
class Document {
    std::string title_;
    std::vector<int>* data_;  // 裸指针，需要深拷贝

public:
    // ❌ 浅拷贝：两个对象共享同一块 data_
    // std::unique_ptr<Shape> clone() const override {
    //     return std::make_unique<Document>(*this);
    // }

    // ✅ 深拷贝：显式复制指针指向的内容
    std::unique_ptr<Shape> clone() const {
        auto copy = std::make_unique<Document>();
        copy->title_ = title_;
        copy->data_ = new std::vector<int>(*data_);  // 深拷贝
        return copy;
    }

    // 💡 最佳实践：用 unique_ptr 或值语义避免手动深拷贝
};
```

> **最佳实践**：在 C++ 中，尽量用 `std::vector`、`std::string`、`std::unique_ptr` 等值语义容器，让编译器自动生成正确的拷贝。避免裸指针，深拷贝的 bug 是 C++ 中最常见的内存问题之一。

## 优缺点

### ✅ 优点
- 避免重复的昂贵初始化（克隆比从头构造快）
- 运行时动态创建对象（无需知道具体类）
- 减少子类数量（克隆替代工厂子类）
- 适合"大量相似对象"场景

### ❌ 缺点
- 深拷贝的循环引用问题
- `clone()` 需要在每个子类中实现，容易遗漏
- C++ 中没有语言级别的 clone 支持（不像 Java 的 `Cloneable`）

## 关键要点

> **原型模式在 C++ 中的本质是"深拷贝的多态封装"**。`clone()` 返回 `unique_ptr<Shape>`，使得调用者可以在不知道具体类型的情况下复制对象。

> **避免裸指针是关键**。如果类内部全部使用值语义（`std::string`、`std::vector`、`std::unique_ptr`），默认的拷贝构造就能正确工作，`clone()` 只需调用 `make_unique<T>(*this)`。

> **原型注册表**将"按名称获取对象"与"克隆"结合，是工厂模式的变体——用克隆替代构造。
