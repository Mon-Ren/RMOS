保护模式是 x86 从 80286 开始引入的现代操作模式，通过 GDT（全局描述符表）、特权级和门描述符实现内存保护和多任务隔离。

## 从实模式到保护模式

```
实模式 (Real Mode):
  - 16 位，1MB 地址空间
  - 地址 = 段基址 × 16 + 偏移
  - 无保护，任何代码可访问任何内存
  - 8086/8088 的原生模式

保护模式 (Protected Mode):
  - 32 位，4GB 地址空间
  - 段选择子 + GDT → 段描述符（基址 + 界限 + 权限）
  - 内存保护：越界访问触发异常
  - 特权级隔离：用户态不能执行内核指令
```

## 进入保护模式

```nasm
; 步骤 1: 关中断
cli

; 步骤 2: 设置 GDT
lgdt    [gdt_descriptor]

; 步骤 3: 设置 CR0.PE = 1
mov     eax, cr0
or      eax, 1
mov     cr0, eax

; 步骤 4: 远跳转刷新流水线（加载 CS）
jmp     0x08:protected_entry     ; 0x08 = 内核代码段选择子

[bits 32]
protected_entry:
    ; 现在在 32 位保护模式
    mov     ax, 0x10             ; 内核数据段选择子
    mov     ds, ax
    mov     es, ax
    mov     ss, ax
    ; ...
```

## GDT — 全局描述符表

GDT 是一个数组，每个条目（段描述符）8 字节，描述一个内存段的属性：

```
段描述符结构 (64 位):
┌──────────┬──────┬──────┬──────┬─────────┬──────┐
│ Base     │Flags │Access│Limit │ Base    │Limit │
│ [31:24]  │      │      │[19:16│ [23:16] │[15:0]│
│          │      │      │]     │         │      │
└──────────┴──────┴──────┴──────┴─────────┴──────┘
  高 32 位                           低 32 位
```

### 访问字节

```
  7   6     5     4     3     2     1     0
┌───┬─────┬─────┬─────┬─────┬─────┬─────┬───┐
│ P │ DPL │ S   │ E   │ DC  │ RW  │ A   │   │
└───┴─────┴─────┴─────┴─────┴─────┴─────┴───┘
P   = Present (1=有效)
DPL = Descriptor Privilege Level (0=内核, 3=用户)
S   = System (0=系统段, 1=代码/数据段)
E   = Executable (1=代码, 0=数据)
DC  = Direction/Conforming
RW  = Readable(代码)/Writable(数据)
A   = Accessed
```

### 构建 GDT

```nasm
section .data
gdt_start:
    ; 0x00: 空描述符（必须）
    dq 0

    ; 0x08: 内核代码段
    ; Base=0, Limit=4GB, 可执行, 可读, DPL=0
    dw 0xFFFF        ; Limit [15:0]
    dw 0x0000        ; Base [15:0]
    db 0x00          ; Base [23:16]
    db 10011010b     ; Access: P=1, DPL=00, S=1, E=1, RW=1
    db 11001111b     ; Flags: G=1, D=1 + Limit [19:16]
    db 0x00          ; Base [31:24]

    ; 0x10: 内核数据段
    ; Base=0, Limit=4GB, 可写, DPL=0
    dw 0xFFFF
    dw 0x0000
    db 0x00
    db 10010010b     ; Access: P=1, DPL=00, S=1, E=0, RW=1
    db 11001111b
    db 0x00

    ; 0x18: 用户代码段 (DPL=3)
    dw 0xFFFF
    dw 0x0000
    db 0x00
    db 11111010b     ; Access: P=1, DPL=11, S=1, E=1, RW=1
    db 11001111b
    db 0x00

    ; 0x20: 用户数据段 (DPL=3)
    dw 0xFFFF
    dw 0x0000
    db 0x00
    db 11110010b     ; Access: P=1, DPL=11, S=1, E=0, RW=1
    db 11001111b
    db 0x00

gdt_end:

gdt_descriptor:
    dw gdt_end - gdt_start - 1   ; GDT 大小
    dd gdt_start                  ; GDT 基址
```

## 段选择子

```
段选择子 (16 位):
┌────────────────────┬───┬─────┐
│  Index (13 位)      │ TI│ RPL │
└────────────────────┴───┴─────┘
  Index: GDT 中的索引号
  TI:    0=GDT, 1=LDT
  RPL:   Requested Privilege Level (0-3)
```

```nasm
; 常用选择子
; 0x00 = 空 (Index=0, TI=0, RPL=0)
; 0x08 = 内核代码 (Index=1, TI=0, RPL=0)  → GDT[1]
; 0x10 = 内核数据 (Index=2, TI=0, RPL=0)  → GDT[2]
; 0x1B = 用户代码 (Index=3, TI=0, RPL=3)  → GDT[3]
; 0x23 = 用户数据 (Index=4, TI=0, RPL=3)  → GDT[4]

; 加载段选择子
mov     ax, 0x10
mov     ds, ax            ; 数据段
mov     es, ax
mov     ss, ax            ; 栈段
```

## 特权级 Ring 0-3

```
Ring 0 — 内核态：完全访问，可执行所有指令
Ring 1 — （很少使用）
Ring 2 — （很少使用）
Ring 3 — 用户态：受限访问，执行特权指令触发 #GP

典型权限：
  CR3 修改      → 只有 Ring 0
  CLI/STI       → 只有 Ring 0 (除非 IOPL 允许)
  IN/OUT        → 由 IOPL 控制
  HLT           → 只有 Ring 0
  段寄存器修改  → Ring 0 (CS 不能直接修改)
  普通内存访问  → 受段/页权限控制
```

### 特权级检查规则

```
CPL (Current Privilege Level)  = CS 寄存器低 2 位
DPL (Descriptor Privilege Level) = 段描述符中的 DPL 字段
RPL (Requested Privilege Level)  = 段选择子低 2 位

数据段访问：max(CPL, RPL) ≤ DPL
代码段跳转：CPL = DPL（同级）或 通过门

Ring 3 → Ring 0：只能通过系统调用/中断/调用门
Ring 0 → Ring 3：通过 IRET 或 SYSEXIT
```

## 门描述符

门描述符允许低特权级代码"穿过门"调用高特权级代码：

```nasm
; 调用门描述符
; 允许 Ring 3 代码调用指定的 Ring 0 代码
dw offset_low         ; 入口偏移 [15:0]
dw 0x08               ; 目标代码段选择子
db 0x00               ; 参数计数（通过栈传参的个数）
db 11101100b          ; P=1, DPL=3, 类型=1100(386 调用门)
dw offset_high        ; 入口偏移 [31:16]

; 中断门描述符
; 用于 IDT 表项
dw handler_low
dw 0x08               ; 内核代码段
db 0x00
db 11101110b          ; P=1, DPL=0, 类型=1110(386 中断门)
dw handler_high

; 陷阱门（与中断门区别：不自动清除 IF）
db 11101111b          ; 类型=1111(386 陷阱门)
```

## TSS — 任务状态段

```nasm
; TSS 在保护模式中用于任务切换和特权级栈切换
; 当从 Ring 3 → Ring 0 时，CPU 从 TSS 读取 Ring 0 的 SS:ESP

tss_entry:
    dw 0              ; 保留
    dw ring0_esp      ; SS0:ESP0 — Ring 0 栈
    dw 0x10           ; SS0 (内核数据段)
    dd ring0_esp
    ; ... 其他字段 ...
```

## 关键要点

> **保护模式 = 内存保护 + 特权级隔离**：用户程序不能直接访问内核内存，也不能执行特权指令。

> **GDT 是核心数据结构**：每个段描述符定义一个内存区域的基址、界限和权限。平坦模型下只用 4 个描述符（内核代码/数据、用户代码/数据）。

> **特权级 0（内核）和 3（用户）**：现代 OS 只用这两个。Ring 3 → Ring 0 只能通过系统调用门或中断门。

> **进入保护模式三步**：设置 GDT → CR0.PE=1 → 远跳转刷新 CS。顺序不能错。
