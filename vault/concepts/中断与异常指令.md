中断指令允许软件主动触发中断，是调试断点、系统调用和异常处理的入口机制。

## INT n — 软件中断

```asm
int 0x80             ; Linux 32 位系统调用
int 0x21             ; DOS 系统调用
int 0x10             ; BIOS 视频服务
int 0x13             ; BIOS 磁盘服务
```

### INT 的执行过程

```
1. PUSH EFLAGS
2. PUSH CS
3. PUSH EIP（返回地址）
4. 从 IDT[n] 获取中断处理程序地址
5. 跳转到处理程序
6. IF = 0, TF = 0（禁止中断和单步）
```

```asm
; 完整等价操作
pushfd                  ; 保存 EFLAGS
push cs                 ; 保存 CS
push eip                ; 保存返回地址
; 根据 IDT[n] 设置 CS:EIP
; 清除 IF 和 TF
jmp interrupt_handler
```

## INT3 — 断点中断

调试器设置断点的机制。INT3 的特殊之处在于它只有 **1 字节**（0xCC）。

```asm
int3                 ; 触发断点异常 (#BP)，调试器捕获
; 编码：0xCC（单字节！）

; vs
int 3                ; 编码：CD 03（两字节）
; 效果相同但编码不同
```

### 调试器的工作原理

```asm
; 原始代码：
mov eax, [var]       ; 地址：0x08048400

; 调试器设置断点后：
0xCC                 ; 地址：0x08048400（替换了第一个字节）
nop                  ; 剩余字节

; 当执行到 INT3 时：
; 1. CPU 触发 #BP 异常
; 2. 调试器捕获，显示断点位置
; 3. 恢复原始字节（MOV）
; 4. 单步执行
; 5. 重新插入 0xCC
; 6. 继续执行
```

```asm
; 程序中手动插入断点
int3                 ; 调试器会在此处暂停
; 在 Debug 构建中常见
```

## INTO — 溢出检查

检查 OF 标志，如果为 1 则触发中断 4。

```asm
; INTO 检查 OF
add eax, ebx
into                 ; 如果 OF=1（有符号溢出），触发 INT 4
; 如果 OF=0，什么都不做

; 等价于：
jo overflow_handler  ; 用条件跳转替代
```

> [!warning] INTO 已废弃
> - 在 x86-64 模式下 INTO 指令已移除
> - 使用 JO 替代 INTO 进行溢出检查
> - 仅在 32 位代码中可能见到

## IRET — 从中断返回

中断处理程序的返回指令，恢复中断前的状态。

```asm
iret                 ; 32 位模式
iretq                ; 64 位模式

; IRET 的等价操作：
pop eip              ; 恢复返回地址
pop cs               ; 恢复代码段
popfd                ; 恢复 EFLAGS（包括 IF, TF 等）
```

### 中断处理程序的典型结构

```asm
; 中断处理程序
my_interrupt_handler:
    ; 保存上下文
    pusha                  ; 保存所有通用寄存器
    push ds                ; 保存段寄存器
    push es
    
    ; 设置内核数据段
    mov ax, KERNEL_DS
    mov ds, ax
    mov es, ax
    
    ; 处理中断
    call handle_interrupt
    
    ; 恢复上下文
    pop es
    pop ds
    popa
    iret                   ; 返回被中断的代码
```

## BOUND — 数组边界检查

检查数组索引是否在边界内，越界则触发异常 5。

```asm
; BOUND r32, m64
; m64 存放 [low_bound, high_bound]

section .data
bounds: dd 0, 9          ; 索引范围 [0, 9]

section .text
mov eax, [index]
bound eax, [bounds]      ; 检查 0 <= EAX <= 9
                         ; 越界则触发 #BR (Bound Range Exceeded)
mov eax, [array + eax*4] ; 安全访问
```

> [!warning] BOUND 性能问题
> - BOUND 指令非常慢（比手动 CMP+JAE 慢得多）
> - 编译器从不生成 BOUND 指令
> - 仅在 x86-32 中可用，x86-64 已移除

## 异常类型速查

| 向量 | 名称 | 类型 | 触发条件 |
|------|------|------|---------|
| 0 | #DE | Fault | DIV/IDIV 除零 |
| 1 | #DB | Fault/Trap | 调试异常（INT3、单步） |
| 2 | NMI | Interrupt | 不可屏蔽中断 |
| 3 | #BP | Trap | INT3 断点 |
| 4 | #OF | Trap | INTO 溢出 |
| 5 | #BR | Fault | BOUND 越界 |
| 6 | #UD | Fault | 无效操作码 |
| 7 | #NM | Fault | 设备不可用 |
| 8 | #DF | Abort | 双重错误 |
| 13 | #GP | Fault | 一般保护错误 |
| 14 | #PF | Fault | 页错误 |

> [!tip] 关键要点
> - INT n 触发 IDT 中第 n 个中断处理程序
> - INT3 是 1 字节断点指令（0xCC），调试器的核心工具
> - IRET 恢复 CS:EIP 和 EFLAGS，用于中断返回
> - INTO 在 x86-64 中已移除，用 JO 替代
> - BOUND 性能极差，编译器不使用
