# 设计模式概述

设计模式是面向对象软件设计中常见问题的可复用解决方案——不是现成代码，而是经过验证的设计思路。

## GoF 23 种模式分类

1994 年《Design Patterns》一书提出 23 种经典模式，分为三大类：

### 创建型模式 (Creational) — 5 种
关注对象创建机制，将实例化过程抽象化。

| 模式 | 核心思路 |
|------|----------|
| [[设计模式-单例模式\|Singleton]] | 确保全局唯一实例 |
| [[设计模式-工厂方法模式\|Factory Method]] | 子类决定实例化哪个类 |
| [[设计模式-抽象工厂模式\|Abstract Factory]] | 创建一族相关对象 |
| [[设计模式-建造者模式\|Builder]] | 分步构建复杂对象 |
| [[设计模式-原型模式\|Prototype]] | 通过克隆创建对象 |

### 结构型模式 (Structural) — 7 种
关注类和对象的组合方式。

- Adapter、Bridge、Composite、Decorator、Facade、Flyweight、Proxy

### 行为型模式 (Behavioral) — 11 种
关注对象间的职责分配和通信。

- Chain of Responsibility、Command、Interpreter、Iterator、Mediator、Memento、Observer、State、Strategy、Template Method、Visitor

## 为什么需要设计模式

> 设计模式的价值不在于"用上了模式"，而在于它帮你实现了**正确的解耦**。

- **解耦 (Decoupling)**：降低模块间的依赖，修改一处不影响全局
- **复用 (Reusability)**：经过验证的方案，减少重复造轮子
- **可维护 (Maintainability)**：统一的 vocabulary，团队沟通成本降低
- **可扩展 (Extensibility)**：对扩展开放，对修改关闭

## SOLID 原则

SOLID 是面向对象设计的五大基本原则，也是理解设计模式的基础。

### S — 单一职责原则 (Single Responsibility Principle)
> 一个类应该只有一个引起它变化的原因。

```cpp
// ❌ 职责混杂
class User {
    void save_to_db();      // 持久化职责
    void send_email();      // 通知职责
    void validate();        // 业务校验
};

// ✅ 职责分离
class User { /* 纯数据 */ };
class UserRepository { void save(const User&); };
class EmailService { void send(const User&, std::string_view msg); };
```

### O — 开闭原则 (Open-Closed Principle)
> 对扩展开放，对修改关闭。通过多态和模板实现新增行为，而非修改已有代码。

### L — 里氏替换原则 (Liskov Substitution Principle)
> 子类对象必须能替换父类对象而不破坏程序正确性。

### I — 接口隔离原则 (Interface Segregation Principle)
> 客户端不应被迫依赖它不使用的接口。用多个细粒度接口替代臃肿接口。

### D — 依赖倒置原则 (Dependency Inversion Principle)
> 高层模块不应依赖低层模块，二者都应依赖抽象。

```cpp
// ✅ 依赖抽象而非具体实现
class ILogger {
public:
    virtual ~ILogger() = default;
    virtual void log(std::string_view msg) = 0;
};

class Application {
    std::unique_ptr<ILogger> logger_;  // 依赖抽象
public:
    explicit Application(std::unique_ptr<ILogger> logger)
        : logger_(std::move(logger)) {}
};
```

## C++ 中模式的特殊考量

C++ 语言特性为设计模式提供了独特的实现手段：

### 模板与编译时多态
```cpp
// CRTP (Curiously Recurring Template Pattern) — 零开销多态
template <typename Derived>
class Shape {
public:
    void draw() const {
        static_cast<const Derived*>(this)->draw_impl();
    }
};

class Circle : public Shape<Circle> {
public:
    void draw_impl() const { /* ... */ }
};
```

### RAII 与资源管理
C++ 的 RAII 天然将资源生命周期与对象绑定，许多模式（如 Proxy、Decorator）可以利用智能指针自动管理内存。

### 移动语义 (Move Semantics)
```cpp
// Builder 模式利用移动语义避免拷贝
class QueryBuilder {
public:
    QueryBuilder& where(std::string clause) & {
        conditions_.push_back(std::move(clause));
        return *this;
    }
    // 支持从临时对象移动
    Query build() && { return Query(std::move(conditions_)); }
};
```

### `constexpr` 与编译期计算
某些模式可以在编译期完成，如有限状态机、策略模式的策略选择。
