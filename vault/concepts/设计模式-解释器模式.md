# Interpreter 解释器模式

为语言定义文法表示，并提供解释器来解释该语言中的句子。

## 意图与场景

- **核心意图**：给定一种语言，定义其文法的表示，并定义一个解释器来解释语言中的句子
- **典型场景**：
  - 正则表达式解析与匹配
  - SQL 查询解析
  - 数学表达式求值
  - 配置文件 DSL
  - 规则引擎（业务规则表达式）
  - 机器人指令解析

> **注意**：解释器模式适合简单文法。复杂语言应使用 parser generator（ANTLR、Flex/Bison）。

## C++ 实现

### AST 节点 + 递归求值（简单算术表达式）

```cpp
#include <memory>
#include <string>
#include <unordered_map>
#include <stdexcept>
#include <iostream>

// 上下文：变量环境
using Context = std::unordered_map<std::string, double>;

// 抽象表达式
class Expression {
public:
    virtual ~Expression() = default;
    virtual double interpret(const Context& ctx) const = 0;
    virtual std::string toString() const = 0;
};

// 终结符：数字字面量
class NumberExpr : public Expression {
    double value_;
public:
    explicit NumberExpr(double v) : value_(v) {}
    double interpret(const Context&) const override { return value_; }
    std::string toString() const override { return std::to_string(value_); }
};

// 终结符：变量引用
class VariableExpr : public Expression {
    std::string name_;
public:
    explicit VariableExpr(std::string name) : name_(std::move(name)) {}
    double interpret(const Context& ctx) const override {
        auto it = ctx.find(name_);
        if (it == ctx.end()) throw std::runtime_error("Undefined variable: " + name_);
        return it->second;
    }
    std::string toString() const override { return name_; }
};

// 非终结符：二元运算
class BinaryExpr : public Expression {
    std::unique_ptr<Expression> left_, right_;
    char op_;
public:
    BinaryExpr(char op, std::unique_ptr<Expression> l, std::unique_ptr<Expression> r)
        : op_(op), left_(std::move(l)), right_(std::move(r)) {}

    double interpret(const Context& ctx) const override {
        auto l = left_->interpret(ctx);
        auto r = right_->interpret(ctx);
        switch (op_) {
            case '+': return l + r;
            case '-': return l - r;
            case '*': return l * r;
            case '/': if (r == 0) throw std::runtime_error("Division by zero");
                      return l / r;
            default: throw std::runtime_error("Unknown operator");
        }
    }
    std::string toString() const override {
        return "(" + left_->toString() + " " + op_ + " " + right_->toString() + ")";
    }
};
```

### 使用示例

```cpp
int main() {
    // 表达式: (x + 2) * (y - 1)
    auto expr = std::make_unique<BinaryExpr>('*',
        std::make_unique<BinaryExpr>('+',
            std::make_unique<VariableExpr>("x"),
            std::make_unique<NumberExpr>(2)),
        std::make_unique<BinaryExpr>('-',
            std::make_unique<VariableExpr>("y"),
            std::make_unique<NumberExpr>(1))
    );

    Context ctx = {{"x", 3}, {"y", 5}};
    std::cout << expr->toString() << " = " << expr->interpret(ctx) << "\n";
    // (x + 2) * (y - 1) = 20
}
```

### 简化：函数式风格

```cpp
#include <functional>
using EvalFn = std::function<double(const Context&)>;

EvalFn num(double v) { return [v](const auto&) { return v; }; }
EvalFn var(std::string name) {
    return [n = std::move(name)](const Context& ctx) { return ctx.at(n); };
}
EvalFn add(EvalFn a, EvalFn b) {
    return [a = std::move(a), b = std::move(b)](const Context& ctx) {
        return a(ctx) + b(ctx);
    };
}
// (x + 2) * (y - 1)
// mul(add(var("x"), num(2)), sub(var("y"), num(1)))
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 易于扩展文法（新表达式节点） | 复杂文法会产生大量类 |
| 文法的每条规则对应一个类 | 解释器模式不擅长处理复杂文法 |
| 实现文法较为直观 | 执行效率通常不高（递归解释） |
| 易于改变和扩展解释方式 | 难以维护大量规则 |

## 关键要点

> **核心思想**：将每条文法规则映射为一个类，通过递归组合形成 AST（抽象语法树），再递归解释求值。

> **终结符 vs 非终结符**：
> - 终结符表达式：叶子节点（字面量、变量），不可再分解
> - 非终结符表达式：内部节点（运算、组合），包含子表达式

> **适用判断**：如果你的"语言"只有简单表达式（< 20 条规则），解释器模式合适。更复杂的场景应该考虑 parser generator 或 [[设计模式-访问者模式|Visitor 模式]] 做 AST 处理。

> **解析阶段**：解释器模式只管"解释 AST"，不管"如何构建 AST"。构建 AST 通常需要单独的 Parser/Lexer。

## 相关模式

- [[设计模式-组合模式|Composite 模式]]：AST 本质是组合结构（树），表达式节点是 Composite 的典型应用
- [[设计模式-访问者模式|Visitor 模式]]：可以用 Visitor 在不修改表达式类的前提下定义新操作（如打印、优化、代码生成）
- [[设计模式-迭代器模式|Iterator 模式]]：可以用 Iterator 遍历 AST 节点
