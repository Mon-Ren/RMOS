# 建造者模式 (Builder)

将复杂对象的构建过程与表示分离，使同样的构建过程可以创建不同的表示。

## 意图与场景

**核心问题**：对象构造参数过多、构造步骤有顺序依赖、或需要多种表示形式时，构造逻辑会变得混乱。

**典型场景**：
- SQL 查询构建（SELECT、FROM、WHERE、ORDER BY 有顺序）
- HTTP 请求构建（URL、Header、Body、Method）
- 复杂配置对象（游戏关卡、文档模板）
- 对象有大量可选参数（替代 telescoping constructor）

## C++ 实现

### 链式 Builder（Fluent API）

现代 C++ 中最常见的实现——返回 `*this` 的链式调用：

```cpp
#include <string>
#include <vector>
#include <optional>
#include <iostream>

class HttpRequest {
public:
    struct Header {
        std::string key;
        std::string value;
    };

    // Getters
    const std::string& method() const { return method_; }
    const std::string& url() const { return url_; }
    const std::vector<Header>& headers() const { return headers_; }
    const std::optional<std::string>& body() const { return body_; }

    void print() const {
        std::cout << method_ << " " << url_ << "\n";
        for (const auto& h : headers_)
            std::cout << h.key << ": " << h.value << "\n";
        if (body_)
            std::cout << "\n" << *body_ << "\n";
    }

private:
    friend class HttpRequestBuilder;  // 只有 Builder 能构造
    HttpRequest() = default;

    std::string method_;
    std::string url_;
    std::vector<Header> headers_;
    std::optional<std::string> body_;
};

class HttpRequestBuilder {
    HttpRequest request_;
public:
    HttpRequestBuilder& method(std::string m) {
        request_.method_ = std::move(m);
        return *this;
    }

    HttpRequestBuilder& url(std::string u) {
        request_.url_ = std::move(u);
        return *this;
    }

    HttpRequestBuilder& header(std::string key, std::string value) {
        request_.headers_.push_back({std::move(key), std::move(value)});
        return *this;
    }

    HttpRequestBuilder& body(std::string b) {
        request_.body_ = std::move(b);
        return *this;
    }

    [[nodiscard]] HttpRequest build() && {
        return std::move(request_);
    }
};

// 使用
auto request = HttpRequestBuilder{}
    .method("POST")
    .url("https://api.example.com/users")
    .header("Content-Type", "application/json")
    .header("Authorization", "Bearer xxx")
    .body(R"({"name": "Alice"})")
    .build();  // 注意 &&，只能从右值 build

request.print();
```

### Director + Builder（经典 GoF 结构）

Director 封装固定的构建步骤序列，Builder 负责具体实现：

```cpp
// 抽象 Builder
class QueryBuilder {
public:
    virtual ~QueryBuilder() = default;
    virtual void add_select(std::string_view field) = 0;
    virtual void add_from(std::string_view table) = 0;
    virtual void add_where(std::string_view condition) = 0;
    virtual std::string get_result() = 0;
};

// 具体 Builder：SQL
class SqlQueryBuilder : public QueryBuilder {
    std::string select_, from_, where_;
public:
    void add_select(std::string_view field) override {
        if (!select_.empty()) select_ += ", ";
        select_ += field;
    }
    void add_from(std::string_view table) override {
        from_ = table;
    }
    void add_where(std::string_view condition) override {
        if (!where_.empty()) where_ += " AND ";
        where_ += condition;
    }
    std::string get_result() override {
        std::string sql = "SELECT " + select_ + " FROM " + from_;
        if (!where_.empty()) sql += " WHERE " + where_;
        return sql;
    }
};

// Director
class QueryDirector {
public:
    std::string build_user_query(QueryBuilder& builder) {
        builder.add_select("id");
        builder.add_select("name");
        builder.add_select("email");
        builder.add_from("users");
        builder.add_where("active = true");
        return builder.get_result();
    }
};

// 使用
// SqlQueryBuilder builder;
// QueryDirector director;
// auto sql = director.build_user_query(builder);
// → "SELECT id, name, email FROM users WHERE active = true"
```

## 优缺点

### ✅ 优点
- **分步构建**：控制复杂对象的创建过程
- **代码可读**：Fluent API 大幅提升可读性
- **复用构建逻辑**：Director 可复用相同的步骤生成不同表示
- **不可变对象友好**：构建完成后返回 `const` 对象

### ❌ 缺点
- 增加类数量（每个产品一个 Builder）
- Builder 本身可能变得复杂
- 如果对象简单，Builder 是过度设计

## 关键要点

> **Builder 的本质是"分步 + 链式"**。当构造函数参数超过 4-5 个时，就该考虑 Builder 了。现代 C++ 中，Fluent API（返回 `*this`）是最实用的变体。

> **`build()` 返回右值引用 `&&`** 可防止 Builder 被意外复用，保证对象只被 build 一次。这是一个小但重要的设计细节。

> **Director 是可选的**。只有当构建步骤有固定的、可复用的序列时才需要 Director。大多数场景用 Fluent Builder 就够了。
