# Iterator 迭代器模式

提供一种方法顺序访问聚合对象中的各个元素，而不暴露其内部表示。

## 意图与场景

- **核心意图**：将遍历逻辑从集合中分离，统一遍历接口
- **典型场景**：
  - STL 容器遍历（`std::vector`, `std::map` 等）
  - 数据库结果集遍历
  - 文件系统目录遍历
  - 树/图结构的多种遍历策略
  - 无限序列（惰性生成）

## C++ 实现

### STL 迭代器体系

C++ 将迭代器模式发挥到了极致，STL 的核心就是迭代器抽象：

```cpp
#include <vector>
#include <list>
#include <algorithm>
#include <iostream>
#include <iterator>

int main() {
    std::vector<int> v = {1, 2, 3, 4, 5};

    // 传统迭代器用法
    for (auto it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << " ";
    }

    // range-based for（底层调用 begin()/end()）
    for (const auto& x : v) {
        std::cout << x << " ";
    }

    // STL 算法与迭代器配合
    auto it = std::find(v.begin(), v.end(), 3);
    if (it != v.end()) std::cout << "Found: " << *it;
}
```

### 迭代器类别（Iterator Categories）

| 类别 | 能力 | 代表容器 |
|------|------|----------|
| **Input** | 只读，单遍 | `std::istream_iterator` |
| **Output** | 只写，单遍 | `std::ostream_iterator` |
| **Forward** | 读写，多遍，单向 | `std::forward_list` |
| **Bidirectional** | 可 `--` | `std::list`, `std::map` |
| **RandomAccess** | 随机跳转 `+n`, `-n`, `[]` | `std::vector`, `std::deque` |
| **Contiguous** (C++20) | 内存连续 | `std::vector`, `std::array` |

### 自定义迭代器示例

```cpp
#include <iterator>

// 一个简单的范围迭代器（生成 [start, end) 的整数序列）
class Range {
    int start_, end_;
public:
    class Iterator {
        int current_;
    public:
        // 迭代器 traits（C++17 风格）
        using iterator_category = std::forward_iterator_tag;
        using value_type = int;
        using difference_type = std::ptrdiff_t;
        using pointer = int*;
        using reference = int&;

        explicit Iterator(int v) : current_(v) {}

        int operator*() const { return current_; }

        Iterator& operator++() {
            ++current_;
            return *this;
        }
        Iterator operator++(int) {
            auto tmp = *this;
            ++current_;
            return tmp;
        }

        bool operator==(const Iterator& other) const { return current_ == other.current_; }
        bool operator!=(const Iterator& other) const { return current_ != other.current_; }
    };

    Range(int start, int end) : start_(start), end_(end) {}

    Iterator begin() const { return Iterator(start_); }
    Iterator end()   const { return Iterator(end_); }
};

// 使用: for (int i : Range(1, 10)) std::cout << i << " ";
```

### C++20 范围（Ranges）

```cpp
#include <ranges>
#include <vector>

int main() {
    auto even = [](int x) { return x % 2 == 0; };
    auto square = [](int x) { return x * x; };

    // 惰性管道
    for (int x : std::views::iota(1, 100)
               | std::views::filter(even)
               | std::views::transform(square)
               | std::views::take(5)) {
        // 4, 16, 36, 64, 100
    }
}
```

### range-based for 的底层机制

```cpp
// for (auto& x : container) { ... }
// 编译器展开为：
{
    auto&& __range = container;
    auto __begin = __range.begin(); // 或 begin(__range)（ADL）
    auto __end = __range.end();
    for (; __begin != __end; ++__begin) {
        auto& x = *__begin;
        // ...
    }
}
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 统一接口遍历不同集合 | 对某些数据结构，迭代器可能低效 |
| 隐藏集合内部实现 | 多种迭代器类别增加学习成本 |
| 支持多种遍历策略 | 并发修改集合可能导致迭代器失效 |
| 可组合（STL 算法库） | 自定义迭代器的 traits 样板代码较多 |

## 关键要点

> **核心思想**：遍历是独立于集合的行为。迭代器将"在哪里"（位置）和"怎么遍历"（策略）从"遍历什么"（数据）中分离。

> **C++ 独特之处**：STL 迭代器是值语义（不持有集合引用），比经典 GoF 基于多态的迭代器更高效。编译时多态（模板）替代运行时多态（虚函数）。

> **失效问题**：在遍历过程中修改容器（特别是 `std::vector` 增删元素）会导致迭代器失效——这是 C++ 中常见的 bug 来源。

> **C++20 趋势**：Ranges 和 Concepts 正在取代原始迭代器对，让代码更简洁：
> ```cpp
> // 旧：std::sort(v.begin(), v.end());
> // 新：std::ranges::sort(v);
> ```
