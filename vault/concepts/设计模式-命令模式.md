# Command 命令模式

将请求封装为独立对象，从而可以用不同的请求参数化客户端，支持排队、日志、撤销/重做。

## 意图与场景

- **核心意图**：将"做什么"和"谁做"解耦，把操作封装为对象
- **典型场景**：
  - 文本编辑器（撤销/重做操作栈）
  - 任务队列 / 线程池任务分发
  - 宏录制与回放
  - 事务系统
  - GUI 按钮绑定不同操作

## C++ 实现

### Command 接口 + 具体命令 + Invoker

```cpp
#include <memory>
#include <vector>
#include <string>
#include <iostream>
#include <stack>

// Command 接口
class Command {
public:
    virtual ~Command() = default;
    virtual void execute() = 0;
    virtual void undo() = 0;
    [[nodiscard]] virtual std::string describe() const = 0;
};

// Receiver：文本编辑器
class TextEditor {
    std::string text_;
public:
    void insert(const std::string& s) { text_ += s; }
    void erase(size_t n) {
        if (n > text_.size()) n = text_.size();
        text_.resize(text_.size() - n);
    }
    [[nodiscard]] const std::string& text() const { return text_; }
};

// 具体命令：插入文本
class InsertCommand : public Command {
    TextEditor& editor_;
    std::string text_;
public:
    InsertCommand(TextEditor& editor, std::string text)
        : editor_(editor), text_(std::move(text)) {}

    void execute() override { editor_.insert(text_); }
    void undo() override { editor_.erase(text_.size()); }
    [[nodiscard]] std::string describe() const override {
        return "Insert: " + text_;
    }
};

// 具体命令：删除最后 N 个字符
class EraseCommand : public Command {
    TextEditor& editor_;
    size_t count_;
    std::string saved_; // 用于 undo
public:
    EraseCommand(TextEditor& editor, size_t count)
        : editor_(editor), count_(count) {}

    void execute() override {
        const auto& t = editor_.text();
        saved_ = t.substr(t.size() >= count_ ? t.size() - count_ : 0);
        editor_.erase(count_);
    }
    void undo() override { editor_.insert(saved_); }
    [[nodiscard]] std::string describe() const override {
        return "Erase: " + std::to_string(count_) + " chars";
    }
};

// Invoker：支持撤销/重做
class CommandManager {
    std::stack<std::unique_ptr<Command>> undoStack_;
    std::stack<std::unique_ptr<Command>> redoStack_;
public:
    void execute(std::unique_ptr<Command> cmd) {
        cmd->execute();
        undoStack_.push(std::move(cmd));
        // 新操作清空 redo 栈
        while (!redoStack_.empty()) redoStack_.pop();
    }

    void undo() {
        if (undoStack_.empty()) return;
        auto& cmd = undoStack_.top();
        cmd->undo();
        redoStack_.push(std::move(cmd));
        undoStack_.pop();
    }

    void redo() {
        if (redoStack_.empty()) return;
        auto& cmd = redoStack_.top();
        cmd->execute();
        undoStack_.push(std::move(cmd));
        redoStack_.pop();
    }

    [[nodiscard]] bool canUndo() const { return !undoStack_.empty(); }
    [[nodiscard]] bool canRedo() const { return !redoStack_.empty(); }
};
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 开闭原则：新增命令无需修改已有代码 | 每个操作都需要一个 Command 子类（类膨胀） |
| 支持撤销/重做 | 命令栈可能消耗大量内存 |
| 支持命令队列、延迟执行 | 复杂命令的 undo 实现困难 |
| 支持宏命令（组合命令） | 引入间接层，增加复杂度 |

## 关键要点

> **核心思想**：把"操作"本身变成一等公民（对象），而非简单的方法调用。一旦操作是对象，就可以存储、传递、排队、撤销。

> **撤销/重做关键**：每个 Command 必须保存足够的逆操作状态。`execute()` 做正向操作，`undo()` 做逆向操作。

> **现代 C++ 趋势**：简单场景用 `std::function` 替代 Command 接口；需要 undo/redo 或序列化时，还是需要完整 Command 类。

> **宏命令**：可以实现 CompositeCommand，内部持有多个子命令，批量执行/撤销——这本质上是 [[设计模式-组合模式|Composite 模式]] 的应用。
