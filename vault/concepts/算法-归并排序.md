# 归并排序

归并排序是经典的分治排序算法，保证 O(n log n) 时间复杂度，且是稳定排序。

## 核心思想

```
1. 分解：将数组对半分成两个子数组
2. 求解：递归排序两个子数组
3. 合并：将两个有序子数组合并为一个有序数组
```

## 递归实现（自顶向下）

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:    # <= 保证稳定性
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

## 迭代实现（自底向上）

```python
def merge_sort_iterative(arr):
    n = len(arr)
    width = 1
    while width < n:
        for i in range(0, n, 2 * width):
            left = arr[i:i+width]
            right = arr[i+width:i+2*width]
            arr[i:i+len(left)+len(right)] = merge(left, right)
        width *= 2
    return arr
```

**优点**：不需要递归栈，O(1) 额外栈空间（仍需 O(n) 合并空间）。

## 复杂度分析

递推式：T(n) = 2T(n/2) + O(n)

```
        O(n)          ← 合并全部元素
       /    \
    O(n/2)  O(n/2)    ← 两半各自排序
    /   \    /   \
  ...   ... ...  ...  ← log₂(n) 层
```

- **时间**：每层 O(n)，共 log n 层 → **O(n log n)**（最好、最坏、平均均如此）
- **空间**：O(n) 合并临时数组 + O(log n) 递归栈

## 原地归并排序

牺牲简洁性换取 O(1) 额外空间（实际很少用，复杂且常数因子大）：

```python
def merge_inplace(arr, l, m, r):
    """原地合并，时间 O(n·log n) 但常数大"""
    left = arr[l:m+1]
    right = arr[m+1:r+1]
    i = j = 0
    k = l
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            arr[k] = left[i]; i += 1
        else:
            arr[k] = right[j]; j += 1
        k += 1
    while i < len(left):
        arr[k] = left[i]; i += 1; k += 1
    while j < len(right):
        arr[k] = right[j]; j += 1; k += 1
```

## 外部排序

当数据量超过内存容量时：
1. 将文件分成能装入内存的块
2. 每个块用归并排序排好序，写回磁盘
3. 用 k 路归并将所有有序块合并

归并排序天然适合外部排序，因为合并操作是顺序读写。

## 关键要点

> [!important] 稳定的 O(n log n) 排序
> 需要保证相等元素相对顺序不变时，归并排序是首选

> [!important] 链表排序的最佳选择
> 归并排序天然适合链表，不需要随机访问，合并操作高效

> [!important] 空间代价
> 标准实现需要 O(n) 额外空间，这是 vs 快排的主要劣势
