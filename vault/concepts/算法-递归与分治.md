# 递归与分治

递归是函数调用自身来解决更小子问题的方法；分治是将大问题分解为多个子问题、分别求解再合并的算法范式。

## 递归三要素

1. **基准情况 (Base Case)**：递归终止条件，防止无限调用
2. **递归调用**：函数调用自身
3. **问题缩小**：每次递归必须让问题规模严格减小，趋向基准情况

```python
# 阶乘：递归三要素演示
def factorial(n):
    if n <= 1:          # 1. 基准情况
        return 1
    return n * factorial(n - 1)  # 2+3. 递归调用 + 问题缩小
```

## 递归树分析

以斐波那契为例，递归树可以直观展示重复子问题：

```
            fib(5)
           /      \
        fib(4)    fib(3)
       /    \      /    \
    fib(3) fib(2) fib(2) fib(1)
    /   \
 fib(2) fib(1)
```

- 树高 = O(n)
- 节点总数 = O(2ⁿ)（大量重复计算）
- 优化方案：[[记忆化]] 或 [[动态规划]]

## 分治范式

```
分治 = 分解 (Divide) → 求解 (Conquer) → 合并 (Combine)
```

### 经典案例：归并排序

```python
def merge_sort(arr):
    if len(arr) <= 1:                # 基准
        return arr
    mid = len(arr) // 2              # 分解
    left = merge_sort(arr[:mid])     # 求解
    right = merge_sort(arr[mid:])    # 求解
    return merge(left, right)        # 合并

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])
    result.extend(right[j:])
    return result
```

**复杂度分析**：
- 分解：O(1)（计算 mid）
- 合并：O(n)
- 递推：T(n) = 2T(n/2) + O(n)
- 由主定理：**T(n) = O(n log n)**
- 空间：O(n)（合并时的临时数组）

### 大整数乘法 (Karatsuba)

将两个 n 位整数各分为两半：

```
X = A·10^(n/2) + B
Y = C·10^(n/2) + D
XY = AC·10^n + (AD+BC)·10^(n/2) + BD

Karatsuba 技巧：AD+BC = (A+B)(C+D) - AC - BD
只需 3 次乘法而非 4 次
T(n) = 3T(n/2) + O(n) → O(n^1.585)
```

## 递归 vs 迭代

| 特性 | 递归 | 迭代 |
|------|------|------|
| 代码简洁性 | 高 | 低 |
| 栈空间 | O(递归深度) | O(1) |
| 尾递归优化 | 可被编译器优化 | — |
| 可读性 | 自然（树、分治） | 线性流程 |

## 关键要点

> [!important] 每个递归必须有基准情况
> 缺少 base case → 栈溢出 (StackOverflow)

> [!important] 递归树法求复杂度
> 画出递归树，每层代价之和即为总复杂度

> [!important] 尾递归优化
> 递归调用是函数最后一步时，编译器可复用当前栈帧，避免 O(n) 栈空间
