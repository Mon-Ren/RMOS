# Proxy 代理模式

为另一个对象提供一个替身或占位符，以控制对该对象的访问。

## 意图与场景

Proxy 与被代理对象实现同一接口，客户端无感知。代理在访问真实对象前后增加控制逻辑。

**常见代理类型：**

| 类型 | 用途 | 示例 |
|------|------|------|
| 虚代理（Virtual Proxy） | 延迟加载（Lazy Init） | 大图片/视频的缩略图 |
| 保护代理（Protection Proxy） | 访问控制 | 权限检查 |
| 智能引用代理 | 引用计数、线程安全 | `std::shared_ptr` |
| 远程代理 | 本地代表远程对象 | RPC 客户端桩 |
| 日志代理 | 记录访问日志 | 调试、审计 |

## C++ 实现

### 虚代理 — 图片懒加载

```cpp
#include <iostream>
#include <memory>
#include <string>

// --- Subject 接口 ---
class Image {
public:
    virtual ~Image() = default;
    virtual void display() = 0;
    virtual std::string info() = 0;
};

// --- Real Subject：加载代价高昂 ---
class HighResImage : public Image {
    std::string filename_;
    // 模拟大量数据
    std::string pixelData_;

    void loadFromDisk() {
        std::cout << "HighResImage: loading '" << filename_
                  << "' from disk (expensive!)\n";
        pixelData_ = "2560x1440 raw pixels...";
    }

public:
    explicit HighResImage(std::string filename)
        : filename_(std::move(filename)) {
        // 构造时不加载
    }

    void display() override {
        if (pixelData_.empty()) loadFromDisk();
        std::cout << "HighResImage: displaying " << filename_ << "\n";
    }

    std::string info() override {
        if (pixelData_.empty()) loadFromDisk();
        return "HighResImage: " + filename_ + " [" + pixelData_ + "]";
    }
};

// --- Proxy：延迟加载 ---
class ImageProxy : public Image {
    std::string filename_;
    std::unique_ptr<Image> realImage_;  // 首次访问时才创建

public:
    explicit ImageProxy(std::string filename)
        : filename_(std::move(filename)) {}

    void display() override {
        ensureLoaded();
        realImage_->display();
    }

    std::string info() override {
        ensureLoaded();
        return realImage_->info();
    }

private:
    void ensureLoaded() {
        if (!realImage_) {
            std::cout << "Proxy: lazy-loading '" << filename_ << "'\n";
            realImage_ = std::make_unique<HighResImage>(filename_);
        }
    }
};

// --- 使用：客户端完全无感知 ---
int main() {
    std::unique_ptr<Image> img = std::make_unique<ImageProxy>("photo.png");

    std::cout << "Image created, but not loaded yet.\n\n";

    // 首次访问触发加载
    img->display();
    std::cout << "\n";

    // 第二次访问直接使用已加载的对象
    img->display();
}
```

### 保护代理 — 权限控制

```cpp
#include <iostream>
#include <memory>
#include <string>

enum class Role { Guest, User, Admin };

class Document {
public:
    virtual ~Document() = default;
    virtual void read() = 0;
    virtual void edit(const std::string& content) = 0;
    virtual void remove() = 0;
};

class SecureDocument : public Document {
    std::string content_;
public:
    void read() override {
        std::cout << "Reading: " << content_ << "\n";
    }
    void edit(const std::string& c) override {
        content_ = c;
        std::cout << "Edited document\n";
    }
    void remove() override {
        std::cout << "Document deleted\n";
    }
};

class DocumentProxy : public Document {
    std::unique_ptr<Document> doc_;
    Role role_;

public:
    DocumentProxy(std::unique_ptr<Document> doc, Role role)
        : doc_(std::move(doc)), role_(role) {}

    void read() override {
        // 所有角色都可读
        doc_->read();
    }

    void edit(const std::string& content) override {
        if (role_ < Role::User) {
            std::cout << "Access denied: edit requires User role\n";
            return;
        }
        doc_->edit(content);
    }

    void remove() override {
        if (role_ < Role::Admin) {
            std::cout << "Access denied: remove requires Admin role\n";
            return;
        }
        doc_->remove();
    }
};
```

### 智能引用代理 — shared_ptr 就是 Proxy

`std::shared_ptr` 本身就是 Proxy 模式的一种实现：

```cpp
#include <memory>
#include <iostream>

class Resource {
public:
    Resource() { std::cout << "Resource acquired\n"; }
    ~Resource() { std::cout << "Resource released\n"; }
    void use() { std::cout << "Resource used\n"; }
};

int main() {
    // shared_ptr 是 Proxy：控制引用计数、自动释放
    std::shared_ptr<Resource> r1 = std::make_shared<Resource>();
    {
        auto r2 = r1;  // 引用计数 +1
        r2->use();     // 通过代理访问真实对象
        std::cout << "refs: " << r1.use_count() << "\n";  // 2
    }  // r2 销毁，引用计数 -1
    std::cout << "refs: " << r1.use_count() << "\n";  // 1
}  // 引用计数归零，Resource 释放
```

## 优缺点

**优点：**
- 客户端无感知：代理与真实对象接口一致
- 延迟加载减少初始开销
- 访问控制集中在代理层
- 单一职责：真实对象专注业务逻辑

**缺点：**
- 增加一次间接调用的开销
- 代理类数量可能较多
- 响应时间可能因延迟加载变慢（首次访问）

## 关键要点

> - Proxy 与真实对象实现同一接口，客户端无从区分
> - 虚代理（Lazy Proxy）是最常用的形式——首次访问时才创建真实对象
> - `std::shared_ptr` 是 C++ 中最常见的智能引用代理
> - `std::unique_ptr` 的自定义 Deleter 也是代理思想的体现
> - 与 [[设计模式-装饰器模式|Decorator]] 的区别：Proxy 控制访问；Decorator 增强功能

## 相关模式
