# 信号量

信号量是由 Dijkstra 提出的同步原语，通过一个整数计数器和两个原子操作（P/V）来控制对共享资源的访问。

## 基本操作

```
P(sem):  // wait / down
  while(sem <= 0)  // 无资源则阻塞
    ;               // （实际实现中进程进入等待队列）
  sem--;

V(sem):  // signal / up
  sem++;            // 释放资源
  // 唤醒一个等待的进程
```

P 来自荷兰语 Proberen（尝试），V 来自 Verhogen（增加）。

## 两种类型

### 二值信号量（Binary Semaphore）

- 值只能是 0 或 1
- 等价于互斥锁（但没有所有权概念）
- 任何进程都可以 V 释放，不限于持有者

### 计数信号量（Counting Semaphore）

- 值可以是任意非负整数
- 用于控制 N 个同类资源的并发访问
- 例：数据库连接池、线程池

## 经典应用

### 生产者-消费者问题

```c
semaphore empty = N;  // 空槽位数
semaphore full = 0;   // 满槽位数
semaphore mutex = 1;  // 缓冲区互斥

// 生产者
P(empty);
P(mutex);
  // 放入数据
V(mutex);
V(full);

// 消费者
P(full);
P(mutex);
  // 取出数据
V(mutex);
V(empty);
```

### 读者-写者问题

```c
semaphore rw_mutex = 1;  // 读写互斥
semaphore mutex = 1;      // 保护 reader_count
int reader_count = 0;

// 读者
P(mutex);
  reader_count++;
  if(reader_count == 1)
    P(rw_mutex);  // 第一个读者锁住写者
V(mutex);
// ... 读 ...
P(mutex);
  reader_count--;
  if(reader_count == 0)
    V(rw_mutex);  // 最后一个读者释放
V(mutex);

// 写者
P(rw_mutex);
// ... 写 ...
V(rw_mutex);
```

## 信号量 vs 互斥锁

| 特性 | 信号量 | 互斥锁 |
|------|--------|--------|
| 计数 | 0 ~ N | 0 ~ 1 |
| 所有权 | 无（任何进程可 V） | 有（谁 lock 谁 unlock） |
| 用途 | 同步 + 互斥 | 仅互斥 |
| 典型场景 | 生产者消费者、资源池 | 临界区保护 |

## xv6 中的实现

xv6 **没有显式的信号量**，但 sleep/wakeup 机制提供了类似功能：

```c
// xv6 的 sleep — 等价于 P
sleep(chan, &lk);

// xv6 的 wakeup — 等价于 V
wakeup(chan);
```

`sleep` 释放传入的锁、将进程标记为 SLEEPING、切换上下文。
`wakeup` 遍历 ptable，唤醒 chan 匹配的进程。

Linux 内核提供了完整的信号量实现（`semaphore` 和 `rw_semaphore`）。

## 关键要点

> 信号量是最通用的同步原语——互斥锁、条件变量、事件通知都可以用信号量实现。理解 P/V 操作是理解并发编程的第一步。
