# State 状态模式

允许对象在其内部状态改变时改变它的行为，对象看起来似乎修改了它的类。

## 意图与场景

- **核心意图**：将状态相关的行为封装到独立的类中，避免庞大的 `if-else` / `switch` 分支
- **典型场景**：
  - TCP 连接状态（Closed → Listen → Established → Closed）
  - 游戏角色状态（Idle → Running → Jumping → Falling）
  - 订单状态（待支付 → 已支付 → 已发货 → 已完成）
  - 文档编辑状态（草稿 → 审核 → 发布）
  - 网络协议状态机

## C++ 实现

### State 接口 + Context 持有当前状态

```cpp
#include <memory>
#include <string>
#include <iostream>

// 前置声明
class Context;

// 抽象状态
class State {
public:
    virtual ~State() = default;
    virtual void enter() {}
    virtual void exit() {}
    virtual void handlePlay(Context& ctx)  = 0;
    virtual void handlePause(Context& ctx) = 0;
    virtual void handleStop(Context& ctx)  = 0;
    virtual std::string name() const = 0;
};

// Context：媒体播放器
class Context {
    std::unique_ptr<State> state_;
public:
    explicit Context(std::unique_ptr<State> initial)
        : state_(std::move(initial)) {
        state_->enter();
    }

    void transitionTo(std::unique_ptr<State> newState) {
        std::cout << "状态转换: " << state_->name()
                  << " → " << newState->name() << "\n";
        state_->exit();
        state_ = std::move(newState);
        state_->enter();
    }

    void play()  { state_->handlePlay(*this); }
    void pause() { state_->handlePause(*this); }
    void stop()  { state_->handleStop(*this); }

    [[nodiscard]] const std::string& currentState() const { return state_->name(); }
};

// 具体状态：就绪（停止）
class StoppedState : public State {
public:
    void handlePlay(Context& ctx) override;
    void handlePause(Context& ctx) override {
        std::cout << "[Stopped] 无操作（已停止）\n";
    }
    void handleStop(Context& ctx) override {
        std::cout << "[Stopped] 已经是停止状态\n";
    }
    std::string name() const override { return "Stopped"; }
};

// 具体状态：播放中
class PlayingState : public State {
public:
    void handlePlay(Context& ctx) override {
        std::cout << "[Playing] 已经在播放\n";
    }
    void handlePause(Context& ctx) override;
    void handleStop(Context& ctx) override;
    std::string name() const override { return "Playing"; }
};

// 具体状态：暂停
class PausedState : public State {
public:
    void handlePlay(Context& ctx) override;
    void handlePause(Context& ctx) override {
        std::cout << "[Paused] 已经是暂停状态\n";
    }
    void handleStop(Context& ctx) override;
    std::string name() const override { return "Paused"; }
};

// 状态转换实现（需要完整类型定义）
void StoppedState::handlePlay(Context& ctx) {
    std::cout << "[Stopped] 开始播放\n";
    ctx.transitionTo(std::make_unique<PlayingState>());
}
void PlayingState::handlePause(Context& ctx) {
    std::cout << "[Playing] 暂停\n";
    ctx.transitionTo(std::make_unique<PausedState>());
}
void PlayingState::handleStop(Context& ctx) {
    std::cout << "[Playing] 停止\n";
    ctx.transitionTo(std::make_unique<StoppedState>());
}
void PausedState::handlePlay(Context& ctx) {
    std::cout << "[Paused] 恢复播放\n";
    ctx.transitionTo(std::make_unique<PlayingState>());
}
void PausedState::handleStop(Context& ctx) {
    std::cout << "[Paused] 停止\n";
    ctx.transitionTo(std::make_unique<StoppedState>());
}
```

### 有限状态机（FSM）表格驱动实现

```cpp
#include <unordered_map>
#include <functional>

enum class State { Idle, Running, Jumping, Falling };
enum class Event { Run, Jump, Land };

struct StateKey {
    State state;
    Event event;
    bool operator==(const StateKey&) const = default;
};

struct StateKeyHash {
    size_t operator()(const StateKey& k) const {
        return std::hash<int>()(static_cast<int>(k.state)) ^
               (std::hash<int>()(static_cast<int>(k.event)) << 1);
    }
};

class FSM {
    State current_ = State::Idle;
    std::unordered_map<StateKey, State, StateKeyHash> transitions_;
    std::unordered_map<State, std::function<void()>, std::hash<int>> onEnter_;
public:
    FSM() {
        transitions_[{State::Idle,    Event::Run}]  = State::Running;
        transitions_[{State::Running, Event::Jump}]  = State::Jumping;
        transitions_[{State::Jumping, Event::Land}]  = State::Running;
        transitions_[{State::Running, Event::Land}]  = State::Idle;
    }

    void onEnter(State s, std::function<void()> fn) {
        onEnter_[s] = std::move(fn);
    }

    void handle(Event e) {
        auto key = StateKey{current_, e};
        if (auto it = transitions_.find(key); it != transitions_.end()) {
            current_ = it->second;
            if (auto fn = onEnter_.find(current_); fn != onEnter_.end()) {
                fn->second();
            }
        }
    }

    [[nodiscard]] State state() const { return current_; }
};
```

## 优缺点

| 优点 | 缺点 |
|------|------|
| 消除庞大的条件分支 | 状态类数量增多 |
| 符合开闭原则（新增状态不动已有代码） | 状态转换逻辑分散在各状态类中 |
| 状态行为集中管理 | 引入间接层，增加复杂度 |
| 状态转换显式且可追踪 | 状态类之间可能有循环依赖 |

## 关键要点

> **核心思想**：将"状态"从一个变量提升为一个对象。每个状态对象知道自己的行为，以及在什么条件下转换到什么状态。

> **State vs Strategy**：两者结构相似，但意图不同——[[设计模式-策略模式|Strategy]] 是客户端主动选择算法，State 是对象内部状态自动驱动行为切换。

> **FSM 表格驱动**：对于状态转换逻辑复杂但固定的场景（如协议栈），用表格驱动比多态更清晰、更易验证。

> **进入/退出行为**：好的状态实现应该有 `enter()` 和 `exit()` 钩子，在状态转换时执行清理/初始化逻辑。

## 相关模式

- [[设计模式-策略模式|Strategy 模式]]：结构类似，但 State 允许状态对象自己控制转换，Strategy 由外部选择
- [[设计模式-命令模式|Command 模式]]：状态转换可以由 Command 触发
- [[设计模式-享元模式|Flyweight 模式]]：如果状态对象无状态（只含行为），可以用 Flyweight 共享
