# 分页与分段对比

分页和分段是两种内存管理方式。现代 OS 几乎都用分页（或段页式），分段在 x86 上更多是历史遗留。

## 分段（Segmentation）

### 原理

内存按逻辑段划分：代码段、数据段、栈段。每个段有基址（base）和界限（limit）。

```
逻辑地址 = 段选择子 : 段内偏移

物理地址 = 段描述符.base + 偏移
          （偏移 < 段描述符.limit，否则异常）
```

### x86 实模式

```
物理地址 = 段寄存器 × 16 + 偏移

例：CS=0x1000, IP=0x0050 → 物理地址 0x10050
```

### x86 保护模式

段描述符存储在 GDT/LDT 中，包含：基址、界限、权限、类型。

### 问题

- **外部碎片**：段大小不一，分配后内存出现空洞
- **管理复杂**：需要紧凑（compaction）操作
- **不利于共享**：段是不同大小的连续区域

## 分页（Paging）

### 原理

虚拟内存和物理内存都分成固定大小的**页**（通常 4KB）。

```
虚拟地址 = 页号 : 页内偏移

物理地址 = 页表[页号].物理页号 : 页内偏移
```

### 优势

- **无外部碎片**：所有块大小相同
- **细粒度保护**：每页独立设置权限
- **易于共享**：两个进程的页表指向同一物理页
- **支持 swap**：以页为单位换入换出

### 问题

- **内部碎片**：最后一个页可能未填满（平均浪费半页）
- **页表开销**：需要存储页表（用多级页表缓解）

## 对比表

| 特性 | 分段 | 分页 |
|------|------|------|
| 粒度 | 可变大小（逻辑单位） | 固定大小（通常 4KB） |
| 碎片类型 | 外部碎片 | 内部碎片 |
| 地址空间 | 二维（段号+偏移） | 一维（线性） |
| 程序员感知 | 可见（段寄存器） | 透明 |
| 共享 | 较难 | 容易（共享页） |
| 现代使用 | x86 兼容模式 | 所有现代 OS |

## x86 的段页式

x86 32位保护模式同时使用分段和分页：

```
逻辑地址 → [分段] → 线性地址 → [分页] → 物理地址
         段描述符            页表
```

Linux 的做法：将所有段的 base 设为 0，limit 设为最大，**让分段"透明化"**，实际上只用分页。

```c
// Linux x86 的 GDT 设置（简化）
// CS, DS, SS 全部 base=0, limit=4GB
// 分段变成恒等映射，分页做所有工作
```

x86-64（长模式）中，大部分段基址强制为 0（除了 FS/GS），分段基本被废弃。

## 关键要点

> 分段是逻辑划分的思路，分页是物理划分的思路。现代 OS 通过将分段设为恒等映射，实质上只用分页管理内存。理解两者区别是理解 x86 内存架构的关键。
