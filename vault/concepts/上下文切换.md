# 上下文切换

上下文切换是 OS 在进程间切换的核心机制。xv6 的实现极其精简。

## swtch.S — 内核态上下文切换

```asm
# void swtch(struct context **old, struct context *new);

swtch:
  movl 4(%esp), %eax    # old: 保存当前 context 的指针的指针
  movl 8(%esp), %edx    # new: 要切换到的 context

  # 保存旧的 callee-saved 寄存器
  pushl %ebp
  pushl %ebx
  pushl %esi
  pushl %edi

  # 切换栈指针
  movl %esp, (%eax)     # *old = 当前栈顶（保存旧 context）
  movl %edx, %esp       # 切换到新栈

  # 恢复新的 callee-saved 寄存器
  popl %edi
  popl %esi
  popl %ebx
  popl %ebp
  ret
```

## struct context — 保存的寄存器

```c
struct context {
  uint edi;
  uint esi;
  uint ebx;
  uint ebp;
  uint eip;   // ret 时自动恢复
};
```

只保存 5 个寄存器！因为 x86 约定：
- **Callee-saved**（被调用者保存）：edi, esi, ebx, ebp → swtch 负责保存
- **Caller-saved**（调用者保存）：eax, ecx, edx → 编译器在调用前已压栈
- **段寄存器**：内核态切换时不变，无需保存

## 切换流程

```
进程A (RUNNING)
  │
  │ swtch(&cpu->scheduler, p->context)
  │   1. 保存 A 的 context 到 A 的内核栈
  │   2. 切换 esp → cpu->scheduler 的栈
  │   3. ret → 回到 scheduler()
  ▼
scheduler()
  │ 选择进程B
  │ swtch(&cpu->scheduler, B->context)
  │   1. 保存 scheduler 的 context
  │   2. 切换 esp → B 的内核栈
  │   3. ret → 回到 B 上次离开的地方
  ▼
进程B (RUNNING)
```

## 三个切换路径

| 场景 | 触发 | old | new |
|------|------|-----|-----|
| 调度器切换到进程 | scheduler() | c->scheduler | p->context |
| 进程让出 CPU | yield() → sched() | p->context | c->scheduler |
| 进程 sleep | sleep() → sched() | p->context | c->scheduler |
| 进程退出 | exit() → sched() | p->context | c->scheduler |

## forkret — 新进程的入口

新进程创建时，`allocproc()` 在内核栈底部设置：
```
栈底 → trapret (返回地址)
     → context {edi=0, esi=0, ebx=0, ebp=0, eip=forkret}
```

调度器 `swtch` 切换到新进程时，ret 跳转到 `forkret` → 设置好 trapframe → `trapret` → 返回用户态。

## 关键要点

> xv6 的上下文切换极其精简——只需保存/恢复 5 个 callee-saved 寄存器。核心技巧是**交换栈指针**：进程的整个执行状态（包括局部变量、返回地址）都保存在它的内核栈上，切换 esp 就等于切换了整个世界。这也是为什么每个进程都需要独立的内核栈。
