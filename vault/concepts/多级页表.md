# 多级页表

x86-32 用二级页表，x86-64 用**四级页表**。级数增加是为了支撑更大的地址空间。

## 为什么需要多级？

### x86-32 二级页表的问题

- 4GB 虚拟空间，4KB 页 → 1M 个 PTE
- 页表本身占 4MB（1024 个页表 × 4KB）
- 对于小进程（几 MB），页表本身就占了一大块

### x86-64 四级页表

48 位虚拟地址，分成 5 部分：

```
47      39 38     30 29     21 20     12 11        0
┌─────────┬─────────┬─────────┬─────────┬───────────┐
│ PML4(9) │ PDPT(9) │ PD (9)  │ PT (9)  │ Offset(12)│
└─────────┴─────────┴─────────┴─────────┴───────────┘
```

## 四级结构

```
CR3 ──→ PML4 (Page Map Level 4)
           └─→ PDPT (Page Directory Pointer Table)
                  └─→ PD (Page Directory)
                        └─→ PT (Page Table)
                              └─→ 物理页
```

每一级 512 个条目 × 8 字节 = 4KB = 刚好一页。

## 优势

### 稀疏地址空间

```
小进程只使用 2MB 虚拟空间：
PML4: 1 页
PDPT: 1 页
PD:   1 页
PT:   1 页
总计: 4 页 = 16KB

vs x86-32 二级: 最少 2 页 = 8KB，但不够灵活
```

只有实际使用的地址范围才需要分配页表页。未使用的部分完全不占内存。

### 大页支持

| 级别 | 页大小 |
|------|--------|
| PD (PS=1) | 2MB |
| PDPT (PS=1) | 1GB |
| PT | 4KB |

2MB 大页减少 TLB 未命中，适合数据库、科学计算。

## Linux 的实现

Linux 4 级页表有自定义名称：

```
PGD → P4D → PUD → PMD → PTE
```

中间加了 P4D 层以支持 5 级页表（57 位虚拟地址）。

## 关键要点

> 多级页表的核心是**按需分配**——只有被使用的地址范围才需要页表页。四级结构让每个页表页刚好 4KB，与普通物理页对齐。代价是 TLB miss 时 page walk 需要 4 次内存访问（vs 2 次），但 TLB 命中率的收益远大于此。大页（2MB/1GB）是 TLB 性能的关键优化。
