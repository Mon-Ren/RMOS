长模式（Long Mode）是 x86-64 的原生 64 位操作模式，将寄存器扩展到 64 位、增加通用寄存器数量、引入 RIP 相对寻址，并基本废弃分段。

## 从 32 位到 64 位的关键变化

```
32 位 (IA-32)              64 位 (x86-64)
─────────────────────────────────────────
8 个 32 位寄存器           16 个 64 位寄存器
EAX, EBX, ECX, EDX        RAX, RBX, RCX, RDX
ESI, EDI, EBP, ESP        RSI, RDI, RBP, RSP
                           R8, R9, R10, R11
                           R12, R13, R14, R15

8 个 XMM 寄存器            16 个 XMM 寄存器 (XMM0-XMM15)

32 位地址空间 (4GB)        48 位虚拟地址 (256TB)
段分段强制                  分段基本废弃
INT 0x80 系统调用          SYSCALL/SYSRET
```

## 通用寄存器扩展

```
64 位:  RAX  RBX  RCX  RDX  RSI  RDI  RBP  RSP
32 位:  EAX  EBX  ECX  EDX  ESI  EDI  EBP  ESP
16 位:   AX   BX   CX   DX   SI   DI   BP   SP
 8 位:   AL   BL   CL   DL   AH   BH   CH   DH
新寄存器:
        R8   R9   R10  R11  R12  R13  R14  R15
32 位:  R8D  R9D  R10D R11D R12D R13D R14D R15D
16 位:  R8W  R9W  R10W R11W R12W R13W R14W R15W
 8 位:  R8B  R9B  R10B R11B R12B R13B R14B R15B
```

```nasm
; 新寄存器的使用
mov     r8, 0x123456789ABCDEF0    ; 64 位立即数
mov     r9d, 42                    ; 写 32 位自动清零高 32 位
add     r10, r11                   ; 64 位运算

; 注意：写 32 位寄存器自动零扩展到 64 位
mov     eax, 1                     ; RAX = 0x0000000000000001
; 写 8/16 位不零扩展，保留高位
```

## 默认 64 位操作数

```nasm
; 长模式下默认操作数是 64 位
mov     rax, rbx                   ; 64 位移动
add     rax, 1                     ; 64 位加
push    rax                        ; 64 位 push

; 使用 REX 前缀的 W 位选择 64 位
; 编译器自动处理

; 32 位操作仍可用
mov     eax, ebx                   ; 32 位（自动零扩展 RAX）
add     eax, 1

; 16 位需要 0x66 前缀
mov     ax, bx                     ; 16 位移动
```

## RIP 相对寻址（关键新特性）

```nasm
; RIP 相对寻址 = 地址 = RIP + 偏移
; 位置无关代码（PIC）的基础

lea     rax, [rel my_var]          ; rax = RIP + my_var 的偏移
mov     rbx, [rel my_var]          ; 读取 my_var 的值
mov     [rel my_var], rcx          ; 写入 my_var

; 绝对寻址（需要 64 位地址，编码大）
mov     rbx, [abs my_var]          ; 10 字节编码 vs RIP 相对 7 字节

section .data
    my_var dq 42
```

**RIP 相对寻址的优势**：
- 位置无关：无论加载到哪个地址都能工作
- 编码更紧凑：32 位偏移 vs 64 位绝对地址
- 共享库必须使用

## SYSCALL/SYSRET — 替代 INT

```nasm
; === 32 位系统调用 ===
mov     eax, sys_write              ; 系统调用号
mov     ebx, 1                      ; fd
mov     ecx, msg                    ; buf
mov     edx, len                    ; count
int     0x80                        ; 陷入内核

; === 64 位系统调用 ===
; SYSCALL 直接跳转到 LSTAR MSR 指定的地址
; 更快（不需要查 IDT），但参数寄存器不同

mov     rax, 1                      ; sys_write (注意：64 位号不同！)
mov     rdi, 1                      ; fd (第一个参数用 RDI，不是 RDX)
mov     rsi, msg                    ; buf
mov     rdx, len                    ; count
syscall                             ; 快速系统调用

; 返回值在 RAX
; 破坏 RCX 和 R11（SYSCALL 保存返回地址到 RCX）
```

### 64 位系统调用参数约定（Linux）

```
32 位 (INT 0x80):    EAX=号, EBX, ECX, EDX, ESI, EDI, EBP
64 位 (SYSCALL):     RAX=号, RDI, RSI, RDX, R10, R8, R9

注意：
- 系统调用号不同！write 在 32 位是 4，64 位是 1
- 参数寄存器顺序不同（ABI 差异）
```

## 分段基本废弃

```nasm
; 长模式下大部分段基址固定为 0
; CS, DS, ES, SS 的 Base = 0（忽略 GDT 中的 Base）
; FS 和 GS 例外 — 可设置非零基址（用于线程本地存储）

; 设置 GS 基址（用于线程本地存储）
mov     ecx, 0xC0000101           ; MSR: GS_BASE
mov     eax, tls_addr_low
mov     edx, tls_addr_high
wrmsr

; 读取 GS 相关数据
mov     rax, [gs:0x10]            ; 通过 GS 段访问 TLS
```

## 64 位调用约定

```
System V AMD64 ABI（Linux/macOS）:
  参数: RDI, RSI, RDX, RCX, R8, R9（其余走栈）
  返回: RAX
  调用者保存: RAX, RCX, RDX, RSI, RDI, R8-R11
  被调用者保存: RBX, RBP, R12-R15

Microsoft x64 ABI（Windows）:
  参数: RCX, RDX, R8, R9（其余走栈）
  返回: RAX
  调用者保存: RAX, RCX, RDX, R8-R11
  被调用者保存: RBX, RBP, RDI, RSI, R12-R15
  要求 32 字节影子空间
```

```nasm
; System V ABI 函数调用示例
; long add(long a, long b, long c)
; a=rdi, b=rsi, c=rdx
add_three:
    lea     rax, [rdi + rsi]
    add     rax, rdx
    ret

; 调用
    mov     rdi, 10
    mov     rsi, 20
    mov     rdx, 30
    call    add_three      ; 返回值在 RAX = 60

; 带栈参数的调用
    sub     rsp, 8         ; 对齐栈（16 字节对齐）
    push    qword 50       ; 第 7 个参数
    mov     rdi, 1
    mov     rsi, 2
    mov     rdx, 3
    mov     rcx, 4
    mov     r8, 5
    mov     r9, 6
    call    seven_args
    add     rsp, 16        ; 恢复栈
```

## 页表扩展

```
32 位页表：2 级（PDE → PTE）
64 位页表：4 级（PML4 → PDP → PD → PT）

虚拟地址分解（48 位）：
┌────────┬────────┬────────┬────────┬──────────┐
│ PML4   │ PDP    │ PD     │ PT     │ Offset   │
│ [47:39]│[38:30] │[29:21] │[20:12] │ [11:0]   │
│ 9 位   │ 9 位   │ 9 位   │ 9 位   │ 12 位    │
└────────┴────────┴────────┴────────┴──────────┘
```

## 关键要点

> **16 个通用寄存器 + 16 个 XMM**：比 32 位多了一倍，减少栈溢出，函数调用更高效。

> **RIP 相对寻址**是位置无关代码的基础——64 位共享库必须使用，编码也更紧凑。

> **SYSCALL 替代 INT 0x80**：更快（不查 IDT），但参数约定不同且破坏 RCX/R11。

> **分段基本废弃**：CS/DS/ES/SS 基址为 0，只有 FS/GS 用于 TLS。平坦内存模型。

> **4 级页表**：支持 48 位虚拟地址（256TB），最多 52 位物理地址（4PB）。
