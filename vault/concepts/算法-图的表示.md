# 图的表示

图 (Graph) 由顶点 V 和边 E 组成，选择合适的表示方式直接影响算法效率。

## 基本概念

- **无向图**：边无方向，(u,v) = (v,u)
- **有向图**：边有方向，(u,v) ≠ (v,u)
- **加权图**：边带权重
- **度**：与顶点相连的边数（有向图分入度/出度）

## 三种表示方式

### 1. 邻接矩阵

```python
# graph[u][v] = 1（或权重）表示有边，0 表示无边
n = 5
graph = [[0] * n for _ in range(n)]

# 添加边
def add_edge(u, v, w=1):
    graph[u][v] = w
    graph[v][u] = w  # 无向图

# 检查边：O(1)
def has_edge(u, v):
    return graph[u][v] != 0

# 遍历邻居：O(V)
def neighbors(u):
    return [v for v in range(n) if graph[u][v]]
```

**空间**：O(V²)

### 2. 邻接表

```python
# 每个顶点维护一个邻居列表
graph = [[] for _ in range(n)]

# 无权图
def add_edge(u, v):
    graph[u].append(v)
    graph[v].append(u)

# 加权图
graph = [[] for _ in range(n)]
def add_edge(u, v, w):
    graph[u].append((v, w))
    graph[v].append((u, w))

# 遍历邻居：O(degree(u))
def neighbors(u):
    return graph[u]
```

**空间**：O(V + E)

### 3. 边列表

```python
# 存储所有边的列表
edges = []

def add_edge(u, v, w=1):
    edges.append((u, v, w))

# 适合 Kruskal 等需要按边权排序的算法
edges.sort(key=lambda x: x[2])
```

**空间**：O(E)

## 对比

| 特性 | 邻接矩阵 | 邻接表 | 边列表 |
|------|----------|--------|--------|
| 空间 | O(V²) | O(V+E) | O(E) |
| 检查边 | ✅ O(1) | O(degree) | O(E) |
| 遍历邻居 | O(V) | ✅ O(degree) | — |
| 添加边 | O(1) | O(1) | O(1) |
| 删除边 | O(1) | O(degree) | O(E) |
| 稠密图 | ✅ 适合 | 可以 | 可以 |
| 稀疏图 | ❌ 浪费 | ✅ 适合 | ✅ 适合 |

## 适用场景

```python
# 邻接矩阵：
# - 稠密图（E ≈ V²）
# - 频繁检查边是否存在
# - Floyd-Warshall 等需要矩阵运算的算法

# 邻接表：
# - 稀疏图（E << V²）—— 最常见
# - BFS、DFS 遍历
# - Dijkstra、Prim 等需要遍历邻居的算法

# 边列表：
# - Kruskal（按边权排序）
# - Bellman-Ford（遍历所有边）
```

## 复杂度总结

| 算法 | 最佳表示 |
|------|----------|
| BFS / DFS | 邻接表 O(V+E) |
| Dijkstra | 邻接表 + 优先队列 |
| Floyd-Warshall | 邻接矩阵 O(V³) |
| Kruskal | 边列表 O(E log E) |
| Prim | 邻接表 + 优先队列 |

## 关键要点

> [!important] 稀疏图用邻接表
> 现实中大多数图是稀疏的（E << V²），邻接表是最常用的选择

> [!important] 邻接矩阵的优势是 O(1) 判断边
> 适合稠密图或需要频繁查边的场景

> [!important] 空间是选择的关键
> 10⁵ 个顶点的邻接矩阵需要 10¹⁰ 个元素，不可行
