# x86 调用约定 cdecl

调用约定定义了函数调用时参数传递、返回值、栈清理的规则。x86-32 常用的是 **cdecl**。

## cdecl 规则

### 参数传递
- 所有参数通过**栈**传递（从右到左压栈）
- 调用者负责清理栈（`add esp, N` 或 `ret N`）

### 寄存器使用

| 寄存器 | 保存方 | 用途 |
|--------|--------|------|
| EAX | 调用者 | 返回值 |
| ECX | 调用者 | 临时 |
| EDX | 调用者 | 临时 |
| EBX | **被调用者** | 必须保存 |
| ESI | **被调用者** | 必须保存 |
| EDI | **被调用者** | 必须保存 |
| EBP | **被调用者** | 栈帧基址 |
| ESP | 特殊 | 栈指针 |

### 调用示例

```c
int add(int a, int b) { return a + b; }
// 调用: add(3, 5)
```

```asm
push 5       ; b (右→左)
push 3       ; a
call add     ; 压入返回地址，跳转
add esp, 8   ; 调用者清理 2 个参数

; add 函数内部:
push ebp
mov ebp, esp
mov eax, [ebp+8]   ; a = 3
add eax, [ebp+12]  ; + b = 5
pop ebp
ret
```

## 栈布局（调用 add(3,5) 时）

```
高地址
┌──────────────┐
│      5       │ [ebp+12] b
├──────────────┤
│      3       │ [ebp+8]  a
├──────────────┤
│  返回地址     │ [ebp+4]  call 压入
├──────────────┤
│  旧 ebp      │ [ebp]    push ebp
├──────────────┤
│  局部变量...  │ [ebp-4]  ...
低地址
```

## 为什么 xv6 用 cdecl？

- 系统调用参数从用户栈提取：`argint(n, &ip)` → `fetchint(tf->esp + 4 + 4*n)`
- 每个参数 4 字节，固定偏移量计算简单
- 调用者清理栈 → 支持变参函数（如 printf）

## 其他调用约定

| 约定 | 参数传递 | 栈清理 | 平台 |
|------|----------|--------|------|
| cdecl | 栈（右→左） | 调用者 | C 默认 |
| stdcall | 栈（右→左） | 被调用者 | Win32 API |
| fastcall | ECX, EDX + 栈 | 被调用者 | MSVC |
| System V AMD64 | RDI, RSI, RDX, RCX, R8, R9 | 调用者 | x86-64 Linux |

## 关键要点

> cdecl 是 x86-32 上最简单的调用约定——所有参数走栈，调用者清理。这让 xv6 的系统调用参数提取变得直观：`trapframe->esp + 偏移量` 就能拿到第 N 个参数。理解调用约定是读懂汇编与 C 交互的基础。
