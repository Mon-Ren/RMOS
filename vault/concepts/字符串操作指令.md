字符串操作指令（String Instructions）以 ESI/EDI 为隐式操作数，配合 REP 前缀可高效处理内存块操作。

## 隐式寄存器

| 指令类型 | 源地址 | 目的地址 |
|---------|--------|---------|
| 默认 | ESI (DS 段) | EDI (ES 段) |
| 段覆盖 | fs:[ESI] 等 | — |

所有字符串指令自动根据 [[标志寄存器 EFLAGS|DF 标志]] 更新 ESI/EDI：
- **DF = 0**：ESI/EDI **递增**（cld）
- **DF = 1**：ESI/EDI **递减**（std）

## 传送指令 — MOVSB/MOVSW/MOVSD

从 [ESI] 复制到 [EDI]，然后更新 ESI 和 EDI。

```asm
; MOVSB — 复制 1 字节
; MOVSW — 复制 2 字节（word）
; MOVSD — 复制 4 字节（dword）

cld                  ; 方向：递增
lea esi, [source]
lea edi, [dest]
movsb                ; [EDI] = [ESI], ESI++, EDI++
movsd                ; [EDI] = [ESI] (4字节), ESI+=4, EDI+=4

; 批量复制：配合 REP
mov ecx, 256         ; 复制 256 个 dword = 1024 字节
rep movsd            ; 重复 ECX 次 MOVSD
```

## 比较指令 — CMPSB/CMPSW/CMPSD

比较 [ESI] 和 [EDI]，设标志位。

```asm
; 比较两个字符串是否相等
lea esi, [str1]
lea edi, [str2]
mov ecx, 20          ; 比较 20 字节
cld
repe cmpsb           ; 重复比较直到不相等或 ECX=0
jz  strings_equal    ; ZF=1 → 完全相等

; 检查前 N 字节是否相等
mov ecx, 8
repe cmpsd           ; 比较 8 个 dword = 32 字节
```

## 扫描指令 — SCASB/SCASW/SCASD

在 [EDI] 开始的内存中扫描 AL/AX/EAX 的值。

```asm
; 在字符串中查找字符 'X'
lea edi, [buffer]
mov al, 'X'
mov ecx, 100         ; 最多扫描 100 字节
cld
repne scasb          ; 重复扫描直到找到或 ECX=0
jnz not_found        ; ZF=0 → 未找到
    ; EDI 指向找到的字符的下一个位置
    sub edi, 1       ; 回退到匹配位置

; 计算字符串长度（查找 '\0'）
lea edi, [str]
xor al, al           ; AL = 0
mov ecx, -1          ; 最大计数
repne scasb          ; 扫描直到找到 0
not ecx              ; ECX = ~ECX
dec ecx              ; ECX = 字符串长度（不含 '\0'）
```

## 加载/存储指令 — LODSB/STOSB

### LODSB/LODSW/LODSD — 从 [ESI] 加载到 AL/AX/EAX

```asm
lea esi, [data]
lodsb                ; AL = [ESI], ESI++
lodsd                ; EAX = [ESI], ESI+=4

; 逐字节处理
lea esi, [buffer]
mov ecx, 100
process_loop:
    lodsb            ; AL = [ESI++]
    ; ... 处理 AL ...
    dec ecx
    jnz process_loop
```

### STOSB/STOSW/STOSD — 将 AL/AX/EAX 存储到 [EDI]

```asm
; 用 0 填充 1024 字节
lea edi, [buffer]
xor al, al           ; AL = 0
mov ecx, 1024
cld
rep stosb            ; [EDI++] = 0, 重复 1024 次

; 用特定值填充 DWORD 数组
lea edi, [array]
mov eax, 0xDEADBEEF
mov ecx, 256
rep stosd            ; array[0..255] = 0xDEADBEEF
```

## REP 前缀族

| 前缀 | 全名 | 终止条件 |
|------|------|---------|
| **REP** | Repeat | ECX = 0 |
| **REPE/REPZ** | Repeat while Equal | ECX = 0 **或** ZF = 0 |
| **REPNE/REPNZ** | Repeat while Not Equal | ECX = 0 **或** ZF = 1 |

```asm
; REP — 无条件重复
rep movsb            ; 复制 ECX 字节

; REPE — 相等时继续（用于比较）
repe cmpsb           ; 比较直到不相等或计数完

; REPNE — 不等时继续（用于搜索）
repne scasb          ; 搜索直到找到或计数完
```

> [!info] REP 前缀的工作原理
> 1. 检查 ECX 是否为 0 → 是则退出
> 2. 执行一次字符串指令
> 3. ECX--
> 4. （REPE/REPNE 额外检查 ZF）
> 5. 回到步骤 1

## 综合示例：内存复制函数

```asm
; memcpy(dst, src, len)
; 参数：[ebp+8]=dst, [ebp+12]=src, [ebp+16]=len
memcpy:
    push ebp
    mov ebp, esp
    push esi
    push edi
    
    mov edi, [ebp+8]      ; dst
    mov esi, [ebp+12]     ; src
    mov ecx, [ebp+16]     ; len
    cld
    
    ; 先复制 DWORD
    mov eax, ecx
    shr ecx, 2            ; len / 4
    rep movsd
    
    ; 再复制剩余字节
    mov ecx, eax
    and ecx, 3            ; len % 4
    rep movsb
    
    pop edi
    pop esi
    pop ebp
    ret
```

> [!tip] 关键要点
> - 字符串指令隐式使用 ESI/EDI，自动增减
> - DF 控制方向：cld=递增，std=递减
> - REP 前缀让单条指令处理整个内存块
> - REPE/REPNE 用于比较和搜索场景
> - 实际优化中，现代编译器常用 SSE/AVX 替代 REP 指令
