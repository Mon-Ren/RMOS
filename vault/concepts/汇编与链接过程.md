汇编源码从 `.s` 文件到可执行程序，经过**汇编**（.s → .o）和**链接**（多个 .o → 可执行文件）两个关键阶段，理解这个过程是掌握底层开发的基础。

## 整体流程

```
  .asm/.s 源码
      │
      ▼  [汇编器 as / nasm]
  .o 目标文件 (ELF Relocatable)
      │
      ▼  [链接器 ld / gcc]
  可执行文件 (ELF Executable)
      │
      ▼  [加载器 kernel]
  进程地址空间
```

## 阶段一：汇编

```bash
# GAS 汇编
as --32 hello.s -o hello.o
# 或用 gcc（自动处理预处理和汇编）
gcc -c -m32 hello.s -o hello.o

# NASM 汇编
nasm -f elf32 hello.s -o hello.o
nasm -f elf64 hello.s -o hello.o    # 64 位
```

汇编器做的事：
1. **解析指令**：将助记符翻译为机器码
2. **计算偏移**：段内地址暂时用占位符
3. **生成符号表**：记录每个符号的定义位置
4. **生成重定位表**：标记需要链接器填入的地址

### 目标文件 (.o) 的内容

```bash
# 查看目标文件信息
objdump -d hello.o          # 反汇编
objdump -t hello.o          # 符号表
objdump -r hello.o          # 重定位表
readelf -S hello.o          # 段头表
readelf -s hello.o          # 符号详情
```

目标文件包含：

| 节区 | 内容 |
|------|------|
| `.text` | 机器码（未重定位） |
| `.data` | 已初始化数据 |
| `.bss` | 未初始化数据的大小信息 |
| `.symtab` | 符号表 |
| `.rel.text` | 代码段的重定位条目 |
| `.rel.data` | 数据段的重定位条目 |
| `.strtab` | 符号名字符串表 |

## 阶段二：链接

```bash
# 链接多个目标文件
ld -m elf_i386 -o program main.o utils.o math.o

# 或通过 gcc（自动链接 C 库）
gcc -m32 -o program main.o utils.o math.o

# 查看最终符号
nm program                  # 符号表
ldd program                 # 动态库依赖
```

链接器做的事：
1. **段合并**：所有 `.o` 的 `.text` 合成一个，`.data` 合成一个，等等
2. **符号解析**：每个未定义符号找到对应的定义
3. **重定位**：将占位地址替换为最终地址
4. **生成入口点**：设置 `_start` 为程序入口

## 符号解析详解

```c
// main.c
extern int shared_var;      // 未定义符号
extern void helper(void);   // 未定义符号

int main_var = 10;          // 已定义全局符号

int main(void) {
    helper();               // 引用外部符号
    return shared_var;
}
```

```nasm
; helper.s
global helper               ; 定义符号
global shared_var

section .data
    shared_var dd 42

section .text
helper:
    ret
```

```bash
# 链接过程
gcc -c main.c -o main.o       # main.o 引用 helper 和 shared_var
nasm -f elf32 helper.s -o helper.o  # helper.o 定义这两个符号
gcc -m32 main.o helper.o -o program # 链接器匹配引用与定义
```

### 符号解析规则

```
强符号（函数体、已初始化全局变量）→ 只能有一个定义
弱符号（未初始化全局变量）→ 可以合并
未定义符号 → 必须在某个 .o 中找到定义
```

## 多文件链接示例

```nasm
; === add.s ===
section .text
global add_numbers
add_numbers:
    mov eax, [esp+4]
    add eax, [esp+8]
    ret

; === main.s ===
section .text
global _start
extern add_numbers

_start:
    push dword 10
    push dword 20
    call add_numbers     ; 调用外部函数
    add esp, 8

    ; exit(eax)
    mov ebx, eax
    mov eax, 1
    int 0x80
```

```bash
nasm -f elf32 add.s -o add.o
nasm -f elf32 main.s -o main.o
ld -m elf_i386 -o program main.o add.o
./program
echo $?    # 输出 30
```

## 静态库 (.a) 链接

```bash
# 创建静态库
ar rcs libmath.a add.o sub.o mul.o div.o

# 链接静态库
ld -m elf_i386 main.o -L. -lmath -o program
# 等价于
ld -m elf_i386 main.o libmath.a -o program
```

## 关键要点

> **汇编器**把 `.s` 变成 `.o`（目标文件），包含机器码 + 符号表 + 重定位表。地址都是占位符。

> **链接器**合并所有 `.o` 的段，解析符号（引用 → 定义），执行重定位（填入最终地址）。

> **符号解析**是链接的核心：每个 undefined symbol 必须在某个 .o 中找到 global 定义，否则报 `undefined reference`。

> **静态库** (`.a`) 是 `.o` 文件的打包，链接器按需提取——只复制被引用的符号所在的 .o。
